# 70620.3 언팩킹 및 복호화 메커니즘

## 1. 복호화 루틴 역공학 (Reverse Engineering)
대부분의 드롭퍼는 정적 분석을 방해하기 위해 내부 페이로드를 암호화하여 저장합니다. 이를 분석하기 위해서는 코드 실행 흐름에서 복호화 알고리즘과 키를 찾아내야 합니다.

### 주요 복호화 패턴
*   **XOR 연산:** 가장 흔한 방식으로, 단순 루프를 통해 데이터와 특정 키를 XOR 합니다.
*   **표준 알고리즘:** AES, RC4 등 표준 라이브러리를 사용하거나 직접 구현한 알고리즘을 사용합니다.
*   **커스텀 인코딩:** Base64 변형이나 비트 시프트(Bit Shift) 연산을 조합합니다.

## 2. 메모리 덤프(Memory Dump) 기법
복호화 루틴을 완전히 분석하는 것은 시간이 많이 걸릴 수 있습니다. 실무에서는 복호화가 완료된 직후, 페이로드가 메모리에 상주하는 시점을 노려 데이터를 추출합니다.

### 덤프 절차
1.  **중단점(Breakpoint) 설정:** `WriteProcessMemory`, `VirtualAllocEx` 또는 복호화 루프의 끝부분에 하드웨어 중단점을 설정합니다.
2.  **메모리 영역 식별:** 복호화된 결과물이 저장된 메모리 주소와 크기를 파악합니다.
3.  **데이터 추출:** 디버거(x64dbg 등)의 'Dump' 기능을 사용하거나 스크립트를 통해 파일로 저장합니다.

## 3. [Go] 단순 XOR 복호화 도구 예제
분석을 통해 알고리즘과 키를 알아냈을 때, 이를 자동화하여 복호화하는 Go 언어 스크립트입니다.

```go
package main

import (
	"fmt"
	"io/ioutil"
	"log"
)

// decryptXOR 함수는 주어진 바이트 슬라이스를 키와 XOR 연산하여 복호화합니다.
func decryptXOR(data []byte, key byte) []byte {
	decrypted := make([]byte, len(data))
	for i := 0; i < len(data); i++ {
		decrypted[i] = data[i] ^ key
	}
	return decrypted
}

func main() {
	// 1. 암호화된 파일 로드 (예: 리소스에서 추출한 .bin 파일)
	encryptedFile := "extracted_payload.bin"
	data, err := ioutil.ReadFile(encryptedFile)
	if err != nil {
		log.Fatalf("[-] Failed to read file: %v", err)
	}

	fmt.Printf("[*] Loaded %d bytes from %s\n", len(data), encryptedFile)

	// 2. 분석을 통해 찾아낸 XOR 키 (예: 0x4A)
	var xorKey byte = 0x4A

	// 3. 복호화 수행
	decryptedData := decryptXOR(data, xorKey)

	// 4. 결과 저장
	outputFile := "decrypted_payload.exe"
	err = ioutil.WriteFile(outputFile, decryptedData, 0644)
	if err != nil {
		log.Fatalf("[-] Failed to write decrypted file: %v", err)
	}

	fmt.Printf("[+] Decryption complete. Saved to %s\n", outputFile)
	
	// 5. 복호화 성공 여부 확인 (Magic Byte 확인)
	if len(decryptedData) > 2 && string(decryptedData[:2]) == "MZ" {
		fmt.Println("[!] Success! PE signature (MZ) detected in decrypted data.")
	}
}
```

## 4. 상세 분석 기술: API 트레이싱
복호화 과정에서 호출되는 시스템 API를 추적하면 실마리를 찾을 수 있습니다.

*   **CryptDecrypt / CryptDeriveKey:** Windows Crypto API를 사용하는 경우입니다.
*   **VirtualAlloc / VirtualProtect:** 복호화된 데이터를 담을 메모리 공간을 할당하고 권한을 변경할 때 사용됩니다.
*   **memcpy / memmove:** 복호화된 데이터를 최종 위치로 복사할 때 호출됩니다.

### 분석 팁
*   디버거에서 `VirtualAlloc` 호출 후 반환된 메모리 주소(EAX/RAX)를 계속 주시하십시오.
*   해당 메모리 영역에 데이터가 써지는 시점에 'Memory Breakpoint on Access'를 걸면 복호화 루틴을 바로 찾을 수 있습니다.
