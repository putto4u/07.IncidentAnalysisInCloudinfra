# 70620.2 정적 분석: 리소스 식별 및 추출

## 1. PE 리소스(Resource) 분석
PE(Portable Executable) 파일 포맷은 아이콘, 메뉴, 대화 상자뿐만 아니라 사용자 정의 데이터(RCDATA)를 저장할 수 있는 `.rsrc` 섹션을 제공합니다. 드롭퍼는 이 영역에 실행 파일이나 스크립트를 숨기는 경우가 많습니다.

### 분석 포인트
*   **리소스 트리 구조:** `IMAGE_RESOURCE_DIRECTORY`를 통해 계층적으로 관리됩니다.
*   **사용자 정의 리소스:** `RT_RCDATA` 또는 `RT_BINARY` 타입으로 저장된 대용량 데이터는 페이로드일 확률이 높습니다.
*   **시각화 도구:** Resource Hacker, CFF Explorer 등을 사용하여 육안으로 확인 가능합니다.

## 2. 오버레이(Overlay) 데이터 분석
오버레이는 PE 헤더가 정의한 파일 크기 이후에 덧붙여진 데이터를 의미합니다. OS의 로더는 이 데이터를 메모리에 로드하지 않지만, 프로그램 내부에서 파일 포인터를 이동시켜 직접 읽어들일 수 있습니다.

### 특징
*   PE 구조 밖의 데이터이므로 일반적인 정적 분석 도구에서 섹션 정보로 나타나지 않습니다.
*   주로 파일 배포 패키지(SFX)나 드롭퍼에서 추가 페이로드를 저장할 때 사용됩니다.

## 3. [Python] 리소스 및 오버레이 추출 도구 작성
`pefile` 라이브러리를 사용하여 PE 파일의 오버레이와 특정 리소스를 자동으로 추출하는 스크립트 예제입니다.

```python
import pefile
import os

def analyze_pe(file_path):
    try:
        pe = pefile.PE(file_path)
        print(f"[*] Analyzing: {file_path}")

        # 1. 오버레이 데이터 확인
        overlay_offset = pe.get_overlay_data_start_offset()
        if overlay_offset:
            overlay_data = pe.get_overlay()
            print(f"[!] Overlay detected! Size: {len(overlay_data)} bytes")
            with open("extracted_overlay.bin", "wb") as f:
                f.write(overlay_data)
            print("[+] Saved overlay to 'extracted_overlay.bin'")
        else:
            print("[-] No overlay found.")

        # 2. 리소스 섹션 분석
        if hasattr(pe, 'DIRECTORY_ENTRY_RESOURCE'):
            print("[*] Resources found:")
            for resource_type in pe.DIRECTORY_ENTRY_RESOURCE.entries:
                if resource_type.name is not None:
                    name = str(resource_type.name)
                else:
                    name = pefile.RESOURCE_TYPE.get(resource_type.id, "UNKNOWN")
                
                for entry in resource_type.directory.entries:
                    if hasattr(entry, 'directory'):
                        for sub_entry in entry.directory.entries:
                            data_rva = sub_entry.data.struct.OffsetToData
                            size = sub_entry.data.struct.Size
                            print(f"  - Type: {name}, RVA: {hex(data_rva)}, Size: {size}")
                            
                            # 특정 조건(예: 크기가 10KB 이상)인 리소스 자동 추출
                            if size > 10240:
                                data = pe.get_data(data_rva, size)
                                output_name = f"res_{name}_{hex(data_rva)}.bin"
                                with open(output_name, "wb") as f:
                                    f.write(data)
                                print(f"    [+] Large resource extracted to {output_name}")

    except Exception as e:
        print(f"[-] Error: {e}")

if __name__ == "__main__":
    # 실행 전 pip install pefile 필요
    target_file = "sample_dropper.exe"
    if os.path.exists(target_file):
        analyze_pe(target_file)
    else:
        print(f"[-] File not found: {target_file}")
```

## 4. 상세 설명 및 실무 팁
*   **엔트로피(Entropy) 측정:** 페이로드가 암호화되어 있다면 리소스 섹션의 엔트로피는 7.5 이상으로 매우 높게 나타납니다.
*   **파일 시그니처 확인:** 추출한 바이너리 데이터의 시작 부분(Magic Byte)을 확인하여 그것이 또 다른 PE(`MZ`), ZIP(`PK`), 또는 특정 스크립트인지 식별합니다.
*   **API 연계 분석:** 정적 분석에서 추출한 데이터가 실제 코드 내에서 `FindResource`, `LoadResource`, `LockResource` API를 통해 참조되는지 확인해야 합니다.
