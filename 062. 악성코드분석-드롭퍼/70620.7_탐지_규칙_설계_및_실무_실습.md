# 70620.7 탐지 규칙 설계 및 실무 실습

드롭퍼(Dropper) 분석의 최종 목적은 해당 위협을 식별하고 차단할 수 있는 유효한 **탐지 규칙(Detection Rules)**을 생성하는 것입니다. 본 섹션에서는 YARA, Sigma 및 IDS 규칙을 설계하는 방법과 실무 실습 과정을 다룹니다.

## 1. 탐지 지표(IoC) 식별

규칙을 설계하기 전, 분석 과정에서 획득한 지표를 분류해야 합니다.

- **호스트 기반 지표**: 파일 해시(MD5/SHA256), 뮤텍스(Mutex), 특정 파일 경로, 레지스트리 키.
- **네트워크 기반 지표**: C2 서버 IP/도메인, 특정 User-Agent, 패킷 페이로드 내 고유 패턴.
- **행위 기반 지표**: 특정 API 호출 순서, 프로세스 인젝션 행위, 부모-자식 프로세스 관계.

## 2. YARA 규칙 설계 (정적 탐지)

YARA는 파일 내부의 고유한 문자열이나 바이너리 패턴을 탐지하는 데 최적화되어 있습니다.

**[YARA 실습: 드롭퍼 탐지 규칙 예시]**
```yara
rule Dropper_MultiStage_Detection {
    meta:
        description = "Detects multi-stage dropper using specific API calls and strings"
        author = "Security Researcher"
        date = "2025-05-20"
        severity = "High"

    strings:
        $s1 = "URLDownloadToFileA" ascii wide
        $s2 = "ShellExecuteA" ascii wide
        $s3 = "WriteProcessMemory" ascii wide
        $h1 = { 4D 5A 90 00 } // MZ Header
        $mutex = "Global\\Malware_Dropper_Mutex_123"

    condition:
        $h1 at 0 and (2 of ($s*)) and $mutex
}
```

## 3. Sigma 규칙 설계 (로그 기반 탐지)

Sigma는 시스템 로그(Sysmon, Event Log)를 기반으로 악성 행위를 탐지하는 범용 표준 포맷입니다.

**[Sigma 예시: 비정상적인 프로세스 생성 탐지]**
```yaml
title: Suspicious Dropper Child Process
id: 5f9a-1234-abcd
status: experimental
description: Detects Microsoft Office spawning PowerShell (Common Dropper behavior)
logsource:
    category: process_creation
    product: windows
detection:
    selection:
        ParentImage|endswith: 
            - '\winword.exe'
            - '\excel.exe'
        Image|endswith: 
            - '\powershell.exe'
            - '\cmd.exe'
    condition: selection
falsepositives:
    - Administrative scripts (rare)
level: high
```

## 4. 탐지 자동화 실무 실습

분석가는 대량의 파일을 처리하기 위해 탐지 과정을 자동화해야 합니다.

**[Python 실습: YARA를 이용한 디렉토리 스캔 도구]**
```python
import yara
import os

def scan_directory(dir_path, rule_path):
    # 규칙 컴파일
    rules = yara.compile(filepath=rule_path)
    
    print(f"[*] Scanning directory: {dir_path}")
    for root, dirs, files in os.walk(dir_path):
        for file in files:
            file_path = os.path.join(root, file)
            try:
                matches = rules.match(file_path)
                if matches:
                    print(f"[!] MATCHED: {file_path} -> {matches}")
            except Exception as e:
                continue

if __name__ == "__main__":
    # 실제 환경에 맞춰 경로 설정
    # scan_directory("C:\\Downloads", "dropper_rules.yar")
    pass
```

## 6. 결론 및 베스트 프랙티스

- **오탐(False Positive) 관리**: 너무 일반적인 문자열(예: "kernel32.dll")은 제외하고 고유한 패턴을 사용하십시오.
- **성능 최적화**: YARA 규칙에서 `$s*`와 같은 와일드카드는 성능을 저하시킬 수 있으므로 주의해서 사용하십시오.
- **버전 관리**: 위협 그룹의 변종 생성에 대비하여 규칙을 주기적으로 업데이트하고 버전화하십시오.
