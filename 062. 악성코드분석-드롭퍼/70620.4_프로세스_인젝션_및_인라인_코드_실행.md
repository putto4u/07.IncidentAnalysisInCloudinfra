# 70620.4 프로세스 인젝션 및 인라인 코드 실행

## 1. 프로세스 할로잉 (Process Hollowing)
프로세스 할로잉은 정상적인 프로세스(예: `explorer.exe`, `svchost.exe`)를 일시 정지 상태로 생성한 뒤, 그 내부의 코드를 비우고 악성 페이로드로 교체하여 실행하는 기법입니다.

### 동작 단계
1.  **Create Process:** 대상 프로세스를 `CREATE_SUSPENDED` 플래그로 생성합니다.
2.  **Unmap Memory:** `NtUnmapViewOfSection`을 사용하여 대상 프로세스의 메모리 영역을 비웁니다.
3.  **Allocate Memory:** `VirtualAllocEx`로 악성 코드가 들어갈 공간을 확보합니다.
4.  **Write Payload:** `WriteProcessMemory`로 할당된 공간에 악성 페이로드를 씁니다.
5.  **Set Thread Context:** `SetThreadContext`를 사용하여 스레드의 진입점(Entry Point)을 악성 코드가 시작되는 위치로 변경합니다.
6.  **Resume Thread:** `ResumeThread`를 호출하여 실행을 재개합니다.

## 2. DLL 인젝션 (DLL Injection)
실행 중인 다른 프로세스의 주소 공간에 악성 DLL을 강제로 로드시키는 기법입니다.

### 핵심 API 흐름
*   `OpenProcess`: 대상 프로세스의 핸들을 획득합니다.
*   `VirtualAllocEx`: 대상 프로세스 내부에 DLL 경로 문자열을 저장할 공간을 할당합니다.
*   `WriteProcessMemory`: 할당된 공간에 DLL 파일 경로를 씁니다.
*   `CreateRemoteThread`: 대상 프로세스 내에서 `LoadLibrary`를 실행하도록 원격 스레드를 생성합니다.

## 3. [Go] Process Hollowing 개념 구현 (시뮬레이션)
Windows API를 직접 호출하여 프로세스 할로잉의 핵심 개념을 구현한 Go 예제입니다. (실제 동작을 위해서는 `golang.org/x/sys/windows` 라이브러리 연동이 필요합니다.)

```go
package main

import (
	"fmt"
	"syscall"
	"unsafe"
)

// 핵심 API 정의 (간략화)
var (
	kernel32            = syscall.NewLazyDLL("kernel32.dll")
	procCreateProcess   = kernel32.NewProc("CreateProcessW")
	procVirtualAllocEx  = kernel32.NewProc("VirtualAllocEx")
	procWriteProcessMem = kernel32.NewProc("WriteProcessMemory")
	procResumeThread    = kernel32.NewProc("ResumeThread")
)

const CREATE_SUSPENDED = 0x00000004

func main() {
	target := "C:\\Windows\\System32\\notepad.exe"
	fmt.Printf("[*] Launching target in suspended state: %s\n", target)

	var si syscall.StartupInfo
	var pi syscall.ProcessInformation
	
	// 1. 일시 정지 상태로 프로세스 생성
	targetPtr, _ := syscall.UTF16PtrFromString(target)
	success, _, _ := procCreateProcess.Call(
		0,
		uintptr(unsafe.Pointer(targetPtr)),
		0, 0, false,
		CREATE_SUSPENDED,
		0, 0,
		uintptr(unsafe.Pointer(&si)),
		uintptr(unsafe.Pointer(&pi)),
	)

	if success == 0 {
		fmt.Println("[-] Failed to create process.")
		return
	}
	fmt.Printf("[+] Process created. PID: %d, Thread: %d\n", pi.ProcessId, pi.ThreadId)

	// --- 실습용 가이드 ---
	// 2. NtUnmapViewOfSection 호출 (ntdll.dll)
	// 3. VirtualAllocEx로 메모리 할당
	// 4. WriteProcessMemory로 페이로드 쓰기
	// 5. Context 조정 (SetThreadContext)
	
	fmt.Println("[*] In a real attack, payload would be injected here...")

	// 6. 스레드 재개
	procResumeThread.Call(uintptr(pi.Thread))
	fmt.Println("[+] Thread resumed.")

	syscall.CloseHandle(pi.Process)
	syscall.CloseHandle(pi.Thread)
}
```

## 4. 분석 및 탐지 대응
프로세스 인젝션은 파일 시스템에 흔적을 남기지 않는 '파일리스(Fileless)' 공격의 핵심 요소이므로 탐지가 까다롭습니다.

### 탐지 포인트
*   **비정상적인 API 호출 체인:** `CreateProcess(SUSPENDED)` -> `NtUnmapViewOfSection` -> `ResumeThread` 패턴은 매우 의심스러운 행위입니다.
*   **메모리 영역 속성:** 프로세스 내부 섹션 중 `PAGE_EXECUTE_READWRITE` (RWX) 권한을 가진 영역이 존재한다면 인젝션을 의심해야 합니다.
*   **부모-자식 관계:** 평소 자식 프로세스를 생성하지 않는 프로세스(예: `lsass.exe`)가 새로운 프로세스를 생성하는지 모니터링합니다.

### 분석 팁
*   디버거에서 `ResumeThread`가 호출되기 직전에 멈추고, 대상 프로세스의 메모리 맵을 확인하여 원본 코드가 변경되었는지 대조하십시오.
*   `Process Hacker` 등의 도구에서 'Strings' 기능을 메모리 영역별로 실행하여 악성 문자열을 검색할 수 있습니다.
