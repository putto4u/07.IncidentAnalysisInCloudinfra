# 104.3 Linux 포렌식: 시스템 로그 및 구성 분석

## 1. 리눅스 로그 아티팩트 분석
리눅스는 텍스트 기반의 로그 시스템을 사용하여 사고의 흐름을 파악하기에 용이합니다.

### 1.1 Syslog 및 메인 로그
- **/var/log/syslog (또는 messages)**: 시스템 전반의 이벤트 로그.
- **/var/log/auth.log (또는 secure)**: 로그인 시도(SSH 포함), sudo 권한 획득, 계정 생성 등 인증 관련 로그.
- **/var/log/cron**: 예약된 작업 실행 기록.

### 1.2 사용자 접속 및 활동 로그
- **last (wtmp)**: 성공한 로그인 기록, 부팅/종료 시간.
- **lastb (btmp)**: 실패한 로그인 시도 기록 (무차별 대입 공격 탐지에 필수).
- **lastlog**: 각 사용자별 마지막 로그인 정보.
- **Shell History**: `.bash_history` 등 사용자가 입력한 명령어 기록 (공격자에 의해 삭제/변조될 가능성 농후).

## 2. 시스템 구성 및 변조 분석
공격자가 시스템 설정을 어떻게 바꾸었는지 확인합니다.

### 2.1 /etc 설정 파일 검사
- **/etc/passwd, /etc/shadow**: 비정상적인 UID 0 계정(Backdoor Account) 유무 확인.
- **/etc/sudoers**: 특정 사용자에게 과도한 권한이 부여되었는지 확인.
- **/etc/crontab**: 악성 스케줄링 작업 확인.

### 2.2 SUID/SGID 비정상 파일 탐색
- 일반 사용자가 실행 시 소유자(보통 root) 권한으로 실행되는 파일.
- 공격자가 권한 상승을 위해 `find / -perm -4000` 등으로 설정해둔 백도어 파일 식별 기법.

### 2.3 SSH 설정 및 키 분석
- `~/.ssh/authorized_keys` 파일에 등록된 공격자의 공개키 확인.
- 패스워드 없는 로그인을 가능하게 하는 설정 변조 확인.
