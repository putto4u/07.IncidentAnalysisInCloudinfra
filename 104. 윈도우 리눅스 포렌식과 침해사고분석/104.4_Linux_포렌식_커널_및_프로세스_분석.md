# 104.4 Linux 포렌식: 커널 및 프로세스 분석

## 1. /proc 파일 시스템 기반 분석
리눅스의 `/proc`은 메모리에 존재하는 프로세스 정보를 파일 형태로 보여주는 가상 파일 시스템입니다.

### 1.1 프로세스 정밀 조사
- **/proc/[PID]/exe**: 실행 중인 프로세스의 실제 바이너리 링크 (삭제된 파일의 경우 `(deleted)` 표시 확인).
- **/proc/[PID]/fd**: 프로세스가 열고 있는 파일 서술자 (네트워크 소켓, 로그 파일 등).
- **/proc/[PID]/cmdline**: 프로세스 시작 시 사용된 전체 명령행 인자.
- **/proc/[PID]/environ**: 프로세스에 설정된 환경 변수 (공격 도구의 설정값 포함).

## 2. 커널 수준의 침해 (Rootkit) 분석
사용자 모드 도구(ps, netstat 등)를 속이는 루트킷을 탐지하는 고난도 기법입니다.

### 2.1 로드 가능한 커널 모듈 (LKM) 분석
- `lsmod` 명령어로 현재 로드된 모듈 확인.
- 숨겨진 모듈 탐색: `/proc/modules`와 `/sys/module` 비교 분석.
- `insmod`, `rmmod` 실행 이력 및 `/lib/modules/$(uname -r)` 내의 의심스러운 `.ko` 파일 조사.

### 2.2 시스템 콜(System Call) 후킹 탐지
- 커널 함수 테이블을 가로채서 특정 파일이나 프로세스를 은닉하는 기법.
- `chkrootkit`, `rkhunter` 등의 도구 활용 및 수동 메모리 심층 분석.

## 3. 리눅스 활성 데이터 수집
- `netstat -anp` (네트워크 연결과 프로세스 매핑).
- `lsof` (열려 있는 파일 전체 목록).
- `top`, `htop` (비정상적인 CPU/메모리 점유율 확인).
