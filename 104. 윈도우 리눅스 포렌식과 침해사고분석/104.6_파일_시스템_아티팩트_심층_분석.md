# 104.6 파일 시스템 아티팩트 심층 분석

## 1. Windows NTFS vs Linux Ext4/XFS 비교
저장 매체에 기록된 데이터 구조의 차이를 이해하여 증거를 추출합니다.

### 1.1 메타데이터 구조
- **NTFS**: MFT(Master File Table)에 파일의 모든 정보를 관리. `$File_Name`, `$Standard_Information` 등 속성 분석.
- **Ext4/XFS**: Inode(아이노드) 구조체 사용. 파일 권한, 소유자, 포인터 정보 관리.

### 1.2 타임스탬프(Timestamps) 비교
- **Windows (8개)**: 생성(Creation), 수정(Modified), MFT 수정(MFT Modified), 액세스(Accessed) - $SI 및 $FN 속성 비교를 통해 타임스탬프 조작(Timestomping) 탐지.
- **Linux**: Access, Modify, Change (상태 변경), Birth (생성 - 일부 파일시스템 지원).

## 2. 저널링 및 삭제 흔적 분석
- **NTFS $LogFile & $UsnJrnl**: 파일 생성, 삭제, 이동 이력을 순차적으로 기록하여 사고 타임라인의 정밀도를 높임.
- **Linux Journaling**: Ext4의 저널 모드 분석을 통해 최근 삭제된 파일의 메타데이터 복구 가능성 검토.

## 3. 타임라인(Timeline) 생성
- **mactime (Sleuth Kit)**: 파일 시스템의 시간 정보를 추출하여 통합된 사건 연대표 생성.
- **Plaso (log2timeline)**: 파일시스템, 레지스트리, 로그를 결합한 Super-Timeline 구축.
