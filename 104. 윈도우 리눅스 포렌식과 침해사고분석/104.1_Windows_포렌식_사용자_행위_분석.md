# 104.1 Windows 포렌식: 사용자 행위 분석

## 1. 레지스트리(Registry) 심층 분석
윈도우 레지스트리는 시스템 설정, 사용자 프로필, 설치된 소프트웨어 정보 등을 담고 있는 데이터베이스로, 사고 분석의 핵심 아티팩트입니다.

### 1.1 주요 하이브 파일 및 분석 포인트
- **SYSTEM**: 시스템 이름, 하드웨어 구성, 서비스 목록, 시간대 설정 정보.
- **SOFTWARE**: 설치된 애플리케이션 정보, 운영체제 버전, 시작 프로그램.
- **SAM (Security Accounts Manager)**: 로컬 계정 정보, 그룹 정보, 마지막 로그인 시간.
- **NTUSER.DAT**: 각 사용자의 개별 설정, 최근 사용 파일(MRU), 검색어 이력.

### 1.2 분석 도구 및 기법
- **RegRipper**: 하이브 파일에서 특정 정보를 플러그인 기반으로 추출.
- **Registry Explorer (Eric Zimmerman)**: GUI 기반의 강력한 레지스트리 분석 및 타임라인 생성 도구.

## 2. 파일 접근 이력 추적 (Usage Artifacts)
사용자가 어떤 파일을 열었거나 실행했는지 확인하기 위한 아티팩트입니다.

### 2.1 LNK (Shortcut) 파일
- 사용자가 파일을 열 때 자동으로 생성되는 바로가기 파일.
- 원본 파일의 경로, 생성/수정/액세스 시간, 볼륨 일련번호 등을 포함.
- `C:\Users\%UserName%\AppData\Roaming\Microsoft\Windows\Recent` 경로에서 확인 가능.

### 2.2 Jump Lists
- 작업 표시줄 아이콘에서 우클릭 시 나타나는 '최근 항목' 리스트.
- **AutomaticDestinations**: 시스템이 자동으로 생성하는 리스트.
- **CustomDestinations**: 애플리케이션이 정의하는 리스트.
- 사용자의 빈번한 작업 패턴 파악에 유리.

### 2.3 ShellBags
- 사용자가 폴더를 탐색한 흔적을 저장하는 레지스트리 키.
- 삭제된 폴더에 대한 접근 이력도 추적 가능하며, 공격자가 생성한 임시 디렉토리 파악에 결정적 증거가 됨.
- `UsrClass.dat` 내의 `Local Settings\Software\Microsoft\Windows\Shell\Bags` 키 분석.
