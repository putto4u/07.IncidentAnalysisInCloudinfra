# 104.2 Windows 포렌식: 실행 및 지속성 흔적

## 1. 실행 이력 분석 (Execution Artifacts)
시스템에서 어떤 프로그램이 언제 실행되었는지 증명하는 것은 사고 분석의 기본입니다.

### 1.1 Prefetch
- 응용 프로그램의 로딩 속도를 높이기 위해 생성되는 파일.
- 파일명, 경로, 실행 횟수, 마지막 실행 시간(최대 8개) 정보를 포함.
- `C:\Windows\Prefetch` 경로에 `.pf` 확장자로 저장.

### 1.2 AMCache.hve
- 실행 파일의 최초 실행 시간, 파일 경로, SHA-1 해시값 등을 저장하는 하이브 파일.
- 안티포렌식으로 Prefetch가 삭제되어도 실행 흔적을 찾을 수 있는 강력한 증거물.

### 1.3 Superfetch / Sysmain
- 메모리 사용 패턴을 분석하여 자주 사용하는 앱을 예측 로딩함. 윈도우 10 이후에는 압축된 메모리 덤프 형태로 존재하기도 함.

## 2. 공격 지속성(Persistence) 분석
공격자가 시스템 재부팅 후에도 권한을 유지하기 위해 사용하는 기법들을 추적합니다.

### 2.1 서비스(Services) 및 작업 스케줄러(Task Scheduler)
- **Services**: `SYSTEM` 하이브의 `CurrentControlSet\Services` 분석. 비정상적인 실행 경로를 가진 서비스 식별.
- **Scheduled Tasks**: `C:\Windows\System32\Tasks` 경로의 XML 파일 분석. 특정 시간에 실행되는 악성 스크립트 탐지.

### 2.2 WMI (Windows Management Instrumentation) 이벤트 가입
- 파일 시스템에 흔적을 거의 남기지 않는 고도화된 지속성 기법.
- `FilterToConsumerBinding`을 통해 특정 조건(예: 부팅 후 5분) 시 악성 코드 실행.
- `OBJECTS.DATA` (WMI 리포지토리) 파일을 분석하여 탐지 가능.

### 2.3 Autoruns 분석 기법
- 레지스트리 `Run`, `RunOnce` 키 외에도 100여 개 이상의 시작 지점을 일괄 검사하는 도구(Sysinternals Autoruns) 활용법.
