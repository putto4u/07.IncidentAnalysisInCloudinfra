# 102.4 메모리 상주 및 파일리스 공격 분석

## 1. 파일리스(Fileless) 공격의 본질
디스크에 악성 실행 파일을 생성하지 않고, 시스템의 정상 도구(PowerShell, WMI 등)를 이용해 메모리 상에서만 동작하는 공격입니다. 전통적인 파일 기반 백신을 우회하는 것이 주 목적입니다.

## 2. 주요 공격 메커니즘
- **Code Injection**: 정상 프로세스(explorer.exe, svchost.exe 등)의 메모리 공간에 악성 코드를 삽입하여 실행.
- **Reflective DLL Injection**: 디스크를 거치지 않고 메모리에서 직접 DLL을 로드.
- **Process Hollowing**: 정상 프로세스를 일시 중단 상태로 실행한 뒤, 내부 코드를 악성 코드로 교체하고 재개.

## 3. 포렌식적 접근: 메모리 포렌식
이러한 공격을 탐지하는 유일한 방법은 RAM을 분석하는 것입니다.
- **Volatility 활용 핵심 플러그인**:
    - `pslist`, `pstree`: 비정상적인 부모-자식 관계 확인.
    - `malfind`: VAD(Virtual Address Descriptor) 태그와 권한(PAGE_EXECUTE_READWRITE) 분석을 통해 주입된 코드 영역 식별.
    - `netscan`: 활성화된 C2 통신 소켓 확인.
    - `ldrmodules`: 프로세스가 로드한 DLL 중 디스크에 경로가 없는(은닉된) 항목 검색.

## 4. 지속성 확보 기법 분석
파일이 없어도 재부팅 후 다시 실행되어야 하므로, 흔적은 결국 어딘가에 남습니다.
- **레지스트리**: `Run` 키에 포함된 난독화된 PowerShell 스크립트.
- **WMI Event Subscription**: 특정 조건(시스템 가동 후 5분 등) 만족 시 악성 코드 실행.
- **Scheduled Tasks**: 정상적인 이름으로 위장된 예약 작업.

## 5. 대응 전략
- **AMSI (Antimalware Scan Interface)**: 메모리 내 스크립트 실행 시 백신이 검사할 수 있도록 로깅 강화.
- **Script Block Logging**: PowerShell의 실제 수행 명령을 전체 기록하도록 설정.
- **EDR 활용**: 프로세스 메모리의 동적 변화를 실시간 모니터링.
