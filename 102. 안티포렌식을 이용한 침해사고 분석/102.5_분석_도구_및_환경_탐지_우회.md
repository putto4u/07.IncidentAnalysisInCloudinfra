# 102.5 분석 도구 및 환경 탐지 우회

## 1. 안티 VM (Anti-VM) 및 안티 샌드박스
악성코드가 자신이 분석 환경(VMware, VirtualBox, Cuckoo Sandbox 등)에 있는지 확인하고, 그럴 경우 동작을 멈추거나 정상 파일처럼 행동합니다.
- **탐지 기법**: 
    - MAC 주소 확인 (00:05:69... 등 특정 벤더 할당 대역).
    - 특정 파일/레지스트리 존재 확인 (vmmouse.sys, VBoxGuest.sys 등).
    - CPU 하드웨어 정보(IDT, GDT 값 등) 비교.

## 2. 안티 디버깅 (Anti-Debugging)
분석가가 디버거(x64dbg, IDA Pro 등)로 코드를 분석하는 것을 방해합니다.
- **기법**: 
    - `IsDebuggerPresent()` API 호출.
    - 예외 처리(Exception) 강제 발생 후 디버거의 반응 확인.
    - 시간 차 분석: 특정 구간 실행 시간이 너무 길면 디버깅 중으로 판단.

## 3. 폭탄 파일 (Zip Bomb / Decompression Bomb)
포렌식 도구가 파일을 스캔하거나 압축을 풀 때, 시스템 자원(CPU, RAM, Disk)을 고갈시켜 마비시키는 파일입니다.
- **구조**: 반복되는 데이터로 채워진 수 GB의 데이터를 극도로 압축하여 수 KB 수준으로 만듦.
- **대응**: 분석 도구의 압축 해제 깊이 제한 설정, 샌드박스 환경 내 자원 할당량 제한.

## 4. 난독화 (Obfuscation) 및 패킹 (Packing)
코드의 가독성을 없애고 정적 분석을 무력화합니다.
- **기법**: 문자열 암호화, 가짜 코드(Junk Code) 삽입, 커스텀 패커 사용.
- **대응**: 언패킹(Unpacking)을 통한 원본 OEP(Original Entry Point) 탐색, 메모리 덤프 후 문자열(Strings) 추출.

## 5. 전문가의 전략
분석 환경을 최대한 실제 PC와 유사하게 꾸미는 'Stealth' 기술이 필요합니다.
- 하드웨어 식별자 변경.
- 분석 도구의 이름을 정상 프로세스명으로 변경.
- 동적 분석 전, 가상 머신 탐지 루틴을 먼저 찾아 무력화(Patching)함.
