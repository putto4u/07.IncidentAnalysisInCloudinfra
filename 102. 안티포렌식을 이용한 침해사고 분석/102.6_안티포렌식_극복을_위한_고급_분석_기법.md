# 102.6 안티포렌식 극복을 위한 고급 분석 기법

## 1. 파일 시스템 저수준 분석
OS나 분석 도구의 API에 의존하지 않고, 바이너리 수준에서 파일 시스템 구조를 직접 해석합니다.
- **MFT ($MFT) 정밀 분석**: 삭제된 항목의 `In-use` 플래그가 해제되었더라도 데이터가 덮어씌워지기 전까지는 메타데이터가 남아있음을 이용.
- **Journaling 파일 분석**: `$LogFile`과 `$UsnJrnl`은 파일의 생성, 삭제, 이름 변경 이력을 순차적으로 기록하므로 공격자의 행위를 시간순으로 복구 가능.

## 2. 레지스트리 하이브(Hive) 복구 및 분석
삭제된 레지스트리 키나 값도 하이브 파일 내부의 빈 공간(Unallocated cells)에 흔적이 남을 수 있습니다.
- **Transaction Logs**: `.log1`, `.log2` 파일 분석을 통해 최근에 변경되었으나 아직 하이브 본체에 반영되지 않은 레지스트리 데이터 확인.

## 3. 브라우저 및 앱 아티팩트 교차 검증
공격자가 방문 기록을 지웠더라도 다음 장소에서 흔적을 찾을 수 있습니다.
- **DNS Cache**: 방문했던 도메인 정보가 메모리에 일시 보관됨.
- **Flash / Silverlight 캐시**: 특정 웹 앱 사용 흔적.
- **WebCache.dat (IE/Edge)**: 삭제된 기록의 인덱스가 남아있는 경우가 많음.

## 4. 타임라인 재구성 (Super Timeline)
모든 아티팩트(파일 시스템, 이벤트 로그, 레지스트리, 웹 히스토리 등)에서 추출한 시간 정보를 하나의 거대한 연대기로 합칩니다.
- **효과**: 공격자가 특정 로그를 지웠더라도, 같은 시간대에 다른 곳(예: Prefetch 생성)에서 남은 흔적을 통해 비어있는 조각을 채울 수 있습니다.
- **도구**: `plaso` (log2timeline)

## 5. 카빙 (Carving) 기법 고도화
파일 시스템 정보가 완전히 파괴된 경우에도, 파일의 고유한 헤더(Header)와 푸터(Footer) 시그니처를 이용하여 데이터를 복구합니다.
- **조각난 파일(Fragmented files) 복구**: 데이터가 불연속적으로 저장된 경우 통계적 분석을 통해 연결 고리를 찾음.

## 6. 전문가의 조언
"안티포렌식은 분석가를 귀찮게 할 뿐, 영원히 숨길 수는 없다." 인내심을 가지고 증거 간의 모순(Contradiction)을 찾는 것이 고급 분석가의 핵심 역량입니다.
