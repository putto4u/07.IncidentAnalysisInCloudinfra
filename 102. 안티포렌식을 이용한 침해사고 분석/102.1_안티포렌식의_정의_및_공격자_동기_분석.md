# 102.1 안티포렌식의 정의 및 공격자 동기 분석

## 1. 안티포렌식(Anti-Forensics)의 정의
안티포렌식은 디지털 증거의 수집, 보존, 분석을 어렵게 하거나 불가능하게 함으로써 자신의 행위를 숨기고 추적을 피하려는 모든 기술적, 절차적 행위를 의미합니다. 단순히 데이터를 지우는 수준을 넘어, 분석가에게 잘못된 정보를 제공하여 수사 방향을 오도하는 기만(Deception) 행위까지 포함합니다.

## 2. 공격자의 주요 동기
- **행위 은닉**: 침입 사실 자체를 숨기거나 최초 침투 경로(Initial Access)를 은폐함.
- **분석 방해**: 포렌식 도구가 오류를 일으키게 하거나 분석 시간을 무한정 늘려 수사 의지를 꺾음.
- **증거 능력 무력화**: 수집된 데이터의 무결성을 의심하게 만들어 법정에서 증거로 채택되지 못하게 함.
- **지속성 유지**: 시스템에 심어둔 백도어나 악성코드가 탐지되지 않고 오래 작동하게 함.

## 3. 침해사고 현장에서의 주요 안티포렌식 징후
전문 분석가는 다음과 같은 현상을 발견했을 때 안티포렌식 시도를 직감해야 합니다.
- **로그의 공백**: 특정 시간대의 이벤트 로그가 통째로 사라졌거나 서비스가 중지된 흔적.
- **부자연스러운 타임스탬프**: 파일의 생성 시간보다 수정 시간이 빠르거나, 밀리초 단위가 '000'으로 끝나는 경우(Timestomping 의심).
- **알려지지 않은 파일 확장자**: 실행 파일(.exe)이 이미지(.jpg)나 텍스트(.txt)로 위장되어 있는 경우.
- **시스템 도구의 오작동**: Task Manager나 특정 분석 도구가 실행되지 않거나 강제 종료됨.

## 4. 안티포렌식의 분류
1. **데이터 은닉 (Data Hiding)**: 존재하지만 보이지 않게 함.
2. **데이터 파괴 (Data Destruction)**: 복구 불가능하게 삭제.
3. **데이터 변조 (Data Counterfeiting)**: 가짜 흔적을 남김.
4. **분석 도구 무력화 (Anti-Analysis)**: 분석 환경 탐지 및 우회.

## 5. 전문가의 관점
"안티포렌식 흔적 자체가 공격자의 존재를 증명하는 가장 강력한 증거가 될 수 있다." 
정상적인 사용자나 시스템은 흔적을 지우려고 과도한 노력을 하지 않습니다. 안티포렌식의 발견은 그 지점부터 정밀 분석을 시작해야 한다는 이정표와 같습니다.
