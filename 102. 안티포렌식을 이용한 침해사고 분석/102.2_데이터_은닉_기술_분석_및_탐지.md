# 102.2 데이터 은닉 기술 분석 및 탐지

## 1. 스테가노그래피 (Steganography)
이미지, 오디오, 비디오 파일의 미세한 픽셀 값이나 주파수를 변조하여 데이터를 숨기는 기법입니다.
- **탐지 방법**: 파일의 해시 값 비교(원본이 있는 경우), 시각적 왜곡 분석, 전용 스테가노그래피 분석 도구(StegExpose 등) 활용.

## 2. ADS (Alternate Data Streams)
NTFS 파일 시스템의 특성을 악용하여, 하나의 파일 이름 아래 보이지 않는 여러 개의 데이터 스트림을 숨기는 기법입니다.
- **공격 예시**: `notepad.exe:malware.exe` 와 같이 정상 파일 뒤에 악성코드를 숨겨 실행.
- **탐지 방법**: `dir /R` 명령어 사용, MFT 분석 도구를 통한 스트림 확인.

## 3. 암호화 볼륨 및 숨겨진 파티션
VeraCrypt와 같은 도구를 사용하여 디스크 내에 암호화된 가상 디스크를 만들거나, 파티션 테이블을 조작하여 일반 OS에서 보이지 않는 영역을 생성합니다.
- **탐지 방법**: 디스크 전체 용량과 파티션 합계의 불일치 확인, 고엔트로피(High Entropy) 영역 검색(암호화된 데이터는 무작위성이 높음).

## 4. 슬랙 공간 (Slack Space) 및 비할당 영역 활용
파일 시스템이 할당한 최소 단위(Cluster)와 실제 파일 크기 사이의 남는 공간에 데이터를 은닉합니다.
- **탐지 방법**: 저수준 파일 시스템 분석(Hex Editor 등)을 통해 클러스터 끝부분의 부자연스러운 데이터 확인.

## 5. 펌웨어 및 하드웨어 레벨 은닉
HDD/SSD의 HPA(Host Protected Area)나 DCO(Device Configuration Overlay) 영역, 혹은 BIOS/UEFI 펌웨어 내부에 코드를 숨깁니다.
- **탐지 방법**: 전용 하드웨어 진단 도구 및 펌웨어 무결성 검사 도구 활용.

## 6. 대응 전략
데이터 은닉 기술을 이기기 위해서는 **'엔트로피 분석'**이 중요합니다. 압축되거나 암호화된 데이터는 통계적으로 균등한 분포를 보이므로, 파일 내부의 엔트로피를 시각화하면 숨겨진 영역을 찾아낼 수 있습니다.
