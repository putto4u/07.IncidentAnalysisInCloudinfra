# 102.3 증거 파괴 및 변조 기법 대응

## 1. 데이터 와이핑 (Wiping) 분석
단순 삭제가 아닌 0이나 무작위 값으로 덮어쓰는 행위입니다.
- **식별**: Hex Editor로 파일을 열었을 때 `00` 또는 `FF`로 가득 차 있거나 의미 없는 데이터로 채워진 경우.
- **대응**: 파일 시스템 저널($LogFile, $UsnJrnl) 분석을 통해 삭제되기 전 파일의 존재 여부와 삭제 시점 특정. 볼륨 섀도 복사본(VSC)에 남아있는 이전 버전 확인.

## 2. 타임스탬프 조작 (Timestomping)
파일의 MAC(Modified, Accessed, Created) 시간을 변경하여 공격 타임라인에서 벗어나게 합니다.
- **탐지 기법**:
    - **$MFT Standard Information vs FileName 속성 비교**: 윈도우는 시간 정보를 두 곳에 저장하는데, 일반적인 도구는 Standard Information만 조작하므로 FileName 속성의 시간과 대조하여 모순을 찾아낼 수 있습니다.
    - **이벤트 로그 대조**: 파일 생성 시간보다 늦은 시간에 해당 파일과 관련된 이벤트(프로세스 생성 등)가 발생했는지 확인.

## 3. 로그 삭제 및 변조
`wevtutil cl system` 명령 등을 통해 이벤트 로그를 삭제하거나, 로그 파일 자체를 수정합니다.
- **탐지 기법**: 
    - **Event ID 1102 (Log cleared)**: 로그 삭제 행위 자체를 기록하는 이벤트 확인.
    - **로그 번호 비연속성**: 로그 일련번호가 중간에 끊기거나 건너뛰는 경우 탐색.
    - **VSS(Volume Shadow Copy)** 내의 과거 로그 파일 추출.

## 4. 파일 위조 및 기만 (Counterfeiting)
정상적인 시스템 파일과 동일한 이름, 크기, 아이콘을 가진 악성 파일을 생성합니다.
- **탐지 기법**:
    - **디지털 서명 확인**: 정상 시스템 파일은 Microsoft 등의 유효한 서명이 있음.
    - **경로 확인**: `svchost.exe`가 `C:\Windows\System32`가 아닌 다른 곳에 있다면 100% 악성.
    - **해시 비교**: 알려진 정상 파일의 해시 데이터베이스(NSRL 등)와 대조.

## 5. 전문가의 조언
"공격자가 완벽하게 모든 흔적을 지울 수는 없다." OS는 복잡한 유기체와 같아서, 한 곳의 흔적을 지워도 그 행위로 인해 발생하는 다른 2차, 3차 아티팩트(Registry, Prefetch, USN Journal 등)가 반드시 남게 마련입니다.
