# 1.3 정적 탐지의 한계와 동적 맥락(Context-aware) 분석으로의 전환

## 1. 개요
지난 수십 년간 보안의 표준이었던 '임계값(Threshold)' 기반의 정적 탐지는 2026년의 정교한 위협을 막기에 역부족입니다. 공격자들은 탐지 기준 바로 아래에서 활동하거나, 정상적인 관리자의 행위를 모방합니다. 이를 극복하기 위한 '맥락 중심 분석' 아키텍처로의 진화가 필요합니다.

## 2. 정적 탐지의 한계점

### 2.1 "모 아니면 도"식의 임계값
*   **예시:** "5분 내 로그인 실패 10회 이상 시 탐지"
*   **한계:** 공격자가 5분에 9회씩 시도하거나, 여러 IP로 분산 공격을 수행할 경우 탐지가 불가능합니다. 또한, 대규모 업데이트나 시스템 점검 시 정상적인 행위도 임계값을 넘어 오탐을 유발합니다.

### 2.2 비즈니스 맥락의 부재
*   테스트 서버에서 발생하는 포트 스캔과 고객 데이터가 담긴 DB 서버에서 발생하는 포트 스캔을 동일한 위험도로 처리합니다. 이는 자원의 효율적 배분을 방해합니다.

## 3. 동적 맥락(Context-aware) 분석의 핵심 요소

### 3.1 자산 정보(Asset Context) 연동
*   해당 IP가 어떤 역할을 수행하는 장비인지, 처리하는 데이터의 등급은 무엇인지 실시간으로 매핑합니다. (예: DMZ 웹서버 vs 내부 인사 시스템)

### 3.2 사용자 행위 프로파일링(UEBA)
*   사용자의 과거 행위 패턴을 학습하여 '평소와 다른 시간', '평소와 다른 위치', '평소와 다른 데이터 전송량'을 감지합니다. 절대적인 수치가 아닌 상대적인 편차(Anomaly)를 기준으로 판단합니다.

### 3.3 위협 단계(Kill-chain)의 연관성
*   단일 로그가 아닌, 일련의 행위를 연결합니다. (외부 유입 -> 권한 상승 -> 내부 전파 -> 데이터 유출) 개별적으로는 위험도가 낮아 보이는 이벤트들도 연결되었을 때 비로소 고위험 인시던트로 식별됩니다.

## 4. 아키텍처의 진화 방향

| 구분 | 정적 탐지 (Legacy) | 동적 맥락 분석 (Next-Gen) |
| :--- | :--- | :--- |
| **판단 기준** | 고정된 값 (If-Then) | 확률 및 통계 기반 스코어링 |
| **데이터 범위** | 단일 보안 솔루션 로그 | 자산, 인사, CTI, 전체 인프라 로그 |
| **대응 방식** | 단순 차단 | 위험 수준에 따른 단계별 대응 |

## 5. 결론
2026년의 보안 아키텍처는 "무엇이 일어났는가?"를 넘어 **"누가, 왜, 어떤 가치를 가진 자산에 접근했는가?"**라는 질문에 답할 수 있어야 합니다. 맥락을 이해하는 보안 체계만이 AI 시대의 보이지 않는 위협을 식별해낼 수 있습니다.
