# 3.1 에지(Edge) 기반 로그 전처리: Vector 및 eBPF를 활용한 노이즈 제거

## 1. 개요
클라우드 인프라가 거대해짐에 따라 발생하는 로그의 양은 SIEM(Security Information and Event Management)의 처리 용량과 비용 한계를 넘어섰습니다. 모든 데이터를 중앙으로 보내는 대신, 발생 지점(Edge)에서 불필요한 노이즈를 사전에 필터링하는 '데이터 다이어트'가 필수적입니다.

## 2. 에지 기반 전처리의 핵심 기술

### 2.1 eBPF (Extended Berkeley Packet Filter)
*   **역할:** 리눅스 커널 레벨에서 시스템 콜이나 네트워크 패킷을 감시합니다.
*   **장점:** 커널 내부에서 필터링이 수행되므로 오버헤드가 극히 적으며, 사용자 공간(User Space)으로 데이터를 올리기 전에 '의미 없는 대량의 로그'를 차단할 수 있습니다.

### 2.2 Vector (High-Performance Data Pipeline)
*   **역할:** Rust 언어로 작성된 경량 로그 수집기 및 변환기입니다.
*   **주요 기능:**
    *   **Deduplication:** 동일한 내용의 반복적인 로그 중복 제거.
    *   **Parsing & Normalization:** 비정형 로그를 수집 단계에서 구조화(JSON 등)하여 중앙 시스템의 부하 경감.
    *   **Sampling:** 공격 징후가 없는 정상 로그는 10%만 샘플링하여 저장 비용 절감.

## 3. 필터링 전략: 무엇을 버릴 것인가?

1.  **정상 서비스 헬스체크:** 로드밸런서의 주기적인 상태 확인 로그는 보안적으로 의미가 거의 없으므로 에지에서 제거합니다.
2.  **알려진 '노이즈' 공인 IP:** 전 세계적으로 스캐닝을 수행하는 Shodan, Censys 등의 IP에서 발생하는 단순 포트 스캔 로그는 필터링합니다.
3.  **중복된 인증 실패:** 짧은 시간 내에 발생하는 동일한 계정의 반복적 인증 실패는 '요약 정보' 하나만 남기고 개별 로그는 버립니다.

## 4. 아키텍처 구성

1.  **Source:** 서버, 컨테이너, 네트워크 장비.
2.  **Edge Agent (Vector/eBPF):** 실시간 필터링, 정규화, 로컬 집계(Aggregation).
3.  **Transit:** Kafka 또는 클라우드 메시지 큐.
4.  **Destination:** OpenSearch, SIEM 등 (가공된 고품질 데이터만 수신).

## 5. 기대 효과
*   **비용 절감:** SIEM 데이터 유입량 감소로 인한 라이선스 및 스토리지 비용 50% 이상 절감.
*   **탐지 속도 향상:** 노이즈가 제거된 상태이므로 검색 쿼리 속도와 탐지 엔진의 성능이 비약적으로 향상됩니다.

## 6. 요약
가장 똑똑한 보안 분석은 **'보지 않아도 될 것을 보지 않는 것'**에서 시작합니다. 에지 기반 전처리는 현대적 클라우드 보안 관제의 지속 가능성을 보장하는 핵심 기술입니다.
