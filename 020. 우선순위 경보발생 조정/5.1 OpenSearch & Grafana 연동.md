# 5.1 OpenSearch & Grafana 연동: 위협 우선순위의 가시화와 심층 분석

## 1. 개요
데이터가 아무리 많아도 한눈에 파악할 수 없다면 정보로서의 가치가 없습니다. OpenSearch의 강력한 검색/분석 엔진과 Grafana의 유연한 시각화 능력을 결합하여, 현재 조직의 위협 상태를 직관적으로 파악하고 즉각적인 드릴다운(Drill-down) 분석이 가능한 환경을 구축합니다.

## 2. 대시보드 설계 원칙 (Operational Dashboard)

### 2.1 위협 요약 (Executive View)
*   **실시간 위협 스코어 추이:** 전체 인프라의 평균 위험 점수 그래프.
*   **Top N 위험 자산:** 현재 가장 높은 점수를 기록 중인 서버 및 사용자 리스트.
*   **위협 단계별 분포:** MITRE ATT&CK 기반의 공격 단계별 인시던트 현황.

### 2.2 분석가 뷰 (Analyst View)
*   **우선순위 큐 (Queue):** 스코어링 모델에 의해 정렬된 '지금 바로 확인해야 할 경보 리스트'.
*   **연관성 지도 (Map):** 소스 IP와 목적지 자산 간의 연결 관계를 지도나 그래프로 시각화.

## 3. 핵심 기술 및 구현 기법

### 3.1 Grafana의 변수(Variables) 활용
*   특정 호스트 ID나 IP를 선택하면 대시보드 전체의 그래프와 로그가 해당 자산 중심으로 즉시 필터링되도록 설계하여 분석 속도를 높입니다.

### 3.2 드릴다운 분석 (Drill-down)
*   대시보드의 '위협 점수 급증' 지점을 클릭하면, 해당 시간대의 원본 로그(OpenSearch Discover)로 즉시 이동하여 세부 행위를 파악할 수 있는 링크를 구성합니다.

### 3.3 실시간 알림 (Alerting)
*   특정 중요 자산의 위협 점수가 임계값을 넘거나, 전례 없는 새로운 공격 기법이 감지될 경우 Grafana 알림 기능을 통해 Slack/PagerDuty로 즉시 전송합니다.

## 4. 가시화의 시사점
가시화는 단순히 '예쁜 차트'를 만드는 것이 아닙니다.
*   **이상 징후 포착:** 수치상으로는 정상이지만 그래프의 패턴이 평소와 다를 때 분석가는 직관적으로 위협을 감지할 수 있습니다.
*   **의사결정 속도 향상:** 복잡한 보고서 대신 실시간 대시보드를 통해 상황을 공유함으로써 유관 부서와의 협업 속도를 높입니다.

## 5. 결론
OpenSearch와 Grafana는 보안 팀의 '눈'과 같습니다. 데이터를 직관적인 정보로 변환하는 시각화 체계는 위협 대응의 골든타임을 지키는 강력한 무기가 됩니다.
