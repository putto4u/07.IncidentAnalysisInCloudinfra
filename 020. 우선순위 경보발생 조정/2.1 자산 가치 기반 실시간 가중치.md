# 2.1 자산 가치 기반 실시간 가중치: 위협의 개인화

## 1. 개요
모든 서버와 데이터가 동일한 가치를 가지지 않음에도 불구하고, 많은 보안 시스템은 모든 자산에 동일한 경보 가중치를 부여합니다. 효과적인 위협 관리를 위해서는 CMDB(Configuration Management Database)와 연동하여 자산의 중요도에 따라 실시간으로 위협 점수를 동기화하는 체계가 필수적입니다.

## 2. 자산 중요도(Asset Criticality)의 구성 요소

### 2.1 비즈니스 임팩트 (Business Impact)
*   **핵심 서비스:** 결제 시스템, 대고객 서비스 웹 서버, 생산 라인 제어 장비 (High)
*   **지원 시스템:** 내부 게시판, 개발 테스트 서버, 단순 파일 공유 서버 (Low)

### 2.2 데이터 민감도 (Data Sensitivity)
*   개인정보(PII), 기밀 정보(Confidential), 소스 코드 등을 포함하고 있는지 여부에 따라 가중치를 부여합니다.

### 2.3 네트워크 가시성 (Network Exposure)
*   외부 인터넷에 노출된 DMZ 구간의 자산은 내부 폐쇄망 자산보다 공격에 노출될 확률이 높으므로 초기 스코어링 시 고려 대상입니다.

## 3. 실시간 연동 및 동기화 메커니즘

### 3.1 CMDB 및 클라우드 태깅 연동
*   Terraform, AWS Tagging 등을 통해 프로비저닝 시점에 자산의 성격을 자동으로 분류하고, 이를 보안 관제 시스템(SIEM)에 API로 연동합니다.

### 3.2 동적 가중치 조정
*   정기 점검 기간이나 특정 프로젝트 기간에는 해당 자산의 위험 점수 민감도를 일시적으로 조정하여 불필요한 경보를 방지하거나 감시를 강화합니다.

## 4. 우선순위 계산 공식 (Conceptual)
$$Risk Score = (Threat Severity \times Asset Weight) \times Confidence$$
*   **Threat Severity:** 공격의 기술적 위험도 (예: 원격 코드 실행)
*   **Asset Weight:** 자산의 중요도 (0.1 ~ 2.0)
*   **Confidence:** 탐지 기술의 신뢰도 (오탐 가능성 반영)

## 5. 기대 효과
1.  **SOC 자원 최적화:** 분석가는 중요도가 낮은 자산의 1,000개 경보보다 중요도가 높은 자산의 1개 경보에 집중하게 됩니다.
2.  **의사결정 지원:** 경영진에게 "보안 위협이 비즈니스 수익에 미치는 실질적인 영향"을 정량적으로 보고할 수 있습니다.

## 6. 요약
자산 중심의 보안 스코어링은 보안 조직이 비즈니스 언어로 소통하게 만드는 첫걸음입니다. 위협의 '심각성' 못지않게 자산의 '가치'를 이해하는 것이 2026년 보안의 핵심입니다.
