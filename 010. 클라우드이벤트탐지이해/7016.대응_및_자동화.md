# 6. 대응 및 자동화 (Incident Response)

탐지된 이벤트에 대해 빠르고 정확하게 대응하는 것이 중요합니다.

## 6.1 가시성 확보 및 알림 (Visibility & Alerting)
분석가가 상황을 즉시 파악할 수 있어야 합니다.
- **Dashboard**: 주요 보안 지표 실시간 시각화.
- **Notification**: 심각도(Severity)에 따른 알림 채널 분화.
    - Critical: 전화, SMS (PagerDuty 연동).
    - High/Medium: Slack, Teams 채널 메시지.

## 6.2 자동화된 조치 (Automated Remediation)
인간의 개입 없이 침해 사고의 확산을 막는 기술입니다.
- **워크플로우 예시**:
    1. **탐지**: AWS GuardDuty에서 'CryptoMining' 탐지.
    2. **이벤트 전달**: EventBridge를 통해 Lambda 함수 호출.
    3. **조치**: 해당 인스턴스의 보안 그룹을 모든 통신 차단(Isolation)으로 변경하고, 격리용 IAM 프로파일 부여.
    4. **보고**: 대응 결과를 Slack으로 전송.

## 6.3 SOAR (Security Orchestration, Automation and Response)
보안 운영의 효율성을 극대화하기 위한 통합 플랫폼입니다.
- **Playbook**: 정형화된 사고 대응 절차를 코드로 관리.
- **이점**: 사고 대응 시간(MTTR) 단축, 반복적인 수동 업무 제거, 분석가 역량 상향 평준화.

## 6.4 포렌식 데이터 보존
자동 대응 시 원본 데이터가 사라지지 않도록 주의해야 합니다.
- **Snapshot 자동 생성**: 격리 전 볼륨 스냅샷 생성.
- **Memory Dump**: 휘발성 데이터 수집 (EBS Direct API 등을 활용).
