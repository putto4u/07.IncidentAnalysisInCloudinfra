# 4. 탐지 기술 및 분석 방법론

## 4.1 규칙 기반 탐지 (Rule-based Detection)
가장 전통적이고 명확한 탐지 방식입니다. '조건문(If-Then)' 형식을 취합니다.
- **특징**: 오탐(False Positive)이 적고 분석 결과가 명확함.
- **예시**: 
    - "동일한 IP에서 5분 이내에 10번 이상 로그인 실패 시 차단"
    - "MFA(다요소 인증) 없이 루트 계정 로그인 시 알림"
- **표준화 도구**: **Sigma** (로그 분석 엔진에 독립적인 범용 규칙 포맷).

## 4.2 이상 징후 탐지 (Anomaly Detection)
평상시의 행위 패턴(Baseline)을 학습한 후, 여기서 벗어나는 행위를 식별합니다.
- **머신러닝 활용**: 클라우드 사업자의 매니지드 서비스(예: AWS GuardDuty)가 내부적으로 사용.
- **탐지 대상**: 
    - 평소에 사용하지 않던 리전(Region)에서 인스턴스 생성.
    - 비정상적으로 높은 네트워크 아웃바운드 트래픽 발생.
    - 평소 접근하지 않던 민감한 데이터 영역에 대량 접근.

## 4.3 위협 인텔리전스 (Threat Intelligence) 활용
외부의 알려진 공격자 정보와 대조하는 방식입니다.
- **IP/도메인 매칭**: 알려진 악성 C&C 서버, 토어(Tor) 출구 노드, 비정상적인 프록시 IP로부터의 접근 탐지.
- **피드 연동**: CSP가 제공하는 위협 피드 또는 외부 상용 위협 인텔리전스 서비스 연동.

## 4.4 런타임 보안 (Runtime Security)
애플리케이션이 실행 중인 상태에서의 행위를 실시간으로 감시합니다.
- **컨테이너 보안**: 컨테이너 내부에서 예상치 못한 쉘(Shell) 실행, 민감 파일(`/etc/shadow` 등) 수정 탐지.
- **주요 도구**: **Falco** (오픈소스 런타임 보안 도구), 클라우드 워크로드 보호 플랫폼(CWPP).
