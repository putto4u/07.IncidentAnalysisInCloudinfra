# 5. 주요 위협 시나리오별 탐지 사례

클라우드 공격 단계(Kill Chain)를 바탕으로 한 주요 탐지 사례입니다.

## 5.1 자격 증명 탈취 및 비정상 로그인 (Initial Access)

공격자가 유출된 액세스 키나 계정 정보를 사용하는 경우입니다.

- **탐지 지표**:
  - **Impossible Travel**: 물리적으로 불가능한 시간 내에 서로 다른 지역에서 로그인.
  - **Console Login without MFA**: 보안 정책을 위반한 관리자 접속.
  - **Access Key Use from Unfamiliar IP**: 평소 사무실이나 개발 환경이 아닌 곳에서의 API 호출.

## 5.2 리소스 오남용 - 크립토재킹 (Impact)

가장 흔한 클라우드 침해 사고 중 하나로, 공격자가 타인의 계정에서 암호화폐 채굴기를 돌리는 행위입니다.

- **탐지 지표**:
  - **Spiking CPU Utilization**: 특정 인스턴스의 CPU 사용률이 지속적으로 100%에 도달.
  - **Unusual Instance Type**: 평소 사용하지 않는 고성능 GPU/CPU 인스턴스 대량 생성.
  - **Mining Pool Communication**: 네트워크 로그에서 알려진 채굴 풀(Pool) 주소로의 통신 포착.

## 5.3 데이터 유출 (Exfiltration)

기업의 기밀 데이터나 개인정보를 외부로 빼내는 행위입니다.

- **탐지 지표**:
  - **S3 Bucket Public Access**: 갑작스럽게 비공개 버킷이 'Public'으로 변경됨.
  - **Large Egress Traffic**: 대용량의 데이터가 외부 인터넷망으로 전송됨.
  - **KMS Key Decryption Spike**: 암호화된 데이터를 복호화하기 위한 API 호출이 급증함.

## 5.4 권한 상승 및 백도어 생성 (Persistence)

공격자가 침투 후 권한을 강화하거나 나중에 다시 들어오기 위한 통로를 만드는 행위입니다.

- **탐지 지표**:
  - **New IAM User/Role Creation**: 알 수 없는 사용자가 생성됨.
  - **Policy Modification**: 기존 유저에게 `AdministratorAccess` 권한을 부여함.
  - **Snapshot Sharing**: 데이터가 포함된 볼륨 스냅샷을 외부 계정과 공유함.
