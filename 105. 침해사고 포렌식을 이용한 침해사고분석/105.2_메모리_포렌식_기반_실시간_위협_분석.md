# 105.2 메모리 포렌식 기반 실시간 위협 분석

## 1. 실행 중인 악성 코드 식별
디스크에는 흔적을 남기지 않는 파일리스(Fileless) 공격 탐지에 필수적인 단계입니다.

### 1.1 비정상 프로세스 탐지 기법
- **Parent-Child 관계 분석**: `explorer.exe` 아래에 `cmd.exe`가 실행되거나, 웹 서버 프로세스 하위에 쉘이 실행되는 등의 이상 징후 포착.
- **프로세스 은닉 탐지**: `pslist`와 `psscan` 결과 비교를 통해 DKOM(Direct Kernel Object Manipulation) 기법으로 숨겨진 프로세스 식별.

### 1.2 메모리 덤프 추출 및 정밀 분석
- 특정 의심 프로세스의 메모리 영역만 덤프하여 내부 문자열, URL, API 호출 이력 분석.

## 2. 고도화된 위협 분석
- **코드 인젝션**: 다른 정상 프로세스(예: svchost.exe)의 메모리 공간에 악성 코드를 삽입하는 기법 분석.
- **루트킷 및 드라이버**: 커널 영역에 로드되어 시스템을 장악하는 악성 드라이버 식별.
- **IAT/EAT 후킹**: 윈도우 API 호출 흐름을 가로채는 후킹 흔적 탐지.
