# 105.3 타임라인 분석(Timeline Analysis) 고도화

## 1. 시간축 통합 분석 (Data Fusion)
단일 로그 분석의 한계를 극복하기 위해 모든 가용 아티팩트를 하나의 타임라인으로 결합합니다.

### 1.1 통합 대상 데이터 소스
- 파일 시스템 메타데이터 (MFT, Inode)
- 윈도우 이벤트 로그 및 리눅스 시스템 로그
- 웹 서버 로그 (IIS, Apache, Nginx)
- 레지스트리 수정 시간
- 메모리 내 프로세스 시작 시간 및 네트워크 연결 시간

## 2. Super-Timeline 생성 및 활용
- **Plaso (log2timeline)**: 수백 개의 파서를 통해 다양한 형식의 데이터를 통합된 CSV/L2T 형식으로 변환.
- **맥락 파악 (Contextualization)**: 특정 시간대에 발생한 모든 행위를 종합하여 "공격자가 파일을 생성한 후 어떤 레지스트리를 수정하고 어떤 네트워크 통신을 했는지"를 논리적으로 재구성.

### 2.1 타임라인 필터링 기법
- 특정 키워드(공격 도구명, 특정 IP) 또는 특정 시간 범위를 기준으로 노이즈를 제거하고 핵심 사건에 집중하는 전략.
