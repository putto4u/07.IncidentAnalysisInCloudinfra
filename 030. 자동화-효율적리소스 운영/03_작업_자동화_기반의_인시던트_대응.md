# Part 3. 작업 자동화 기반의 인시던트 대응 (SOAR)

## 1. 개요
보안 사고 발생 시 사람의 개입 없이 자동으로 대응(Remediation)하여 피해를 최소화하는 자동화 아키텍처를 학습합니다.

## 2. 주요 학습 내용
### 3.1 Event-Driven Remediation
- **AWS Lambda & EventBridge:** 보안 경보(Event) 발생 시 즉각적으로 코드를 실행하여 공격 대상 리소스를 격리.
- **실시간 차단:** 비정상적인 IAM 권한 사용이나 이상 IP의 접근을 즉시 무효화.

### 3.2 자동화된 워크플로우 설계
- **n8n / Tines 활용:** 다양한 보안 장비와 API로 연동하여 복잡한 대응 프로세스를 시각적으로 자동화.
- **Playbook:** 사고 유형별 대응 시나리오를 자동화된 워크플로우로 사전에 정의.

### 3.3 Self-Healing 인프라
- **OpenTofu (Terraform):** 인프라 설정 오류(Drift)를 탐지하고 원래 정의된 보안 상태로 자동 복구.

## 3. 핵심 요약
- 탐지부터 차단까지의 시간을 초 단위로 단축하는 것이 목표입니다.
- 자동 대응은 오탐 시의 리스크를 고려하여 설계해야 합니다.
