# 3.1 Event-Driven Remediation: 자동화된 인시던트 대응 (SOAR)

## 상세 설명
보안 사고 발생 시 수동 대응의 한계를 극복하기 위해 이벤트 기반의 자동 대응(Event-Driven Remediation) 아키텍처가 도입되고 있습니다. SOAR(Security Orchestration, Automation and Response) 개념을 클라우드 네이티브하게 구현합니다.
- **Remediation 워크플로우:** 탐지 이벤트 발생 -> EventBridge/PubSub 전송 -> Lambda/Cloud Functions 실행 -> 리소스 격리 또는 권한 회수.
- **SOAR 도구:** Tines, n8n 등 로우코드(Low-code) 도구를 활용하여 복잡한 보안 운영 프로세스를 자동화합니다.
- **Self-Healing:** IaC(Terraform, OpenTofu)를 통해 인프라의 상태를 항상 최적의 보안 설정으로 자동 복구합니다.

## 핵심 요약
- 위협 탐지 즉시 자동 대응이 이루어지는 파이프라인 구축이 핵심입니다.
- API 기반의 연동을 통해 다양한 보안 장비와 클라우드 서비스를 유기적으로 연결합니다.
- 인적 오류를 줄이고 대응 시간(MTTR)을 획기적으로 단축할 수 있습니다.
