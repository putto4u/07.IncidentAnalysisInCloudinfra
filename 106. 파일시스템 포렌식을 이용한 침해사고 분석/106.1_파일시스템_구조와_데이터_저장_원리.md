# 106.1 파일시스템 구조와 데이터 저장 원리

## 1. 저수준 데이터 저장 단위
디지털 증거가 물리적 매체에 어떻게 기록되는지 이해하는 것은 포렌식의 기초입니다.

### 1.1 섹터(Sector)와 클러스터(Cluster)
- **섹터**: 하드 디스크의 최소 물리적 저장 단위 (보통 512 바이트 또는 4KB).
- **클러스터**: 운영체제가 파일을 관리하는 논리적 단위 (여러 섹터의 묶음). 파일 크기가 클러스터 단위보다 작아도 최소 한 클러스터를 점유함.

### 1.2 슬랙 공간 (Slack Space)
- 파일의 실제 크기와 할당된 클러스터 크기 사이의 남는 공간.
- 이전 데이터의 파편이 남아있을 수 있어, 공격자가 숨긴 데이터를 찾거나 삭제된 파일의 흔적을 복구하는 데 사용됨.

## 2. 주요 파일시스템 구조 비교
- **NTFS (Windows)**: MFT를 통한 고도로 구조화된 관리. 저널링 지원.
- **FAT32**: 호환성이 좋으나 대용량 파일 제약 및 메타데이터 부족.
- **Ext4 (Linux)**: Inode와 Block Group 기반 관리. 효율적인 대용량 파일 처리.
- **APFS (Apple)**: Copy-on-Write 방식의 컨테이너 기반 구조.
