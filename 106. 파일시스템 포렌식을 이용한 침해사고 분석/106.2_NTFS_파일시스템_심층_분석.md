# 106.2 NTFS 파일시스템 심층 분석

## 1. MFT (Master File Table) 구조
NTFS에서 모든 파일과 디렉토리는 MFT 엔트리라는 레코드로 관리됩니다.

### 1.1 주요 속성(Attribute) 분석
- **$STANDARD_INFORMATION ($SI)**: 파일의 생성/수정/접근 시간(MACB) 정보. 일반적인 도구에서 보여주는 시간.
- **$FILE_NAME ($FN)**: 파일 이름, 부모 디렉토리 정보, $SI보다 조작하기 어려운 타임스탬프 포함.
- **$DATA**: 실제 파일 데이터가 저장된 위치 또는 데이터 자체(Resident Data).

### 1.2 상주(Resident) vs 비상주(Non-resident) 데이터
- 파일 크기가 매우 작으면(보통 700바이트 미만) MFT 엔트리 내부에 직접 데이터를 저장(Resident).
- 이 경우 파일이 삭제되어도 MFT 엔트리만 확보하면 즉시 복구 가능.

## 2. 시스템 파일 분석
- **$LogFile**: 파일 시스템의 메타데이터 변경 이력을 기록하는 저널 파일.
- **$UsnJrnl**: 파일의 생성, 삭제, 이름 변경 등의 이력을 기록. 사고 분석 시 특정 시간대 행위 증명에 결정적.
