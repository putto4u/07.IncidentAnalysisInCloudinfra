# 106.5 파티션 및 볼륨 분석

## 1. 디스크 구조 이해
물리적 디스크가 논리적 볼륨으로 나뉘는 과정을 분석합니다.

### 1.1 MBR (Master Boot Record) vs GPT (GUID Partition Table)
- **MBR**: 2TB 이하 제한, 4개의 주 파티션. 레거시 시스템 분석 시 필수.
- **GPT**: UEFI 기반 최신 시스템. 다수의 파티션 및 복구용 백업 테이블 존재.

### 1.2 숨겨진 영역 탐색
- **HPA (Host Protected Area)**: 운영체제에는 보이지 않도록 설정된 디스크 끝부분의 숨겨진 영역.
- **Unallocated Space**: 파티션이 할당되지 않은 공간에 숨겨진 볼륨이나 데이터 파편 탐색.

## 2. 복합 환경 분석
- **가상 디스크 (VHD, VMDK)**: 클라우드나 가상화 환경에서의 디스크 이미지 마운트 및 분석.
- **RAID 분석**: 여러 디스크에 분산 저장된 데이터를 소프트웨어적으로 재구성하여 하나의 논리 볼륨으로 복원하는 기법.
