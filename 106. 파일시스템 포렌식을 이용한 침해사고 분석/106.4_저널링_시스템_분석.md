# 106.4 저널링(Journaling) 시스템 분석

## 1. 저널링의 목적과 원리
데이터 쓰기 작업 중 갑작스러운 전원 차단 시 발생할 수 있는 파일시스템 오류를 방지하기 위한 기록입니다.

### 1.1 트랜잭션(Transaction) 로그
- 파일 생성, 삭제, 이동 등의 메타데이터 변경 작업을 실제로 수행하기 전 저널에 먼저 기록.
- 포렌식 관점에서는 '의도된 행위'와 '수행된 행위'의 타임라인을 제공함.

## 2. 공격 흔적 재구성
- 공격자가 흔적을 지우기 위해 파일을 삭제(Shift+Delete)해도, 저널링 로그에는 해당 파일의 이름과 삭제 시점이 기록되어 남아있는 경우가 많음.
- **NTFS $LogFile 분석**: 최근 수 분에서 수 시간 내에 발생한 파일 시스템 변경 이력을 초 단위로 복원.
- **Ext4 Journal 분석**: 삭제된 파일의 Inode 번호와 삭제 시간을 확인하여 복구 단서 확보.
