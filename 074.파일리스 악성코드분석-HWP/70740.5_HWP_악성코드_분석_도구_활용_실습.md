# 5. HWP 악성코드 분석 도구 활용 실습

## 5.1 정적 분석 도구

### 5.1.1 HWPScan2
국내에서 개발된 HWP 전용 분석 도구로, 문서의 구조와 각 스트림의 내용을 직관적으로 확인할 수 있습니다.
- **주요 기능**: 스트림 추출, zlib 압축 해제, 악성 의심 항목 하이라이트.
- **실습**: `BinData` 스트림을 선택하고 마우스 오른쪽 버튼을 클릭하여 `저장(Extract)`을 선택합니다.

### 5.1.2 SSView (Structured Storage Viewer)
CFBF 구조의 파일을 계층적으로 보여주는 도구입니다.
- **용도**: OLE 구조가 복잡하게 얽힌 파일 분석 시 유용합니다.

### 5.1.3 oledump.py
파이썬 기반의 범용 OLE 분석 스크립트입니다.
- **명령어 예시**:
  ```bash
  python oledump.py sample.hwp
  # 'E' 표시가 있는 스트림은 EPS, 'O'는 OLE 개체를 의미
  python oledump.py -s 5 -d sample.hwp > stream5.bin
  ```

## 5.2 동적 분석 도구

### 5.2.1 x64dbg / OllyDbg
HWP 실행 프로세스(`Hwp.exe`)에 디버거를 Attach하여 메모리 변화를 관찰합니다.
- **분석 지점**: `VirtualProtect`, `CreateProcessInternalW` 등에 브레이크포인트(BP) 설정.

### 5.2.2 Process Monitor (Procmon)
파일 시스템, 레지스트리, 네트워크 활동을 실시간으로 기록합니다.
- **필터 설정**: `Process Name is Hwp.exe`.

## 5.3 분석 시나리오 실습
1. `oledump`를 통해 악성 스트림 번호 식별.
2. 해당 스트림 추출 후 `CyberChef`를 활용하여 XOR 복호화 시도.
3. 복호화된 결과물에서 쉘코드 패턴(예: `0x55 0x8B 0xEC`) 확인.
4. 디버거를 통해 쉘코드가 최종적으로 실행하는 API(`URLDownloadToFile` 등)와 C2 주소 추출.
