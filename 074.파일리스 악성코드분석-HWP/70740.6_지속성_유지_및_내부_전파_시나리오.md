# 6. 지속성 유지 및 내부 전파 시나리오

## 6.1 지속성 유지 (Persistence) 기법
HWP 취약점을 통해 실행된 초기 페이로드는 시스템 재부팅 후에도 살아남기 위해 다음과 같은 기법을 사용합니다.

### 6.1.1 레지스트리 자동 실행 등록
- `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 경로에 악성 스크립트 실행 명령 등록.
- **파일리스 방식**: `regsvr32.exe`나 `mshta.exe`를 이용하여 외부 URL의 스크립트를 직접 실행하도록 설정.

### 6.1.2 예약 작업 (Scheduled Tasks)
`schtasks.exe`를 사용하여 주기적으로 악성 페이로드를 다운로드하거나 실행합니다.

### 6.1.3 WMI Event Subscription
WMI(Windows Management Instrumentation)를 이용하여 특정 조건(예: 시스템 시작 5분 후)이 만족되면 악성 코드가 실행되도록 영구 등록합니다.

## 6.2 내부망 전파 (Lateral Movement) 시나리오

### 6.2.1 신뢰 관계 이용
감염된 PC 내의 주소록이나 메일 기록을 분석하여, 지인이나 업무 담당자에게 악성 HWP를 재배포합니다.

### 6.2.2 공유 폴더 활용
내부망의 공유 폴더(SMB)에 존재하는 정상 HWP 파일들을 찾아 악성 코드를 삽입하거나, 악성 파일을 정상 파일로 위장하여 업로드합니다.

### 6.2.3 정보 탈취 도구 추가 배포
초기 침투 성공 후, `Mimikatz`와 같은 자격 증명 탈취 도구를 메모리 상에서 실행(`Reflective DLL Injection`)하여 관리자 권한을 획득합니다.

## 6.3 포렌식 흔적 분석
- **Prefetch**: `HWP.EXE` 및 관련 시스템 도구의 실행 기록.
- **UserAssist**: 사용자가 실행한 파일 및 실행 횟수 기록.
- **Shimcache/Amcache**: 악성 코드가 실행된 흔적 조사.
