# 7. HWP 보안 강화 및 대응 전략

## 7.1 예방 전략

### 7.1.1 EPS 차단 설정
최신 버전의 한글은 EPS 처리를 제한하고 있으나, 보안 강화를 위해 레지스트리 설정을 확인해야 합니다.
- **경로**: `HKEY_CURRENT_USER\Software\HNC\Hwp\10.0\HwpPage\Option` (버전에 따라 상이)
- **설정**: EPS 렌더링 관련 옵션을 비활성화(0)로 설정.

### 7.1.2 소프트웨어 업데이트 자동화
한글과컴퓨터에서 제공하는 보안 패치를 즉시 적용합니다. 특히 제로데이(0-Day) 취약점 대응을 위해 자동 업데이트 기능을 활성화합니다.

## 7.2 탐지 및 대응

### 7.2.1 YARA 룰 작성 예시
HWP 파일 내의 비정상적인 EPS나 OLE 구조를 탐지하기 위한 규칙입니다.
```yara
rule HWP_Suspicious_EPS {
    strings:
        $ps_header = "%!PS-Adobe"
        $xor_op = "xor"
        $hex_data = /[0-9a-fA-F]{100,}/
    condition:
        uint32(0) == 0xE011CFD0 and $ps_header and $xor_op and $hex_data
}
```

### 7.2.2 EDR (Endpoint Detection and Response) 활용
- **동작 기반 탐지**: `Hwp.exe`가 `powershell.exe`나 `cmd.exe`를 자식 프로세스로 생성하는 행위를 차단합니다.
- **메모리 스캔**: 주기적으로 메모리 내의 ROP 가젯이나 쉘코드 패턴을 스캔합니다.

## 7.3 사고 대응 (Incident Response) 절차
1.  **식별**: 비정상적인 네트워크 연결이나 프로세스 생성 식별.
2.  **격리**: 감염 의심 호스트의 네트워크 차단.
3.  **제거**: 악성 레지스트리 키, 예약 작업 및 생성된 임시 파일 삭제.
4.  **복구**: 정상적인 HWP 엔진 재설치 및 백업 데이터 복구.

## 7.4 결론
HWP 악성코드는 국내 환경에 특화되어 있으며 지속적으로 정교해지고 있습니다. 기술적 방어체계 구축과 더불어 출처가 불분명한 문서에 대한 사용자 교육이 병행되어야 합니다.
