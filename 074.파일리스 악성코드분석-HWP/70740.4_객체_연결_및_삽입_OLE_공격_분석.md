# 4. 객체 연결 및 삽입(OLE) 공격 분석

## 4.1 OLE (Object Linking and Embedding) 개요
OLE는 윈도우 응용 프로그램 간에 데이터를 공유하고 제어할 수 있는 기술입니다. HWP 문서 내에 MS Excel, Word 문서나 실행 파일을 개체 형태로 삽입할 수 있습니다.

## 4.2 OLE 개체 악용 시나리오

### 4.2.1 Packager 개체 삽입
공격자는 악성 스크립트(VBS, JS, PowerShell)나 실행 파일을 OLE 객체로 삽입한 후, 아이콘을 문서 파일이나 이미지처럼 위장합니다.
- **공격 방식**: "이 문서를 보려면 아이콘을 클릭하세요"와 같은 사회 공학적 기법을 사용.

### 4.2.2 자동 실행 (Auto-execution) 취약점
특정 OLE 핸들러의 취약점을 이용하여 사용자의 클릭 없이도 문서 로딩 시점에 개체가 실행되도록 유도합니다.

### 4.2.3 외부 페이로드 링크 (Linking)
객체를 문서에 직접 포함하지 않고, 외부 URL을 통해 악성 페이로드를 가져오도록 설정합니다. 이는 파일 크기를 줄이고 탐지를 회피하는 데 유리합니다.

## 4.3 분석 방법

1.  **OLE 구조 분석**: `oledump.py`를 사용하여 HWP 내부에 포함된 OLE 스트림을 식별합니다.
2.  **개체 추출**: `BinData` 스트림에서 OLE 바이너리를 추출한 후, 내부의 `Contents` 스트림을 분석합니다.
3.  **동적 분석**:
    - 한글 프로그램에서 개체를 클릭할 때 발생하는 프로세스 트리 모니터링 (`Procmon`).
    - `temp` 디렉토리에 생성되는 임시 파일 확인.
4.  **스크립트 분석**: 추출된 페이로드가 VBScript나 PowerShell인 경우, 난독화를 해제하여 최종 C2 주소를 확보합니다.

## 4.4 주요 특징
- **파일리스 연계**: OLE 개체가 실행되면서 PowerShell을 호출하여 메모리 상에서만 동작하는 페이로드를 다운로드하는 방식이 흔히 사용됩니다.
- **사회 공학적 설계**: 문서 내용과 연관된 파일명(예: `증빙서류.scr`)을 사용하여 사용자의 의심을 피합니다.

## 4.5 분석 도구 예시
- **oledump.py -s [ID] -d**: 특정 스트림의 데이터를 덤프합니다.
- **SSView**: OLE 구조를 트리 형태로 시각화하여 보여줍니다.
