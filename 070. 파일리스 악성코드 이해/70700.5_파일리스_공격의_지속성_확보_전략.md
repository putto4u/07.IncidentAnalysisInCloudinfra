# 파일리스 공격의 지속성 확보 전략
## 1. 개요
재부팅 시 메모리가 초기화되는 파일리스 공격의 단점을 보충하기 위해, 공격자는 시스템 설정에 악성 코드를 심어 지속성을 유지합니다.

## 2. 상세 분석 내용
### WMI 이벤트 구독(Event Subscription) 활용
- **이벤트 트리거:** 특정 시간, 시스템 시작, 특정 프로세스 실행 등의 이벤트 발생 시 악성 스크립트가 실행되도록 WMI 데이터베이스에 등록합니다.
- **파일 없는 자동 실행:** 디스크에 파일이 없어도 WMI 서비스에 의해 스크립트가 자동으로 메모리에 로드됩니다.
### 레지스트리 Run 키에 인코딩된 스크립트 삽입
- **실행 경로 위장:** `Run`, `RunOnce` 등 자동 실행 레지스트리 키에 파워쉘 한 줄 명령어를 삽입하고, 실제 코드는 Base64 등으로 인코딩하여 숨깁니다.
- **간접 실행:** 부팅 시 레지스트리 값을 읽어 정상 인터프리터가 악성 행위를 수행하게 합니다.

## 3. 대응 및 방어 전략
- **WMI 데이터베이스 감사:** 주기적으로 등록된 WMI 이벤트 필터와 소비자를 점검하여 승인되지 않은 항목을 제거합니다.
- **자동 실행 설정 모니터링:** 레지스트리의 자동 실행 영역에 대한 변경 사항을 실시간으로 추적합니다.

## 4. 요약 및 결론
파일리스 공격도 지속성을 위해 시스템 설정(WMI, 레지스트리)을 건드립니다. 이러한 설정 변경 흔적을 찾는 것이 포렌식의 핵심입니다.
