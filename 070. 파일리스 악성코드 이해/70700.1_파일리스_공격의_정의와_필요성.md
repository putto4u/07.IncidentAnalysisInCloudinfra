# 파일리스 공격의 정의와 필요성
## 1. 개요
파일리스(Fileless) 공격은 전통적인 방식과 달리 디스크에 실행 파일을 생성하지 않고, 시스템의 메모리나 정상 도구를 활용하여 악성 행위를 수행하는 기법입니다.

## 2. 상세 분석 내용
### 시그니처 기반 보안 솔루션의 한계와 공격자의 선택
- **탐지 회피:** 기존 백신(AV)은 파일의 해시값이나 시그니처를 기반으로 탐지하는데, 파일리스 공격은 검사할 '파일' 자체가 없으므로 이를 효과적으로 우회합니다.
- **공격 효율성:** 시스템에 흔적을 적게 남겨 사후 분석과 포렌식을 어렵게 만듭니다.
### 'Living off the Land(LotL)' 기법의 개념 이해
- **정상 도구 활용:** OS에 이미 설치된 PowerShell, WMI, Mshta 등 신뢰할 수 있는 도구를 악용하여 공격자의 의도를 실행합니다.
- **신뢰 기반 실행:** 시스템이 신뢰하는 프로세스 내부에서 동작하므로 보안 정책을 우회하기 용이합니다.

## 3. 대응 및 방어 전략
- **행위 기반 분석:** 파일 검사 위주에서 프로세스의 이상 행위(예: 비정상적인 자식 프로세스 생성)를 모니터링하는 방식으로 전환합니다.
- **정상 도구 권한 제한:** 일반 사용자의 PowerShell 실행 권한을 제한하거나 실행 정책을 강화합니다.

## 4. 요약 및 결론
파일리스 공격은 보안 솔루션을 우회하기 위해 '파일 없는 공격'을 지향하며, 시스템 도구를 역이용하는 LotL 기법이 핵심입니다.
