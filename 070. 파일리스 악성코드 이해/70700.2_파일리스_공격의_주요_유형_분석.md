# 파일리스 공격의 주요 유형 분석
## 1. 개요
파일리스 공격은 구현 방식에 따라 메모리 상주형, 스크립트 기반, 레지스트리 활용형 등 다양한 형태로 나타납니다. 각 유형의 특성을 이해하는 것이 탐지의 첫걸음입니다.

## 2. 상세 분석 내용
### 메모리 상주형(Memory-only) 악성코드
- **반사적 인젝션:** DLL 파일을 디스크에 쓰지 않고 메모리에 직접 로드하여 실행하는 Reflective DLL Injection 기법을 사용합니다.
- **프로세스 할로잉:** 정상 프로세스를 생성한 후, 그 내부를 악성 코드로 교체하여 실행하는 기법입니다.
### 스크립트 기반(PowerShell, WMI, VBS) 공격
- **파워쉘 악용:** 난독화된 스크립트를 메모리에서 바로 실행하여 외부 C2와 통신하거나 정보를 탈취합니다.
- **WMI 활용:** WMI 쿼리를 통해 시스템 정보를 수집하거나 공격 명령을 전달합니다.
### 레지스트리 상주형 공격 기법
- **데이터 은닉:** 악성 페이로드를 레지스트리 키 값에 인코딩하여 숨기고, 부팅 시 이를 읽어 실행하도록 설정합니다.

## 3. 대응 및 방어 전략
- **메모리 스캐닝:** 주기적으로 프로세스 메모리 영역을 검사하여 비정상적인 실행 코드나 주입된 흔적을 찾습니다.
- **스크립트 로깅:** PowerShell 스크립트 블록 로깅 등을 활성화하여 실행되는 스크립트의 실제 내용을 기록합니다.

## 4. 요약 및 결론
파일리스 공격은 메모리와 시스템 설정을 교묘히 활용합니다. 각 유형별로 발생하는 아티팩트(메모리 변조, 스크립트 로그 등)를 파악하는 것이 중요합니다.
