# 탐지 및 분석의 어려움과 대응 방안
## 1. 개요
파일리스 공격은 가시성이 낮고 증거가 휘발되기 쉽기 때문에 기존 보안 체계로는 탐지와 분석에 많은 어려움이 따릅니다.

## 2. 상세 분석 내용
### 파일 검사 중심 보안의 맹점 분석
- **스캔 대상 부재:** 실시간 감시기가 검사할 파일이 생성되지 않으므로, 게이트웨이나 엔드포인트의 파일 스캔 단계에서 탐지되지 않습니다.
- **정상 프로세스 오탐:** 정상적인 관리 도구의 동작과 악의적인 사용을 구별하기 어렵습니다.
### AMSI(Antimalware Scan Interface)의 역할과 한계
- **메모리 스캔 표준:** 스크립트가 실행되기 직전, 메모리에 올라온 평문 코드를 보안 솔루션에 전달하여 검사받게 하는 윈도우 표준 인터페이스입니다.
- **우회 기법:** 공격자는 AMSI 기능을 메모리에서 무력화(Patching)하거나, 난독화를 통해 탐지를 회피합니다.

## 3. 대응 및 방어 전략
- **프로세스 메모리 덤프:** 의심 징후 발생 시 해당 프로세스의 메모리를 덤프하여 정밀 분석을 수행합니다.
- **컨텍스트 분석:** 개별 명령어보다는 명령어의 선후 관계와 실행 목적을 파악하는 상관분석을 강화합니다.

## 4. 요약 및 결론
파일 중심의 방어는 파일리스 공격에 무력합니다. 메모리 기반의 검사와 스크립트 엔진 수준의 모니터링(AMSI)이 보완되어야 합니다.
