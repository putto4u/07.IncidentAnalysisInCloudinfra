# 공격 매커니즘: 초기 침투에서 실행까지
## 1. 개요
파일리스 공격이 어떻게 시작되고, 어떤 단계를 거쳐 시스템 내부에서 실행되는지 전체적인 라이프사이클을 분석합니다.

## 2. 상세 분석 내용
### 매크로 포함 문서 파일을 통한 초기 진입
- **사회 공학적 유도:** 피싱 메일에 첨부된 문서 파일 실행 시, 매크로(VBA)가 동작하여 파워쉘 등의 스크립트를 호출합니다.
- **드롭퍼 역할:** 문서 자체가 악성코드는 아니지만, 메모리에서 동작하는 실제 악성 코드를 내려받는 통로 역할을 합니다.
### 익스플로잇 킷을 이용한 브라우저 메모리 오염
- **웹 취약점 활용:** 취약한 브라우저나 플러그인을 통해 웹 서핑 중 자동으로 악성 스크립트가 메모리에 주입되도록 합니다.
- **Drive-by Download:** 사용자가 인지하지 못하는 사이에 메모리 내에서 쉘코드가 실행됩니다.

## 3. 대응 및 방어 전략
- **이메일 보안:** 첨부 파일 내 매크로 실행을 원천 차단하거나 신뢰된 문서만 허용합니다.
- **브라우저 패치:** 웹 브라우저와 관련 플러그인을 항상 최신 버전으로 유지하여 취약점 악용을 방지합니다.

## 4. 요약 및 결론
초기 침투 단계에서는 주로 사용자 실수를 유발하는 문서나 브라우저 취약점이 활용됩니다. 진입점에서의 차단이 가장 효과적인 방어책입니다.
