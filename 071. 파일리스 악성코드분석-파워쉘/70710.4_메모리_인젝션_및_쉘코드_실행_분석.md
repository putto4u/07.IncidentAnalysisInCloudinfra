# 메모리 인젝션 및 쉘코드 실행 분석
## 1. 개요
파워쉘을 통해 디스크에 파일 없이 메모리에 직접 악성 코드를 주입하고 실행하는 정교한 기법을 심층 분석합니다.

## 2. 상세 분석 내용
### `Add-Type` 및 `Reflective DLL Injection` 기법 분석
- **Add-Type 활용:** 파워쉘 내에서 C# 코드를 컴파일하여 Win32 API를 직접 호출하는 라이브러리를 생성합니다.
- **반사적 인젝션:** DLL 파일을 메모리에 직접 로드하고 실행 지점을 제어하여 보안 솔루션의 파일 검사를 우회합니다.
### 파워쉘을 이용한 API 호출(Win32 API) 프로세스 이해
- **메모리 관련 API:** `VirtualAlloc`, `WriteProcessMemory`, `CreateRemoteThread` 등을 호출하여 쉘코드를 메모리에 배치하고 실행합니다.
- **프로세스 할로잉:** 정상 프로세스를 생성한 후 메모리 공간을 악성 코드로 대체하는 과정을 파워쉘로 구현합니다.

## 3. 대응 및 방어 전략
- **메모리 모니터링:** 프로세스 간 메모리 쓰기 시도나 비정상적인 실행 권한이 부여된 메모리 페이지를 탐지합니다.
- **제한된 언어 모드:** 중요 시스템에서는 PowerShell을 `Constrained Language Mode`로 설정하여 Win32 API 호출을 차단합니다.

## 4. 요약 및 결론
메모리 인젝션은 파일리스 공격의 핵심 기술입니다. API 호출 수준에서의 가시성을 확보하는 것이 탐지의 관건입니다.
