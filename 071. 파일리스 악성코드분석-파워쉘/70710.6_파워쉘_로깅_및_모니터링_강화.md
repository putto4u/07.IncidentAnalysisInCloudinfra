# 파워쉘 로깅 및 모니터링 강화
## 1. 개요
파워쉘 기반 공격에 대한 실질적인 방어 체계를 구축하기 위해 윈도우의 로깅 기능을 최적화하고 활용하는 방법을 학습합니다.

## 2. 상세 분석 내용
### 스크립트 블록 로깅(Event ID 4104) 활성화 및 분석
- **로깅 원리:** 실행되는 스크립트의 평문 내용을 윈도우 이벤트 로그에 기록하여 사후 분석 및 실시간 탐지를 가능케 합니다.
- **모듈 로깅(Event ID 4103):** 파워쉘 모듈 내부에서 수행되는 명령과 매개변수 정보를 기록합니다.
### 실행 정책(Execution Policy) 우회 기법과 방어 전략
- **우회 사례 분석:** `-ExecutionPolicy Bypass`와 같이 실행 정책을 무력화하는 다양한 인자값 사용 사례를 파악합니다.
- **강력한 실행 정책:** 그룹 정책(GPO)을 통해 실행 정책을 강제하고 사용자의 변경을 제한합니다.

## 3. 대응 및 방어 전략
- **로그 통합 분석:** 엔드포인트의 파워쉘 로그를 SIEM으로 집중시켜 상관분석 및 위협 헌팅에 활용합니다.
- **비정상 행위 경보:** 비정상적으로 긴 명령어 실행이나 특정 위험 키워드 포함 시 보안팀에 즉시 알림을 생성합니다.

## 4. 요약 및 결론
로깅은 파워쉘 방어의 근간입니다. 특히 4104 이벤트 로그를 통한 가시성 확보가 파일리스 공격 대응의 핵심입니다.
