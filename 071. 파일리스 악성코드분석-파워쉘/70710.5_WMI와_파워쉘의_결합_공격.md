# WMI와 파워쉘의 결합 공격
## 1. 개요
윈도우 관리 인프라인 WMI와 파워쉘을 연계하여 공격을 자동화하고 시스템 내부에 은밀하게 잔존하는 기법을 분석합니다.

## 2. 상세 분석 내용
### WMI 클래스를 이용한 악성 페이로드 은닉 및 실행
- **데이터 은닉:** 악성 스크립트를 WMI 클래스의 속성 값으로 저장하여 레지스트리나 파일 스캔을 회피합니다.
- **이벤트 구독:** 특정 시스템 이벤트 발생 시 저장된 파워쉘 코드가 자동으로 실행되도록 설정하여 지속성을 확보합니다.
### 원격 관리 기능(WinRM)을 이용한 내부 확산 분석
- **원격 명령 실행:** 탈취한 계정 정보를 활용하여 인접 시스템에 WinRM을 통해 파워쉘 명령을 전달하고 감염을 확산시킵니다.
- **신뢰 기반 통신:** 정상적인 관리 포트와 프로토콜을 사용하므로 네트워크 보안 장비 탐지가 어렵습니다.

## 3. 대응 및 방어 전략
- **WMI 데이터베이스 점검:** 비정상적으로 등록된 WMI 네임스페이스나 이벤트 필터를 주기적으로 감사합니다.
- **WinRM 접근 제어:** 필요한 시스템에서만 WinRM을 활성화하고 화이트리스트 기반의 IP 접근 제어를 적용합니다.

## 4. 요약 및 결론
WMI와 파워쉘의 결합은 탐지가 매우 어려운 공격 방식입니다. 시스템 관리 기능의 오남용을 감시하는 체계가 필요합니다.
