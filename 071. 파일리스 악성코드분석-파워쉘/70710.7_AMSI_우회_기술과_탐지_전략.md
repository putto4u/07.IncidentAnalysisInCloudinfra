# AMSI 우회 기술과 탐지 전략
## 1. 개요
윈도우의 스크립트 보호 기능인 AMSI를 무력화하려는 공격자의 기술을 분석하고 이를 탐지하기 위한 전략을 수립합니다.

## 2. 상세 분석 내용
### 메모리 내 AMSI 패치 기법 분석
- **AMSI 동작 원리:** 스크립트 실행 전 메모리 상의 코드를 보안 솔루션에 전달하여 검사하는 과정을 이해합니다.
- **패치 기법:** `amsi.dll`의 특정 함수 결과를 항상 '성공'으로 조작하거나, 메모리 상의 함수 주소를 변조하여 검사 과정을 건너뛰는 기술을 분석합니다.
### 행위 기반 파워쉘 공격 탐지 룰 설계
- **탐지 룰 작성:** AMSI 우회 시 자주 사용되는 특이한 함수 호출 패턴이나 문자열 조각을 기반으로 탐지 룰을 설계합니다.
- **상관 분석:** AMSI 패치 시도 직후에 발생하는 비정상적인 네트워크 연결이나 파일 생성 행위를 연계하여 탐지합니다.

## 3. 대응 및 방어 전략
- **AMSI 무결성 검사:** 프로세스 메모리 내 `amsi.dll`의 코드 영역이 변조되었는지 실시간으로 검사합니다.
- **다층 보안:** AMSI 외에도 EDR의 실시간 행위 로그 분석을 통해 상호 보완적인 방어 체계를 구축합니다.

## 4. 요약 및 결론
AMSI는 강력하지만 우회 가능성이 항상 존재합니다. 우회 시도 자체를 위협 지표로 삼아 더욱 정교한 탐지 전략을 운영해야 합니다.
