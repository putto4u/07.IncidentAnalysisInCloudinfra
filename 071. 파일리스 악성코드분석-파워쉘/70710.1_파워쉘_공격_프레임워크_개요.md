# 파워쉘 공격 프레임워크 개요
## 1. 개요
파워쉘(PowerShell)은 강력한 윈도우 관리 도구인 동시에, 공격자들에게는 파일 없이 악성 행위를 수행할 수 있는 완벽한 프레임워크를 제공합니다.

## 2. 상세 분석 내용
### PowerShell Empire, PowerSploit 등 주요 공격 도구 분석
- **공격 프레임워크:** Empire는 포스트 익스플로잇 단계에서 C2 통신 및 권한 상승을 자동화하며, PowerSploit은 정찰 및 정보 탈취를 위한 스크립트 모음입니다.
- **모듈화된 공격:** 공격자는 이미 잘 짜여진 모듈을 호출하기만 하면 복잡한 공격을 손쉽게 수행할 수 있습니다.
### 파워쉘이 공격자에게 매력적인 이유와 권한 체계
- **기본 탑재 도구:** 모든 윈도우 OS에 기본 설치되어 있어 별도의 파일 업로드가 필요 없습니다.
- **강력한 API 접근:** .NET Framework와 Win32 API에 직접 접근할 수 있어 운영체제 깊숙한 곳까지 제어가 가능합니다.

## 3. 대응 및 방어 전략
- **버전 관리:** 구버전(PowerShell 2.0)을 비활성화하여 현대적인 보안 기능(로깅 등)을 우회하지 못하도록 합니다.
- **Just Enough Administration(JEA):** 사용자에게 필요한 최소한의 파워쉘 기능만 할당하는 권한 관리 정책을 도입합니다.

## 4. 요약 및 결론
파워쉘 공격 도구들은 고도화되어 있으며, 이를 방어하기 위해서는 시스템 기본 기능으로서의 파워쉘에 대한 세밀한 권한 통제가 필요합니다.
에게 강력한 무기가 될 수 있는 프레임워크를 제공합니다. 이를 이해하는 것은 파워쉘 기반 공격에 대응하기 위한 필수 단계입니다.
