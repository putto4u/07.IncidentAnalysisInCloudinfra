# 101.2 운영체제별 주요 아티팩트 생성 원리

## 1. 윈도우(Windows) 주요 아티팩트
윈도우는 사용자 편의를 위해 다양한 캐시와 기록 파일들을 생성하며, 이는 포렌식의 보물창고가 됩니다.

- **프리페치(Prefetch)**: 응용 프로그램의 실행 속도를 높이기 위해 생성. 실행 파일명, 실행 횟수, 마지막 실행 시간, 참조된 라이브러리 목록을 담고 있어 '실행 흔적' 분석의 핵심입니다.
- **링크 파일(LNK)**: 최근 문서나 바탕화면 아이콘 등을 통해 생성. 원본 파일의 경로, 생성/수정/액세스 시간, 볼륨 일련번호 등을 포함하여 파일 열람 행위를 입증합니다.
- **점프 리스트(Jump Lists)**: 작업 표시줄 아이콘의 '최근 항목'. 사용자가 특정 앱에서 어떤 파일을 열었는지 상세히 기록합니다.
- **레지스트리(Registry)**: 시스템 설정 및 사용자 활동의 거대 데이터베이스. USB 연결 이력(USBSTOR), 실행된 명령어(RunMRU), 자동 실행 항목(Run keys) 등을 분석합니다.
- **MFT (Master File Table)**: NTFS 파일 시스템의 심장. 파일의 이름, 크기, 시간 정보, 물리적 위치를 관리하며 삭제된 파일의 흔적을 찾는 데 사용됩니다.

## 2. 리눅스/유닉스(Linux/Unix) 주요 아티팩트
리눅스는 "모든 것은 파일이다"라는 철학 아래 텍스트 기반 로그가 발달해 있습니다.

- **Syslog (/var/log)**: 커널 및 애플리케이션의 동작 기록 (auth.log, kern.log, syslog 등).
- **Bash History (~/.bash_history)**: 사용자가 터미널에서 입력한 명령어 기록. (공격자가 삭제할 1순위 타겟)
- **Cron Jobs (/etc/cron*)**: 예약된 작업 목록. 악성코드가 지속성(Persistence)을 유지하기 위해 자주 활용합니다.
- **Wtmp / Utmp**: 로그인/로그아웃 이력 및 현재 접속자 정보.

## 3. 아티팩트 분석 시 유의사항: 시간 정보(Timestamp)
- **UTC vs Local Time**: 분석 도구가 보여주는 시간이 기준 시간(UTC)인지 로컬 시간인지 반드시 확인해야 합니다.
- **Time Skew**: 대상 시스템의 시계가 실제 시간과 일치하지 않을 수 있으므로, 분석 전 타임 오프셋을 계산해야 합니다.

## 4. 전문가의 통찰
단순히 아티팩트를 나열하는 것이 아니라, **"왜 이 시점에 이 아티팩트가 생성되었는가?"**를 OS 동작 원리에 기반하여 설명할 수 있어야 합니다. 예를 들어, USB를 꽂았을 때 레지스트리의 어느 키가 변하고 어떤 로그가 남는지 매커니즘을 이해하는 것이 중요합니다.
