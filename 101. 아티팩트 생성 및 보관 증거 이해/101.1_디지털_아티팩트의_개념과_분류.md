# 101.1 디지털 아티팩트(Artifact)의 개념과 분류

## 1. 디지털 아티팩트의 정의
디지털 아티팩트란 컴퓨터 시스템이나 디지털 기기를 사용하는 과정에서 운영체제(OS), 애플리케이션, 네트워크 통신 등에 의해 자동으로 생성되는 '디지털 흔적'을 말합니다. 사용자가 직접 생성한 파일(문서, 사진 등)뿐만 아니라 시스템이 관리 목적으로 남기는 보이지 않는 기록들이 모두 포함됩니다.

## 2. 아티팩트의 주요 분류
분석의 효율성을 위해 아티팩트는 크게 두 가지 관점에서 분류됩니다.

### A. 주체에 따른 분류
- **시스템 아티팩트**: OS 부팅, 드라이버 로드, 업데이트 등 시스템 운영 과정에서 생성 (예: Registry, Event Logs, File System Metadata)
- **사용자 행위 아티팩트**: 사용자의 프로그램 실행, 파일 열람, 웹 서핑, 외부 저장장치 연결 등에 의해 생성 (예: Prefetch, LNK, Browser History, Shellbags)

### B. 데이터 성격에 따른 분류
- **휘발성 증거 (Volatile Data)**: 전원이 차단되면 사라지는 데이터 (예: RAM, 네트워크 연결 정보, 프로세스 정보)
- **비휘발성 증거 (Non-Volatile Data)**: 전원이 차단되어도 저장 매체에 남아있는 데이터 (예: HDD/SSD 내 파일, 로그, 레지스트리)

## 3. 증거 수집의 우선순위 (Order of Volatility)
포렌식 분석가는 데이터가 사라지기 쉬운 순서대로 수집해야 합니다 (RFC 3227 기준).
1. 레지스터, 캐시 (Registers, Cache)
2. 라우팅 테이블, ARP 캐시, 프로세스 테이블, 커널 통계 (Memory)
3. 임시 파일 시스템 (Temporary File Systems)
4. 디스크 (Disk)
5. 원격 로깅 및 모니터링 데이터 (Remote Logging)
6. 물리적 구성, 네트워크 토폴로지
7. 보관된 문서, 백업 매체

## 4. 아티팩트 분석의 가치
아티팩트는 공격자가 "무엇을 했는지"를 넘어 "어떤 의도를 가졌는지"와 "실제 행위의 타임라인"을 구성하는 결정적 단서가 됩니다. 특히 안티 포렌식을 시도하더라도 여러 곳에 흩어져 있는 아티팩트들 사이의 정합성을 완전히 맞추기는 매우 어렵기 때문에, 교차 분석을 통해 진실을 밝힐 수 있습니다.
