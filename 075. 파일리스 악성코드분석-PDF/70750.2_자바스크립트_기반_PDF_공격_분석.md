# 2. 자바스크립트 기반 PDF 공격 분석

## 2.1 PDF 자바스크립트 개요
Adobe Acrobat 및 대부분의 PDF 뷰어는 동적인 문서 처리를 위해 자바스크립트(AcroJS) 엔진을 내장하고 있습니다. 이는 PDF 공격의 가장 흔한 수단 중 하나입니다.

## 2.2 악용 기법

### 2.2.1 자동 실행 (Auto-execution)
공격자는 `/OpenAction` 또는 `/AA` (Additional Actions) 사전을 사용하여 문서가 열리거나 특정 페이지로 이동할 때 자바스크립트가 즉시 실행되도록 설정합니다.
- **예시**: `<< /Type /Action /S /JavaScript /JS (악성 코드) >>`

### 2.2.2 힙 스프레이 (Heap Spraying)
자바스크립트를 사용하여 메모리에 대량의 쉘코드를 배치합니다. 이는 뷰어 프로그램의 메모리 손상 취약점(예: Use-After-Free)과 결합되어 ASLR을 우회하는 데 사용됩니다.

### 2.2.3 API 취약점 악용
PDF 전용 자바스크립트 API(`app.doc.saveAs`, `util.printf`, `collab.collectEmailInfo` 등)의 보안 허점을 이용하여 임의 파일을 생성하거나 정보를 유출합니다.

## 2.3 난독화 (Obfuscation) 기법
탐지 회피를 위해 자바스크립트 코드를 다양한 방식으로 숨깁니다.
- **String Manipulation**: `String.fromCharCode`, `unescape`, `eval` 등을 사용한 문자열 조합.
- **Hex/Octal Encoding**: 코드를 16진수나 8진수로 인코딩.
- **Logical Obfuscation**: 복잡한 조건문과 무의미한 루프 삽입.

## 2.4 분석 방법

1.  **자바스크립트 추출**: `pdf-parser.py` 또는 `peepdf`를 사용하여 `/JavaScript` 액션이 포함된 객체를 찾습니다.
2.  **수동 디코딩**: 추출된 스크립트에서 `eval`이나 `app.execMenuItem` 등을 찾아 실제 수행되는 기능을 분석합니다.
3.  **동적 분석**: 뷰어 설정에서 자바스크립트 디버깅 기능을 활성화하거나, `SpiderMonkey` 같은 독립적인 JS 엔진에서 스크립트를 실행해 봅니다.

## 2.5 기술적 예시 (Pseudocode)
```javascript
// 힙 스프레이 예시
var shellcode = unescape("%u4141%u4141..."); // NOP + Shellcode
var spray = unescape("%u0c0c%u0c0c");
while (spray.length < 0x100000) spray += spray;
var heap = new Array();
for (var i = 0; i < 200; i++) heap[i] = spray + shellcode;
```
위 코드는 메모리의 특정 위치(`0x0C0C0C0C`)에 쉘코드가 위치하도록 유도하는 전형적인 패턴입니다.
