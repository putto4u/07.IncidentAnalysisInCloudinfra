# 70750.6 PDF 분석 전용 도구 활용 실습

## 1. 개요
PDF는 구조적 복잡성과 다양한 인코딩 방식 때문에 수작업 분석이 어렵습니다. 본 장에서는 침해사고 분석 실무에서 필수적으로 사용되는 도구들의 활용법과 분석 워크플로우를 학습합니다.

## 2. 주요 분석 도구 소개
### 2.1 PDFID
- **용도**: PDF 파일의 위험 요소(JavaScript, Launch, OpenAction 등)를 빠르게 스캔하는 도구입니다.
- **주요 명령어**: `python pdfid.py [파일명]`
- **결과 해석**: `/JS`, `/JavaScript`, `/AA`, `/OpenAction` 항목의 카운트가 0보다 크다면 정밀 분석이 필요합니다.

### 2.2 pdf-parser
- **용도**: PDF 내부의 개별 객체와 스트림을 상세히 탐색하고 추출하는 도구입니다.
- **주요 명령어**:
  - 특정 객체 내용 확인: `pdf-parser.py -o [객체번호] [파일명]`
  - 필터링된 스트림 해제: `pdf-parser.py -o [객체번호] -f [파일명]`
  - 키워드 검색: `pdf-parser.py -s /JavaScript [파일명]`

### 2.3 peepdf
- **용도**: 대화형(Interactive) PDF 분석 프레임워크입니다.
- **특징**: 객체 간의 참조 관계를 시각적으로 파악하기 쉽고, 내장된 JS 분석 기능을 제공합니다.

## 3. 분석 실무 워크플로우
### 1단계: 정적 트리거 식별
- `pdfid`를 사용하여 문서가 열릴 때 자동으로 실행되는 액션이 있는지 확인합니다.
- 예: `/OpenAction`이 가리키는 객체 번호를 추적합니다.

### 2단계: 스트림 추출 및 디코딩
- `pdf-parser`를 사용하여 의심스러운 객체의 스트림을 확인합니다.
- `/Filter /FlateDecode`로 압축된 경우 `-f` 옵션으로 압축을 해제하여 원본 텍스트(주로 JS 코드)를 추출합니다.

### 3단계: JavaScript 분석 및 페이로드 복원
- 추출된 JS 코드가 난독화되어 있다면 `SpiderMonkey`나 브라우저 디버거를 사용하여 복호화합니다.
- 최종적으로 실행되는 쉘코드(Shellcode) 패턴(예: `%u9090%u9090`)을 찾아냅니다.

## 4. 실습 예제: 난독화된 JS 추출
```bash
# 1. 의심 객체 검색
pdf-parser.py -s /JS sample.pdf

# 2. 해당 객체(예: 5번)의 스트림 디코딩 및 파일 저장
pdf-parser.py -o 5 -f sample.pdf > extracted_js.txt

# 3. 추출된 JS에서 쉘코드 유무 확인
grep -i "unescape" extracted_js.txt
```

## 5. 주의 사항
- 분석 도구 자체의 취약점을 이용한 안티-분석 기법이 존재할 수 있으므로, 반드시 격리된 분석 VM 환경에서 수행하십시오.
