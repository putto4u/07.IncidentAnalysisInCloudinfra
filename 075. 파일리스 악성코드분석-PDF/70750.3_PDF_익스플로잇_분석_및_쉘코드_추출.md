# 3. PDF 익스플로잇 분석 및 쉘코드 추출

## 3.1 취약점 유형별 분석

### 3.1.1 스택 및 힙 오버플로우
특정 PDF 객체의 길이를 처리하는 과정에서 발생하는 취약점입니다. 예를 들어 `/Index` 배열의 경계를 넘어서는 접근이나, `/Size` 값의 조작을 통해 발생합니다.

### 3.1.2 Use-After-Free (UAF)
메모리에서 해제된 객체를 다시 참조할 때 발생하는 취약점입니다. 주로 폼 필드(AcroForm)나 주석(Annotation) 처리 루틴에서 발견됩니다.

### 3.1.3 Type Confusion
객체의 타입을 오인하여 잘못된 메모리 접근을 수행하는 취약점입니다.

## 3.2 쉘코드 추출 및 분석 절차

1.  **쉘코드 위치 식별**: 주로 자바스크립트 내 문자열 변수나, 별도의 `/Stream` 객체 내에 존재합니다.
2.  **데이터 추출**: `pdf-parser.py -o [ID] -f` 명령어를 사용하여 필터(예: `/FlateDecode`)가 해제된 원본 데이터를 추출합니다.
3.  **패턴 탐색**: 쉘코드의 전형적인 시작 패턴(`0x55 0x8B 0xEC`, `0xEB`, `0xE8` 등)이나 NOP Sled(`0x90`)를 검색합니다.
4.  **역공학(Reverse Engineering)**:
    - 추출된 바이너리를 `scdbg` 같은 쉘코드 에뮬레이터에 로드합니다.
    - `x64dbg` 등 디버거에서 쉘코드만 별도로 실행시켜 API 호출 시퀀스를 확인합니다.

## 3.3 쉘코드의 주요 행위
- **API Resolving**: `GetProcAddress`, `LoadLibrary` 등을 직접 구현하여 필요한 시스템 함수를 찾습니다.
- **Dropping/Downloading**: 추가 페이로드를 임시 폴더에 생성하고 실행합니다.
- **Process Injection**: `Explorer.exe`나 `Svchost.exe` 같은 정상 프로세스에 코드를 삽입합니다.

## 3.4 기술적 팁
PDF 내의 쉘코드는 종종 `/ASCII85Decode`나 `/ASCIIHexDecode`로 인코딩되어 있습니다. 분석 도구를 사용할 때 이러한 필터 체인을 정확히 해제하는 것이 중요합니다.
- **도구**: `CyberChef` (Filter: From Hex, From Base64, From ASCII85 등)
