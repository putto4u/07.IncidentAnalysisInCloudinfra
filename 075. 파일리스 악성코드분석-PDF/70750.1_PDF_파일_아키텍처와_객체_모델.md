# 1. PDF 파일 아키텍처와 객체 (Object) 모델

## 1.1 PDF 개요
PDF(Portable Document Format)는 장치 독립적인 문서 표현을 위해 설계된 복잡한 구조의 파일 형식입니다. 단순한 텍스트를 넘어 폰트, 이미지, 하이퍼링크, 자바스크립트, 3D 모델 및 멀티미디어 콘텐츠를 포함할 수 있어 공격자들에게 광범위한 공격 표면을 제공합니다.

## 1.2 PDF 물리적 구조
PDF 파일은 크게 네 부분으로 나뉩니다.
1.  **Header**: PDF 버전 정보 (예: `%PDF-1.7`).
2.  **Body**: 문서의 내용을 구성하는 다양한 객체(Object)들의 집합.
3.  **Cross-Reference Table (Xref)**: 각 객체의 위치(오프셋) 정보를 담고 있는 테이블. 빠른 임의 접근을 가능하게 함.
4.  **Trailer**: Xref 테이블의 위치, 루트 객체(Catalog)의 참조 번호 등을 담고 있는 파일의 끝 부분.

## 1.3 PDF 논리적 구조 (객체 모델)
PDF는 8가지 기본 유형의 객체를 사용합니다.
- **Boolean**: `true`, `false`
- **Number**: 정수 및 실수
- **String**: `(Text String)` 또는 `<Hexadecimal String>`
- **Name**: `/`로 시작하는 고유 식별자 (예: `/Root`, `/Type`)
- **Array**: `[obj1 obj2 ...]`
- **Dictionary**: `<< /Key1 /Value1 /Key2 /Value2 >>` (핵심 구조)
- **Stream**: 대량의 데이터(이미지, 코드 등)를 포함하는 바이너리 블록. `stream`과 `endstream` 사이 위치.
- **Null**: `null`

## 1.4 핵심 객체 및 공격 지점
- **Catalog (/Root)**: 문서 객체 계층 구조의 최상위 노드.
- **Pages**: 각 페이지 정보를 담고 있으며, 페이지 렌더링 시 취약점이 발생할 수 있습니다.
- **Action (/OpenAction, /AA)**: 문서가 열릴 때 특정 동작(예: 자바스크립트 실행)을 수행하게 함.
- **EmbeddedFiles**: 문서 내에 다른 파일을 포함시키며, 악성 실행 파일의 저장소로 악용됩니다.

## 1.5 요약
PDF 분석의 핵심은 **Xref**와 **Trailer**를 통해 **Catalog**를 찾고, 거기서 파생되는 **Action**과 **Stream** 객체들을 추적하는 것입니다.
