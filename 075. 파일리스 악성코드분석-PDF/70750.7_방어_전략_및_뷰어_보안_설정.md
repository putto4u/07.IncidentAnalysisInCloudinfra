# 70750.7 방어 전략 및 뷰어 보안 설정

## 1. 개요
PDF 기반 공격은 주로 클라이언트 측 소프트웨어(뷰어)의 취약점과 기능을 악용합니다. 따라서 조직 차원의 보안 정책 설정과 사용자 교육, 그리고 기술적 방어 체계 구축이 병행되어야 합니다.

## 2. 뷰어(Adobe Acrobat Reader) 보안 설정 최적화
가장 효과적인 방어 방법 중 하나는 뷰어 자체의 위험 기능을 제한하는 것입니다.

- **자바스크립트 비활성화**:
  - `편집 -> 기본 설정 -> JavaScript`에서 'Acrobat JavaScript 사용' 체크 해제.
  - 대다수의 PDF 악성코드가 JS를 기반으로 하므로 가장 강력한 방어책입니다.
- **보호 모드(Protected Mode) 및 샌드박싱**:
  - `보안(고급)` 탭에서 '시작할 때 보호 모드 활성화'와 '보호 뷰'를 설정합니다.
  - 이는 익스플로잇이 성공하더라도 시스템의 다른 영역으로 전이되는 것을 차단합니다.
- **외부 연결 및 첨부 파일 제한**:
  - '신뢰 관리자'에서 PDF 파일이 외부 웹사이트에 액세스하거나 다른 파일을 실행하는 것을 금지합니다.

## 3. 엔터프라이즈 관리 정책 (GPO/Registry)
관리자는 그룹 정책이나 레지스트리를 통해 조직 내 모든 단말의 PDF 설정을 강제할 수 있습니다.

- **Registry 경로**: `HKEY_CURRENT_USER\Software\Adobe\Acrobat Reader\[버전]\JSPermissions`
- **GPO 활용**: 중앙 집중식 정책 배포를 통해 보안 업데이트를 강제하고 위험한 기능을 일괄 비활성화합니다.

## 4. 네트워크 및 게이트웨이 보안
- **이메일 게이트웨이 스캔**: 첨부된 PDF 내의 `/JS`, `/Launch` 등 위험 키워드를 포함한 문서를 격리합니다.
- **콘텐츠 무해화(CDR, Content Disarm & Reconstruction)**:
  - PDF 내의 액티브 콘텐츠(JS, Macro, OLE 등)를 제거하고 안전한 텍스트/이미지 정보만 포함된 새로운 PDF로 재구성하여 전달하는 기술입니다.
  - 파일리스 공격 대응에 매우 효과적입니다.

## 5. 모니터링 및 탐지 가시성 확보 (EDR/XDR)
- **프로세스 모니터링**: PDF 뷰어(Acrobat.exe)가 하위 프로세스로 `cmd.exe`, `powershell.exe`, `wscript.exe` 등을 생성하는 행위를 탐지 룰로 설정합니다.
- **파일 생성 감시**: 뷰어가 임시 디렉토리에 실행 파일이나 스크립트 파일을 드롭하는 행위를 실시간 모니터링합니다.

## 6. 결론 및 베스트 프랙티스
- 가능한 경우 웹 브라우저 내장 PDF 뷰어(보통 샌드박싱이 더 강화됨)를 사용하십시오.
- 정기적인 소프트웨어 패치 관리를 통해 알려진 취약점(N-Day)을 제거하십시오.
- 출처가 불분명한 PDF 문서를 열기 전에는 반드시 온라인 멀티 스캔 서비스(예: VirusTotal)를 활용하십시오.
