# 4. 악성 요소 은닉 기법 분석

## 4.1 필터(Filter)를 이용한 난독화
PDF는 스트림 데이터를 압축하거나 인코딩하기 위해 다양한 필터를 지원합니다. 공격자는 이를 중첩하여 사용하여 탐지를 회피합니다.
- **주요 필터**:
  - `/FlateDecode`: zlib/deflate 압축.
  - `/ASCII85Decode`: 바이너리를 텍스트로 변환.
  - `/LZWDecode`: LZW 압축 알고리즘.
  - `/Crypt`: 암호화된 스트림.
- **공격 기법**: `/Filter [/ASCII85Decode /FlateDecode]`와 같이 다중 필터를 적용하여 정적 분석 도구를 무력화합니다.

## 4.2 증분 업데이트 (Incremental Updates) 악용
PDF는 파일 끝에 새로운 데이터를 추가하여 기존 내용을 수정할 수 있는 기능을 지원합니다.
- **공격 방식**: 파일 전반부는 정상적인 내용으로 구성하고, 파일 끝(Trailer 뒤)에 악성 객체와 새로운 Xref 테이블을 추가합니다. 일반적인 뷰어는 마지막에 추가된 악성 객체를 우선적으로 처리합니다.

## 4.3 객체 숨기기 (Object Hiding)
- **비정상적인 이름 사용**: `/JavaScript` 대신 `/J#61vaScript`와 같이 16진수 인코딩을 섞어 키워드 탐지를 우회합니다.
- **간접 참조 활용**: 악성 코드를 여러 개의 작은 객체로 분할하고, 실행 시점에만 이를 결합하도록 설계합니다.

## 4.4 폴리글랏(Polyglot) 파일
하나의 파일이 PDF이면서 동시에 다른 형식(예: ZIP, JAR, HTML)으로 인식되도록 만드는 기법입니다.
- **예시**: 파일 헤더는 PDF이지만, 중간에 ZIP 구조를 포함시켜 자바 아카이브(JAR)로 실행되게 함.

## 4.5 분석 대응 전략
- **전수 조사**: 특정 키워드에 의존하지 않고 모든 `/Stream`과 `/Action` 객체를 전수 조사해야 합니다.
- **구조 복원**: `peepdf` 등의 도구를 사용하여 증분 업데이트된 파일의 모든 버전(Revision)을 개별적으로 분석합니다.
- **메타데이터 분석**: 제작 도구, 작성 일시 등 메타데이터의 비정상적인 수정을 확인합니다.
