# 061. 악성코드분석-다운로더 - 03. 네트워크 통신 프로토콜 분석

## 개요
다운로더 악성코드는 대부분 외부 명령제어(C2) 서버와 통신하여 추가적인 악성 페이로드를 다운로드하거나 명령을 수신합니다. 따라서 네트워크 통신을 분석하는 것은 악성코드의 전체적인 작동 방식을 이해하고, C2 서버의 위치 및 다운로드되는 파일을 식별하는 데 매우 중요합니다. 이 챕터에서는 HTTP/HTTPS 기반 C2 서버 통신 분석 방법과 유입되는 바이너리 추출 및 데이터 패킷 암호화 해제 기술에 대해 학습합니다.

## 1. HTTP/HTTPS 기반 명령제어(C2) 서버 통신 분석
대부분의 다운로더는 탐지 회피를 위해 일반적인 웹 트래픽과 유사한 HTTP 또는 HTTPS 프로토콜을 사용하여 C2 서버와 통신합니다. 이러한 통신을 분석하기 위한 도구와 기법을 이해하는 것이 중요합니다.

### 1.1 네트워크 트래픽 캡처 및 분석 도구
-   **Wireshark:** 네트워크 인터페이스를 통해 지나가는 모든 패킷을 캡처하고 상세하게 분석할 수 있는 대표적인 네트워크 프로토콜 분석기입니다. 다운로더 분석 시 C2 서버 IP, 포트, 통신 내용 등을 파악하는 데 필수적입니다.
    -   **필터링:** `ip.addr == [C2_IP]` 또는 `http` 필터를 사용하여 관련 트래픽만 걸러냅니다.
    -   **팔로우 TCP/HTTP 스트림:** 특정 세션의 전체 통신 내용을 한눈에 볼 수 있어 요청/응답 패턴을 파악하기 용이합니다.
-   **Fiddler / Burp Suite:** HTTP/HTTPS 프록시 도구로, 웹 트래픽을 가로채고 수정하며, SSL/TLS 트래픽을 복호화하여 볼 수 있습니다. 다운로더가 웹 프로토콜을 사용할 경우 매우 유용합니다.
    -   **HTTPS 복호화:** Fiddler나 Burp Suite의 CA 인증서를 설치하면 HTTPS 통신도 복호화하여 평문으로 볼 수 있습니다. 이는 암호화된 C2 통신 분석에 필수적입니다.
-   **Procmon (Process Monitor):** 이전 챕터에서 학습한 Procmon도 네트워크 통신 이벤트를 보여주므로, 악성 프로세스가 어떤 원격 주소와 연결하는지 빠르게 확인할 수 있습니다.

### 1.2 C2 서버 통신 패턴 분석
-   **접속 IP 주소 및 도메인:** 악성코드가 접속하는 IP 주소나 도메인 이름을 식별합니다. 이는 위협 지표(IoC)로 활용될 수 있습니다.
-   **요청/응답 패턴:** C2 서버와의 통신이 어떤 방식으로 이루어지는지 분석합니다.
    -   **정기적인 접속 (Beaconing):** 일정 주기로 C2 서버에 접속하여 명령을 대기하는 패턴.
    -   **특정 데이터 전송:** 감염된 시스템의 정보(OS 버전, IP 주소, 사용자 이름 등)를 C2 서버로 전송.
    -   **명령 수신:** C2 서버로부터 추가 페이로드 다운로드, 특정 명령 실행 등의 지시 수신.
-   **User-Agent / HTTP 헤더:** 악성코드가 사용하는 User-Agent 문자열이나 커스텀 HTTP 헤더를 분석하여 악성코드의 종류를 유추하거나, 시그니처로 활용합니다.

## 2. 유입되는 바이너리 추출 및 데이터 패킷 암호화 해제
C2 서버로부터 다운로드되는 추가 페이로드(바이너리)를 추출하고, 만약 통신 내용이 암호화되어 있다면 이를 해제하여 실제 악성코드의 내용을 파악해야 합니다.

### 2.1 다운로드되는 바이너리 추출
-   **네트워크 포렌식:** Wireshark에서 TCP/HTTP 스트림을 팔로우하여 HTTP 응답의 `Body` 부분에서 다운로드되는 파일의 원본 데이터를 추출할 수 있습니다. `File -> Export Objects -> HTTP` 기능을 활용하면 HTTP 통신을 통해 다운로드된 파일들을 쉽게 추출할 수 있습니다.
-   **프록시 도구 활용:** Fiddler나 Burp Suite를 사용하여 다운로드되는 파일을 직접 저장하거나, `Response` 탭에서 원본 데이터를 복사하여 바이너리 파일로 저장할 수 있습니다.
-   **메모리 덤프 (Memory Dump):** 악성코드가 메모리에 로드된 후 추가 페이로드를 다운로드하고 실행하는 경우, 메모리 덤프를 통해 실행 중인 프로세스의 메모리 영역에서 다운로드된 바이너리를 추출할 수 있습니다. (예: Volatility Framework, Process Hacker)
-   **파일 시스템 모니터링:** Procmon을 통해 악성코드가 파일을 다운로드하여 저장하는 경로를 파악하고, 해당 파일이 생성되는 시점에 추출합니다.

### 2.2 데이터 패킷 암호화 해제
다운로더와 C2 서버 간의 통신이 암호화되어 있는 경우, 이를 해제하여 실제 전송되는 데이터를 분석해야 합니다.
-   **HTTPS (SSL/TLS) 트래픽 복호화:**
    -   **프록시 도구 (Fiddler/Burp Suite):** 악성코드가 프록시를 신뢰하도록 설정되어 있거나, 시스템의 프록시 설정을 따르는 경우, 프록시 도구의 CA 인증서를 설치하여 HTTPS 트래픽을 복호화할 수 있습니다.
    -   **SSL/TLS 키 추출:** 일부 악성코드 분석 환경에서는 웹 브라우저나 시스템의 SSL/TLS 세션 키를 추출하여 Wireshark에서 복호화하는 고급 기법을 사용하기도 합니다.
-   **커스텀 암호화/인코딩 해제:** 악성코드가 자체적인 암호화 알고리즘(XOR, Base64, RC4 등)이나 인코딩 방식을 사용하는 경우, 정적 분석(리버스 엔지니어링)을 통해 해당 알고리즘을 파악하고, 파이썬 스크립트 등으로 복호화 루틴을 작성하여 데이터를 해독합니다.
    -   **패턴 식별:** 암호화된 데이터의 특징적인 헤더나 푸터, 반복되는 패턴 등을 통해 암호화 방식을 유추할 수 있습니다.
    -   **디버거 활용:** 디버거를 사용하여 암호화/복호화 함수 호출 지점을 찾고, 인자 값 및 반환 값을 확인하여 암호화 로직을 이해합니다.

## 핵심 요약
다운로더 악성코드의 네트워크 통신 분석은 C2 서버의 위치, 통신 패턴, 그리고 다운로드되는 추가 페이로드의 정체를 파악하는 데 필수적입니다. Wireshark, Fiddler/Burp Suite와 같은 도구를 사용하여 HTTP/HTTPS 트래픽을 캡처 및 분석하고, 다운로드되는 바이너리를 추출하며, 암호화된 통신은 적절한 기법(프록시 복호화, 커스텀 알고리즘 해제)을 통해 해독해야 합니다.
