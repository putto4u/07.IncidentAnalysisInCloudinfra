# 061. 악성코드분석-다운로더 - 05. 우회 및 은닉 기법 분석

## 개요
다운로더 악성코드는 분석 환경을 탐지하고 우회하여 실제 악성 행위를 숨기려는 시도를 합니다. 이는 분석가에게 큰 도전 과제가 됩니다. 이 챕터에서는 다운로더가 가상 환경 및 분석 도구를 탐지하고 우회하는 기능, 그리고 사용자 행위 대기 및 시간 지연 기법을 식별하는 방법에 대해 학습합니다.

## 1. 가상환경 및 분석 도구 탐지 기능 우회 분석
악성코드는 자신이 샌드박스, 가상 머신, 디버거 등 분석 환경에서 실행되고 있음을 인지하면 악성 행위를 중단하거나 다른 정상적인 동작을 보여 분석가를 속이려 합니다. 이러한 탐지 및 우회 기법을 이해하고 무력화하는 것이 중요합니다.

### 1.1 가상 환경 탐지 (Virtual Machine Detection)
-   **레지스트리 키 확인:** 악성코드는 특정 가상화 소프트웨어(VMware, VirtualBox)가 설치될 때 생성되는 레지스트리 키(예: `HKLM\HARDWARE\DESCRIPTION\System\BIOS\Vendor`에 `VMware` 또는 `VirtualBox` 문자열)를 검색하여 가상 환경 여부를 판단합니다.
-   **프로세스 및 서비스 확인:** `vmtoolsd.exe`, `VBoxService.exe` 등 가상 머신 확장 도구와 관련된 프로세스나 서비스가 실행 중인지 확인합니다.
-   **파일 경로 확인:** 가상 머신 관련 파일 경로(예: `C:\Program Files\VMware`)가 존재하는지 확인합니다.
-   **하드웨어 정보 확인:** CPU ID (CPUID 명령어 결과), MAC 주소 접두사(예: `00:50:56` for VMware), BIOS 정보 등을 확인하여 가상 환경의 특징적인 값을 찾습니다.
-   **메모리 크기/디스크 크기 확인:** 가상 머신은 일반적으로 실제 시스템보다 작은 메모리나 디스크 공간을 할당하므로, 이 값을 확인하여 가상 환경 여부를 추정합니다.
-   **탐지 우회 방법:**
    -   레지스트리 키, 프로세스명, 파일 경로를 수동으로 변경하거나 삭제합니다.
    -   가상 머신 설정 파일(`.vmx` 등)을 편집하여 하드웨어 정보를 위조합니다.
    -   게스트 확장 도구를 설치하지 않습니다.

### 1.2 분석 도구 탐지 (Analysis Tool Detection)
-   **디버거 탐지:**
    -   `IsDebuggerPresent()` API 호출: 현재 프로세스가 디버거에 의해 디버깅되고 있는지 확인합니다.
    -   PEB(Process Environment Block)의 `BeingDebugged` 플래그 확인.
    -   `NtGlobalFlag` 확인: 디버거 실행 시 특정 플래그가 설정됩니다.
    -   타이밍 공격: 디버거에서 명령 실행 시 지연이 발생하는 것을 이용하여 탐지.
    -   **우회 방법:** 디버거에서 해당 플래그를 수동으로 변경하거나, API 후킹(Hooking)을 통해 `IsDebuggerPresent()` 호출 시 항상 `False`를 반환하도록 조작합니다.
-   **샌드박스 탐지:**
    -   `GetTickCount()` 또는 `QueryPerformanceCounter()`를 사용하여 실행 시간 측정: 샌드박스는 빠르게 분석하기 위해 코드를 빠르게 실행시키므로, 실행 시간이 비정상적으로 짧으면 샌드박스로 판단합니다.
    -   파일 시스템 접근 권한 확인: 샌드박스는 제한된 권한으로 실행될 수 있으므로, 특정 시스템 파일에 대한 쓰기/삭제 권한을 시도하여 실제 시스템 여부를 판단합니다.
    -   사용자 입력 확인: 샌드박스 환경에서는 일반적으로 마우스 움직임이나 키보드 입력이 없으므로, 이러한 사용자 활동이 없는 경우 악성 행위를 지연시키거나 중단합니다.
    -   **우회 방법:** 시간 지연 기법을 무력화하기 위해 디버거에서 타이머 관련 함수를 스킵하거나, 분석 스크립트에서 가짜 사용자 입력을 시뮬레이션합니다.

## 2. 사용자 행위 대기 및 시간 지연 기법 식별
악성코드는 분석가가 특정 행위를 수행하기를 기다리거나, 일정 시간 동안 실행을 지연시켜 동적 분석 환경을 회피하려 합니다. 이는 분석가가 악성코드를 실행하자마자 바로 악성 행위를 관찰하는 것을 어렵게 만듭니다.

### 2.1 사용자 행위 대기 (User Interaction Evasion)
-   **목적:** 분석가가 직접 마우스를 움직이거나, 특정 버튼을 클릭하거나, 파일을 열 때까지 악성 행위를 숨깁니다.
-   **탐지 방법:**
    -   동적 분석 환경에서 마우스 움직임, 키보드 입력, 프로세스 생성 등의 사용자 활동을 기록하고 분석합니다.
    -   Procmon에서 `WriteFile`이나 `RegSetValue`와 같은 이벤트가 사용자 활동 이후에만 발생하는지 확인합니다.
-   **우회 방법:** 자동화된 샌드박스 분석 시스템에서는 가짜 사용자 입력을 시뮬레이션하여 악성코드의 대기 조건을 충족시킵니다.

### 2.2 시간 지연 기법 (Time-Based Evasion)
-   **목적:** `Sleep()`, `GetTickCount()`, `QueryPerformanceCounter()` 등 시간 관련 API 함수를 사용하여 악성코드 실행을 일정 시간 동안 지연시킵니다. 분석가가 짧은 시간만 분석하고 종료할 경우 악성 행위를 놓치게 만듭니다.
-   **탐지 방법:**
    -   디버거를 사용하여 `Sleep()` 함수 호출 지점을 찾아 인자 값(지연 시간)을 확인합니다.
    -   Procmon에서 프로세스 시작부터 주요 악성 행위가 발생하기까지의 시간을 측정하여 비정상적인 지연이 있는지 확인합니다.
    -   API 호출 모니터링 도구(API Monitor)를 사용하여 시간 관련 API 호출을 추적합니다.
-   **우회 방법:**
    -   디버거에서 `Sleep()` 함수 호출 시점의 EIP(Instruction Pointer)를 변경하여 해당 함수를 건너뛰거나, `Sleep()` 함수의 인자 값을 `0`으로 수정하여 지연 시간을 없앱니다.
    -   시스템 시간을 빠르게 진행시키는 가상 머신 스크립트를 사용합니다.

### 2.3 네트워크 가용성 확인 (Network Connectivity Check)
-   **목적:** 악성코드는 C2 서버와의 통신이 가능한지 확인하기 위해 특정 도메인(예: google.com, microsoft.com)에 접속을 시도하여 네트워크 연결 상태를 확인합니다. 격리된 환경에서는 이러한 접속이 실패하므로 악성 행위를 중단할 수 있습니다.
-   **탐지 방법:** Wireshark에서 DNS 쿼리 또는 HTTP/HTTPS 접속 시도를 모니터링합니다.
-   **우회 방법:** 샌드박스 환경 내에 가짜 DNS 서버 또는 웹 서버를 구축하여 악성코드가 네트워크 연결에 성공했다고 믿게 만듭니다 (FakeNet, INetSim).

## 핵심 요약
다운로더 악성코드는 가상 환경, 분석 도구, 사용자 활동 및 시간 지연 등 다양한 기법을 통해 탐지를 우회하고 은닉하려 합니다. 분석가는 이러한 우회 및 은닉 기법을 이해하고, 레지스트리/파일 시스템 조작, API 후킹, 디버거 우회, 가짜 네트워크 서비스 제공 등의 방법을 통해 악성코드의 실제 행위를 드러내야 합니다.
