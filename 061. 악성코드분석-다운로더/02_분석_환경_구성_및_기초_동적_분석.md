# 061. 악성코드분석-다운로더 - 02. 분석 환경 구성 및 기초 동적 분석

## 개요
다운로더 악성코드를 안전하고 효과적으로 분석하기 위해서는 통제된 환경을 구축하고, 악성코드의 동작을 실시간으로 관찰하는 동적 분석(Dynamic Analysis) 기법을 숙지해야 합니다. 이 챕터에서는 악성코드 분석을 위한 안전한 샌드박스 환경 구성 방법과 프로세스, 파일 시스템, 레지스트리 변화를 모니터링하는 기초 동적 분석 방법에 대해 학습합니다.

## 1. 안전한 샌드박스 환경 구축 및 네트워크 설정
악성코드를 실제 시스템에서 실행하는 것은 매우 위험하므로, 반드시 외부와 격리된 안전한 샌드박스(Sandbox) 환경에서 분석해야 합니다. 가상 머신(Virtual Machine)을 활용하여 이러한 환경을 구축하는 것이 일반적입니다.

### 1.1 샌드박스 환경 구성
-   **가상화 소프트웨어:** VMware Workstation/Fusion, Oracle VirtualBox, Hyper-V 등.
-   **호스트 운영체제:** 안정적인 Windows 또는 Linux (가상 머신 관리용).
-   **게스트 운영체제 (분석 대상):** Windows 7, Windows 10, 또는 필요한 경우 Linux 등 악성코드가 실행될 환경.
-   **스냅샷 (Snapshot):** 분석 전 깨끗한 상태로 스냅샷을 생성하여, 악성코드가 시스템을 변경하더라도 쉽게 초기 상태로 복원할 수 있도록 합니다. 각 분석 전에 항상 스냅샷으로 되돌리는 습관이 중요합니다.
-   **게스트 확장 도구 설치 금지:** 가상 머신에 VMware Tools, VirtualBox Guest Additions와 같은 게스트 확장 도구를 설치하지 않습니다. 악성코드가 이러한 도구를 탐지하여 가상 환경임을 인지하고 악성 행위를 중단할 수 있기 때문입니다.
-   **리소스 할당:** 악성코드가 원활하게 실행될 수 있도록 충분한 CPU, 메모리, 디스크 공간을 할당합니다.

### 1.2 네트워크 설정
악성코드는 외부 C2(Command & Control) 서버와 통신하거나 추가 페이로드를 다운로드할 수 있으므로, 네트워크 설정을 신중하게 구성해야 합니다.
-   **NAT (Network Address Translation):** 가장 일반적으로 사용되는 설정으로, 가상 머신이 호스트의 IP를 통해 외부 네트워크에 접근할 수 있도록 합니다. 외부와의 통신 모니터링에 용이합니다.
-   **Host-Only:** 가상 머신과 호스트 간에만 통신이 가능하며, 외부 네트워크와는 완전히 단절됩니다. 인터넷 접근이 필요 없는 정적 분석이나 특정 격리 환경에 유용합니다.
-   **Internal Network:** 여러 가상 머신이 서로 통신할 수 있지만, 호스트나 외부 네트워크와는 단절됩니다. 멀티 스테이지 악성코드 분석 시 유용합니다.
-   **네트워크 모니터링 도구 설치:** Wireshark, Fiddler, Procmon 등 네트워크 트래픽을 캡처하고 분석할 수 있는 도구를 샌드박스 환경 내에 준비합니다.

## 2. 프로세스, 파일 시스템, 레지스트리 변화 모니터링
악성코드가 시스템에서 실행될 때 발생하는 다양한 변화를 모니터링하는 것은 동적 분석의 핵심입니다. 시스템 도구를 활용하여 악성코드의 행위를 파악할 수 있습니다.

### 2.1 프로세스 변화 모니터링
-   **Procmon (Process Monitor):** Sysinternals Suite의 강력한 도구로, 프로세스, 파일 시스템, 레지스트리, 네트워크 활동 등 Windows 시스템에서 발생하는 거의 모든 이벤트를 실시간으로 모니터링하고 기록합니다.
    -   **다운로더 분석 시 활용:** 악성코드가 어떤 프로세스를 생성하는지, 어떤 파일에 접근하는지, 어떤 레지스트리 키를 조작하는지, 어떤 네트워크 연결을 시도하는지 등을 파악할 수 있습니다.
    -   **필터링:** 수많은 이벤트 중에서 의미 있는 정보만을 추출하기 위해 적절한 필터를 적용하는 것이 중요합니다 (예: 악성코드 프로세스 이름, 특정 파일 경로).
-   **Process Explorer:** 현재 실행 중인 프로세스의 상세 정보를 보여주는 도구입니다. 프로세스 트리, DLL 로드 정보, 핸들 정보 등을 통해 악성 프로세스를 식별할 수 있습니다.

### 2.2 파일 시스템 변화 모니터링
-   **Procmon 활용:** Procmon을 통해 악성코드가 어떤 파일을 생성, 수정, 삭제하는지, 어떤 디렉토리에 접근하는지 등을 확인할 수 있습니다.
-   **ChangesView (NirSoft):** 시스템의 파일 및 레지스트리 변경 사항을 감지하고 목록으로 보여주는 도구입니다. 특정 시점의 스냅샷을 찍어 비교 분석하는 데 유용합니다.
-   **HxD (Hex Editor):** 악성코드가 생성하거나 수정하는 파일을 직접 열어 헥스 코드 수준에서 내용을 확인하고 분석합니다. 특히 다운로드된 추가 페이로드의 바이너리 구조를 파악하는 데 유용합니다.
-   **Directory Listing:** `dir /s` 명령어나 `tree` 명령어를 사용하여 파일 시스템의 전체 구조를 파악하고, 의심스러운 파일이나 디렉토리를 식별합니다.

### 2.3 레지스트리 변화 모니터링
-   **Procmon 활용:** 악성코드가 어떤 레지스트리 키를 생성, 수정, 삭제하는지, 특히 `Run` 키나 서비스 관련 키 등 지속성(Persistence)을 확보하려는 시도를 집중적으로 모니터링합니다.
-   **Regshot:** 시스템의 레지스트리 스냅샷을 찍어 비교 분석하는 도구입니다. 악성코드 실행 전후의 레지스트리 변화를 쉽게 파악할 수 있습니다.
-   **Autoruns (Sysinternals):** 시스템 시작 시 자동으로 실행되는 모든 프로그램, 서비스, 드라이버 등을 목록으로 보여줍니다. 악성코드가 지속성을 위해 등록한 항목을 식별하는 데 매우 효과적입니다.

## 핵심 요약
다운로더 악성코드 분석을 위한 기초 동적 분석은 안전한 샌드박스 환경 구축과 네트워크 설정을 기반으로 합니다. Procmon, Process Explorer, Regshot, Autoruns와 같은 시스템 도구를 활용하여 악성코드 실행 시 발생하는 프로세스 생성, 파일 시스템 변경, 레지스트리 조작 등 시스템의 변화를 실시간으로 모니터링하고 기록함으로써 악성 행위를 파악할 수 있습니다.
