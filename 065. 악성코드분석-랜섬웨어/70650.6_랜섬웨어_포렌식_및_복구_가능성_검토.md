# 랜섬웨어 포렌식 및 복구 가능성 검토
## 1. 개요
사고 발생 후 피해 최소화를 위해 암호화된 데이터의 복구 가능성을 타진하는 과정입니다. 암호화 알고리즘의 구현 취약점이나 메모리 내 잔존 정보를 탐색하는 것이 핵심입니다.

## 2. 상세 분석 내용
### 메모리 내 암호화 키 추출 가능성
- **메모리 덤프 분석:** 암호화가 진행 중이거나 완료된 직후, 프로세스 메모리 영역 내에 평문으로 남아 있는 대칭키 또는 개인키의 흔적을 검색합니다.
- **키 스케줄 분석:** 특정 알고리즘(AES 등)의 키 스케줄링 과정에서 생성되는 라운드 키를 역추적하여 마스터 키를 복원합니다.
### 암호화 구현 취약점 탐색 및 복구 도구
- **난수 생성기 결함:** 암호학적으로 안전하지 않은 의사 난수 생성기(PRNG)를 사용한 경우, 키 값을 예측하여 복구 도구를 제작할 수 있습니다.
- **논리적 오류:** 암호화 과정에서의 프로그래밍 실수(예: 초기화 벡터 재사용 등)를 찾아내어 복호화 로직을 개발합니다.

## 3. 대응 및 방어 전략
- **신속한 메모리 보존:** 감염 의심 시 즉시 시스템을 종료하지 않고 메모리 덤프를 수행하여 휘발성 증거를 확보합니다.
- **공식 복구 툴 확인:** 'No More Ransom' 프로젝트 등 신뢰할 수 있는 기관에서 제공하는 복구 도구 존재 여부를 먼저 확인합니다.

## 4. 요약 및 결론
랜섬웨어 복구는 암호학적 분석과 포렌식 기술의 결합입니다. 완벽한 복구는 어려울 수 있으나, 구현상의 허점을 찾는 노력이 실무적으로 매우 중요합니다.
