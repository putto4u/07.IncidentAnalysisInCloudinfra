# 키 관리 및 C2 서버 통신 방식
## 1. 개요
랜섬웨어의 핵심은 암호화된 데이터를 복구하기 위한 키 관리 체계에 있습니다. 공격자는 피해자가 스스로 데이터를 복구할 수 없도록 강력한 비대칭 암호화 알고리즘을 사용하며, 키를 안전하게 전달받기 위해 익명 네트워크를 활용합니다.

## 2. 상세 분석 내용
### 로컬 생성 키와 서버 수신 키의 처리 프로세스
- **로컬 키 생성:** 감염 초기 단계에서 피해자 PC에서 대칭키(AES 등)를 생성하여 파일을 암호화합니다.
- **키 보호:** 생성된 대칭키는 공격자의 공개키로 암호화되어 서버로 전송되거나 파일 끝에 첨부됩니다. 이를 통해 공격자의 개인키 없이는 대칭키를 복구할 수 없게 만듭니다.
### 오프라인 암호화 및 다크웹(Tor) 기반 통신 기술
- **오프라인 암호화:** C2 서버와 통신이 불가능한 환경에서도 암호화를 수행하기 위해 내장된 공개키를 사용합니다.
- **Tor 기반 통신:** 추적을 피하기 위해 .onion 도메인을 사용하는 다크웹(Tor) 네트워크를 통해 키를 교환하고 협상을 진행합니다.

## 3. 대응 및 방어 전략
- **네트워크 모니터링:** Tor 노드와의 통신이나 비정상적인 도메인으로의 대량 데이터 전송을 차단합니다.
- **키 유출 방지:** 암호화 프로세스 도중 메모리에 남아있을 수 있는 암호 키를 보호하기 위한 메모리 무결성 검사를 수행합니다.

## 4. 요약 및 결론
랜섬웨어의 키 관리는 비대칭 암호화를 기반으로 하며, Tor 네트워크를 통해 공격자의 익명성을 보장합니다. 통신 단계에서의 차단이 피해를 최소화하는 핵심입니다.
