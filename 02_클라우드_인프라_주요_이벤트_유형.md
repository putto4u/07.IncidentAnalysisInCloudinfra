# 2. 클라우드 인프라 내 주요 이벤트 유형

클라우드 환경의 이벤트는 크게 제어 평면, 데이터 평면, 네트워크, 워크로드 네 가지 계층으로 나뉩니다.

## 2.1 제어 평면 (Control Plane) 이벤트
클라우드 리소스를 관리하는 API 호출 기록입니다. 가장 중요한 보안 감시 대상입니다.
- **주요 내용**: 사용자의 로그인(Console/CLI), 가상 머신 생성/삭제, 보안 그룹(Firewall) 규칙 변경, IAM 권한 수정 등.
- **주요 서비스**: AWS CloudTrail, Azure Activity Log, GCP Cloud Audit Logs.
- **보안 중요도**: 공격자가 관리자 권한을 탈취했을 때 발생하는 모든 행위가 여기에 기록됩니다.

## 2.2 데이터 평면 (Data Plane) 이벤트
리소스 내부의 데이터에 접근하거나 조작하는 행위입니다.
- **주요 내용**: S3 버킷 내 객체 읽기/쓰기, 데이터베이스 쿼리 로그, Key Management Service(KMS) 사용 기록.
- **특징**: 이벤트 발생량이 매우 방대하므로 비용 최적화를 위해 민감한 데이터 영역에 대해서만 활성화하는 것이 일반적입니다.

## 2.3 네트워크 이벤트
가상 네트워크 내에서 발생하는 트래픽 흐름입니다.
- **VPC Flow Logs**: IP 트래픽의 시작/종료, 프로토콜, 포트, 허용/차단 여부 기록.
- **DNS 로그**: 리소스가 외부 도메인에 연결을 시도하는 기록 (C&C 서버 통신 탐지에 유용).
- **로드 밸런서 로그**: 애플리케이션 계층(L7)의 HTTP 요청 및 응답 코드 정보.

## 2.4 워크로드 및 애플리케이션 이벤트
실제 실행 중인 인스턴스나 컨테이너 내부의 기록입니다.
- **OS 로그**: SSH 접속 시도, sudo 권한 사용, 시스템 에러 로그.
- **컨테이너 이벤트**: Docker/Kubernetes API 호출, Pod 생성 및 삭제, 런타임 보안 이벤트.
- **애플리케이션 로그**: 비즈니스 로직 오류, 사용자 인증 실패 기록 등.
