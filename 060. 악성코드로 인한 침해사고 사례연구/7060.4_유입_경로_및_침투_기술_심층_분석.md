# 050. 악성코드로 인한 침해사고 사례연구 - 04. 유입 경로 및 침투 기술 심층 분석

## 개요
악성코드 침해사고를 효과적으로 방어하기 위해서는 공격자가 시스템에 처음 침투하는 경로와 초기 침투 이후 내부 네트워크에서 어떻게 이동하는지(Lateral Movement)에 대한 깊은 이해가 필수적입니다. 이 챕터에서는 스피어 피싱, 워터링 홀, 취약점(RCE) 활용 기법을 비교하고, 초기 침투 이후의 내부 이동 기술에 대해 심층 분석합니다.

## 1. 스피어 피싱, 워터링 홀, 취약점(RCE) 활용 기법 비교
악성코드가 시스템에 유입되는 초기 침투(Initial Compromise) 방법은 다양하며, 공격자는 대상의 특성에 따라 가장 효과적인 방법을 선택합니다.

### 1.1 스피어 피싱 (Spear Phishing)
-   **정의:** 특정 개인이나 조직을 대상으로 맞춤형 정보를 활용하여 신뢰를 얻은 후 악성코드 유포 또는 자격 증명 탈취를 시도하는 피싱 공격의 한 형태입니다. 일반 피싱보다 훨씬 정교하고 성공률이 높습니다.
-   **공격 방법:**
    -   대상자의 이름, 직책, 소속, 업무 내용 등 개인화된 정보를 이메일 내용에 포함합니다.
    -   수신자가 관심을 가질 만한 주제(청구서, 프로젝트 관련 문서, 인사 알림 등)로 위장합니다.
    -   악성 첨부 파일(매크로 포함 문서, 실행 파일) 또는 악성 웹사이트 링크를 포함합니다.
-   **침투 메커니즘:** 첨부 파일 실행 또는 링크 클릭 시 악성코드 다운로드 및 실행.
-   **방어 전략:** 이메일 보안 솔루션(ATP), 사용자 보안 교육, 첨부 파일 검역, 매크로 비활성화 정책.

### 1.2 워터링 홀 (Watering Hole)
-   **정의:** 공격 대상이 자주 방문하는 합법적인 웹사이트를 미리 감염시켜, 해당 사이트 방문 시 자동으로 악성코드를 다운로드하거나 취약점을 악용하여 시스템에 침투하는 공격 기법입니다.
-   **공격 방법:**
    -   공격 대상의 업무 특성을 파악하여 자주 방문하는 웹사이트를 식별합니다 (예: 산업 관련 뉴스 사이트, 파트너사 웹사이트).
    -   식별된 웹사이트의 취약점(예: 웹 서버 취약점, CMS 취약점)을 악용하여 악성 코드를 삽입합니다.
    -   방문자가 해당 페이지에 접근하면 웹 브라우저 취약점 또는 드라이브 바이 다운로드(Drive-by Download)를 통해 악성코드가 실행됩니다.
-   **침투 메커니즘:** 웹사이트 방문 시 자동 악성코드 감염.
-   **방어 전략:** 웹 방화벽(WAF), 웹사이트 취약점 패치, 엔드포인트 보안 솔루션(EPP/EDR), 웹 필터링, 사용자 보안 교육.

### 1.3 원격 코드 실행(RCE) 취약점 활용 (Remote Code Execution)
-   **정의:** 소프트웨어 또는 시스템의 취약점을 악용하여 공격자가 원격에서 임의의 코드를 실행할 수 있게 하는 공격 기법입니다. 이는 공격자에게 시스템에 대한 완전한 제어권을 부여할 수 있어 가장 위험한 취약점 중 하나입니다.
-   **공격 방법:**
    -   웹 서버, 애플리케이션, 운영 체제, 미들웨어 등의 알려진 또는 제로데이(Zero-Day) 취약점을 스캔하고 식별합니다.
    -   취약점을 통해 원격에서 명령을 실행하거나 악성 파일을 업로드 및 실행합니다.
    -   (예시) SQL Injection을 통한 OS 명령 실행, 웹 셸(Web Shell) 업로드, 버퍼 오버플로우를 이용한 코드 실행.
-   **침투 메커니즘:** 원격에서 코드 실행을 통해 악성코드 설치 또는 직접적인 시스템 제어.
-   **방어 전략:** 정기적인 보안 패치 적용, 웹/애플리케이션 취약점 점검(SAST/DAST), 침입 방지 시스템(IPS), 입력 값 검증 강화, 최소 권한 원칙 적용.

## 2. 초기 침투 이후의 내부 이동(Lateral Movement) 기법 분석
공격자가 초기 침투에 성공한 후에는 네트워크 내부에서 공격 목표를 달성하기 위해 다른 시스템으로 이동하며 권한을 확장하는 내부 이동(Lateral Movement)을 시도합니다.

### 2.1 내부 이동의 주요 목적
-   **권한 상승:** 더 높은 권한(예: 도메인 관리자)을 가진 계정을 확보하여 시스템 제어권 강화.
-   **정보 수집:** 중요한 데이터가 저장된 서버 또는 시스템 식별.
-   **영향 범위 확장:** 더 많은 시스템을 감염시켜 공격의 효과 극대화.
-   **지속성 유지:** 여러 시스템에 거점을 확보하여 한 곳이 탐지되더라도 다른 경로로 접근 가능.

### 2.2 대표적인 내부 이동 기법
-   **자격 증명 탈취 (Credential Dumping):** Mimikatz와 같은 도구를 사용하여 메모리, 레지스트리 등에서 평문 또는 해시된 비밀번호를 탈취합니다.
-   **Pass-the-Hash / Pass-the-Ticket:** 탈취한 비밀번호 해시 또는 Kerberos 티켓을 사용하여 패스워드를 몰라도 다른 시스템에 인증합니다.
-   **원격 서비스 악용:**
    -   **PsExec:** Windows 시스템에서 원격으로 프로세스를 실행하거나 명령을 수행합니다.
    -   **WMI (Windows Management Instrumentation):** 원격 시스템에서 스크립트를 실행하거나 시스템 정보를 수집합니다.
    -   **RDP (Remote Desktop Protocol) 악용:** 탈취한 자격 증명으로 다른 시스템에 원격 데스크톱 연결을 시도합니다.
-   **공유 폴더 및 드라이브 악용:** 네트워크 공유 폴더를 통해 악성 파일을 유포하거나, 악성 스크립트를 실행합니다.
-   **SSH Brute-Force / Keylogging:** Linux/Unix 환경에서 SSH 자격 증명을 탈취하거나 무작위 대입 공격을 시도합니다.

## 핵심 요약
악성코드 침투 기술은 스피어 피싱, 워터링 홀, RCE 취약점 활용 등 다양하며, 각 기법에 대한 이해는 초기 침투 방어에 필수적입니다. 초기 침투 후 공격자는 자격 증명 탈취, 원격 서비스 악용, 공유 폴더 악용 등을 통해 내부 이동(Lateral Movement)을 시도하므로, 내부 네트워크 모니터링 및 세그멘테이션 강화가 중요합니다.
