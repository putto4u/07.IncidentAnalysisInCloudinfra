# 050. 악성코드로 인한 침해사고 사례연구 - 05. 공격자 관점의 악성코드 은닉 및 지속성 유지

## 개요
성공적인 사이버 공격은 단순히 시스템 침투를 넘어, 탐지를 회피하고 시스템 내부에 오랫동안 머무르면서 목표를 달성하는 데 달려있습니다. 이 챕터에서는 공격자가 악성코드를 은닉하고 시스템 내에서 지속성을 유지하기 위해 사용하는 다양한 기법들을 공격자 관점에서 심층 분석합니다. 난독화, 패킹, 파일리스 기법의 실제 사례와 레지스트리, 서비스, WMI 등을 활용한 지속성 확보 전략에 대해 학습합니다.

## 1. 탐지 우회를 위한 난독화 및 패킹 기법의 실제 사례
공격자는 악성코드의 코드나 실행 파일을 변형하여 시그니처 기반의 안티바이러스 솔루션이나 행위 기반 탐지 시스템의 눈을 피하려 합니다. 대표적인 기법으로 난독화와 패킹이 있습니다.

### 1.1 난독화 (Obfuscation)
-   **정의:** 악성코드의 소스 코드나 바이너리 코드를 분석하기 어렵게 만들지만, 기능적으로는 동일하게 작동하도록 변형하는 기법입니다. 이는 리버스 엔지니어링을 방해하고, 정적 분석 도구의 탐지를 우회하는 데 사용됩니다.
-   **주요 기법:**
    -   **문자열 난독화:** 악성코드 내의 중요한 문자열(C2 서버 주소, 파일명 등)을 암호화하거나 인코딩하여 저장하고, 실행 시점에 복호화하여 사용합니다. (예: Base64 인코딩, XOR 연산)
    -   **코드 난독화:** 불필요한 코드 삽입, 변수 및 함수명 변경, 제어 흐름 왜곡 등을 통해 코드의 가독성을 떨어뜨립니다.
    -   **API 호출 난독화:** 직접적인 API 호출 대신 동적으로 API 주소를 찾아 호출하거나, 간접적인 방법을 사용하여 탐지를 회피합니다.
-   **실제 사례:** Emotet, TrickBot과 같은 뱅킹 트로이목마 및 정보 탈취형 악성코드에서 널리 사용됩니다.

### 1.2 패킹 (Packing)
-   **정의:** 실행 가능한 파일을 압축하거나 암호화하여 원본 코드의 형태를 숨기는 기법입니다. 이는 파일 크기를 줄이는 목적도 있지만, 주로 안티바이러스 소프트웨어의 시그니처 탐지를 우회하기 위해 사용됩니다.
-   **주요 기법:**
    -   **표준 패커:** UPX, ASpack 등 공개된 패커를 사용합니다. (최근에는 탐지율이 높음)
    -   **커스텀 패커 (Crypter):** 공격자가 직접 개발하거나 구매한 패커로, 탐지 우회에 더 효과적입니다.
    -   **런타임 압축 해제:** 패킹된 악성코드는 실행 시점에 메모리에서 압축이 해제되고 원래의 코드가 실행됩니다.
-   **실제 사례:** 수많은 상업용 및 불법 악성코드에서 패킹 기술을 사용하며, 제로데이 공격 시에도 사용됩니다.

### 1.3 파일리스 악성코드 (Fileless Malware)
-   **정의:** 디스크에 파일을 생성하지 않고 메모리 상에서 직접 실행되거나, 운영 체제의 합법적인 도구(PowerShell, WMI 등)를 악용하여 동작하는 악성코드입니다. 이는 시그니처 기반의 파일 검사를 회피하고, 포렌식 분석을 어렵게 만듭니다.
-   **주요 기법:**
    -   **메모리 상주:** 공격 스크립트가 직접 메모리에 로드되어 실행되거나, 합법적인 프로세스에 인젝션되어 동작합니다.
    -   **레지스트리 상주:** 악성 코드를 레지스트리에 저장하고, 특정 이벤트 발생 시 PowerShell 등으로 이를 불러와 실행합니다.
    -   **합법적인 도구 악용 (Living off the Land, LotL):** 운영 체제에 기본적으로 설치된 PowerShell, WMIC, Certutil, Bitsadmin 등의 도구를 이용하여 악성 행위를 수행합니다.
-   **실제 사례:** Astaroth, fileless WannaCry, 여러 APT 그룹의 공격에서 파일리스 기법이 사용됩니다.

## 2. 레지스트리, 서비스, WMI 등을 활용한 지속성 확보 전략
악성코드가 시스템에 침투하여 활동하는 것만큼 중요한 것은 시스템 재부팅 후에도 자동으로 다시 실행되어 지속적인 통제권을 유지하는 것입니다. 공격자는 이를 위해 다양한 운영 체제 기능을 악용합니다.

### 2.1 레지스트리 (Registry) 활용
-   **Run 키:** `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 또는 `HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run` 등의 레지스트리 키에 악성코드 실행 경로를 등록하여 Windows 시작 시 자동 실행되도록 합니다.
-   **쉘 확장(Shell Extensions):** 특정 파일 형식이나 쉘 이벤트와 연결된 레지스트리 키를 조작하여 악성 코드가 실행되도록 합니다.

### 2.2 서비스 (Services) 활용
-   **서비스 등록:** Windows 서비스 관리자(services.msc)에 악성 서비스를 새로 등록하거나, 기존의 합법적인 서비스를 변조하여 시스템 시작 시 자동으로 악성 코드가 실행되도록 합니다.
-   **Persistence via Service:** 악성 실행 파일을 서비스 실행 경로로 지정하고, 서비스를 자동 시작 유형으로 설정합니다.

### 2.3 작업 스케줄러 (Scheduled Tasks) 활용
-   **자동 실행 등록:** `schtasks` 명령어나 GUI 도구를 사용하여 특정 시간, 로그인 시, 시스템 이벤트 발생 시 악성 코드가 실행되도록 예약 작업을 등록합니다.
-   **장점:** 재부팅 후에도 지속성을 유지하며, 특정 조건을 만족할 때만 실행되도록 설정하여 탐지를 회피할 수 있습니다.

### 2.4 WMI (Windows Management Instrumentation) 활용
-   **Event Consumers:** WMI 이벤트 시스템을 이용하여 특정 시스템 이벤트(예: 프로세스 생성, 파일 변경) 발생 시 악성 코드가 자동으로 실행되도록 설정합니다.
-   **WMI Permanent Event Subscription:** 스크립트나 명령어가 시스템 이벤트에 반응하여 실행되도록 영구적인 WMI 이벤트를 구독합니다. 이는 파일리스 지속성을 확보하는 강력한 방법입니다.

### 2.5 다른 합법적인 도구 및 경로 악용
-   **시작 프로그램 폴더:** `Startup` 폴더에 악성 실행 파일이나 바로가기 파일을 복사하여 로그인 시 자동 실행되도록 합니다.
-   **환경 변수 조작:** PATH 환경 변수를 조작하여 악성 실행 파일의 우선순위를 높이거나, 시스템 PATH에 악성코드 경로를 추가합니다.
-   **DLL 하이재킹:** 합법적인 프로그램이 로드하는 DLL 파일을 위장하거나 교체하여 악성 코드를 실행합니다.

## 핵심 요약
공격자는 난독화, 패킹, 파일리스 기법을 통해 악성코드의 탐지를 우회하고, 레지스트리, 서비스, 작업 스케줄러, WMI 등을 활용하여 시스템 내에서 지속성을 확보합니다. 이러한 공격 기법에 대한 이해는 효과적인 악성코드 탐지 및 제거 전략을 수립하는 데 필수적입니다. EDR 솔루션, 행위 기반 분석, 포렌식 분석, 그리고 최소 권한 원칙 적용 등이 중요한 방어 수단입니다.
