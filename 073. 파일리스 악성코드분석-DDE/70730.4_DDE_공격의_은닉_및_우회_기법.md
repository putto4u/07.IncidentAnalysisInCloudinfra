# 70730.4 DDE 공격의 은닉 및 우회 기법

## 1. 정적 탐지 우회 (Static Bypass)

### 1.1 필드 코드 분할 및 숨김
- Word 문서 내에서 글자 크기를 1pt로 설정하거나 흰색 글자로 설정하여 사용자 눈에 보이지 않게 처리.
- 텍스트 박스나 이미지 뒤에 필드 코드를 숨김.

### 1.2 XML 구조 조작
- `.docx` 파일은 실제 압축 파일(ZIP) 형태이며, 내부 `word/document.xml`에 데이터가 저장됩니다.
- 공격자는 XML 태그 사이에 무의미한 주석이나 태그를 삽입하여 `DDEAUTO` 문자열 검색을 우회합니다.

## 2. 사용자 경고창 우회 및 유도

### 2.1 심리적 기법 (Social Engineering)
- 첫 번째 경고창에서 '예'를 누르도록 문서 상단에 "데이터 연결을 활성화하려면 '예'를 클릭하십시오"라는 가짜 안내 이미지를 삽입.

### 2.2 경고 메시지 커스텀 (제한적)
- DDE `Item` 파라미터에 긴 공백이나 개행 문자를 넣어 실제 실행되는 명령어(`cmd.exe` 등)가 경고창 하단으로 밀려나 보이지 않게 유도.

## 3. 파일 포맷 변형
- **RTF(Rich Text Format) 활용:** `.doc` 뿐만 아니라 `.rtf` 포맷에서도 DDE 개체 삽입이 가능하며, RTF는 구조가 복잡하여 분석 도구의 파싱 에러를 유발하기 쉽습니다.
- **CSV/Excel DDE:** CSV 파일 내에서 `=cmd|' /c calc'!A1`과 같은 수식 형태로 DDE를 트리거할 수 있습니다.

## 4. LOLBins(Living Off the Land Binaries) 결합
- `mshta.exe`, `regsvr32.exe`, `rundll32.exe` 등을 활용하여 신뢰할 수 있는 프로세스 트리 뒤로 숨어 탐지를 피합니다.
