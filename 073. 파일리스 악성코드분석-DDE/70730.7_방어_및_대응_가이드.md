# 70730.7 방어 및 대응 가이드

## 1. 기술적 방어 설정 (OS/Registry)

### 1.1 레지스트리를 통한 DDE 비활성화
가장 확실한 방법은 Office에서 DDE 자동 업데이트 기능을 강제로 끄는 것입니다.
- **경로:** `HKEY_CURRENT_USER\Software\Microsoft\Office\<version>\<App>\Options`
- **값 이름:** `DontUpdateLinks` (DWORD)
- **데이터:** `1` (비활성화)

### 1.2 DDE 요청 수락 여부 설정
- `HKEY_CURRENT_USER\Software\Microsoft\Office\<version>\<App>\Options`
- **값 이름:** `DDEAlerts` (DWORD)
- **데이터:** `0` (경고 창 없이 차단)

## 2. 그룹 정책(GPO) 활용
기업 환경에서는 AD(Active Directory) GPO를 통해 일괄 적용합니다.
- `사용자 구성` > `관리 템플릿` > `Microsoft Word 2016` > `Word 옵션` > `고급` > `문서를 열 때 자동 링크 업데이트` 선택 해제.

## 3. 보안 솔루션 강화

### 3.1 EDR/AV 정책
- Office 프로세스의 자식 프로세스 생성을 차단하는 **공격 표면 축소(ASR, Attack Surface Reduction)** 규칙 적용.
- `Block all Office applications from creating child processes` (GUID: `d4f940ab-401b-4efc-aadc-ad5f3c50688a`)

### 3.2 메일 게이트웨이 (SEG)
- 첨부 파일의 XML 구조를 검사하여 `DDE`, `DDEAUTO` 키워드가 포함된 경우 격리.
- RTF 파일에 대한 심층 검사 강화.

## 4. 사용자 교육 (User Awareness)
- "데이터 업데이트" 또는 "애플리케이션 실행" 경고창이 뜰 경우 절대로 '예'를 누르지 않도록 교육.
- 출처가 불분명한 문서의 링크 업데이트 거부 강조.

## 5. 사후 대응 (Incident Response)
- DDE 공격 발생 시 해당 문서의 해시(MD5/SHA256)를 블랙리스트에 등록.
- 영향을 받은 호스트의 네트워크 연결을 차단하고 C2 통신 흔적 조사.
