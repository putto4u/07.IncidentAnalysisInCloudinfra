# 70730.1 DDE(Dynamic Data Exchange) 기술의 이해

## 1. DDE 개요
DDE(Dynamic Data Exchange)는 Microsoft Windows에서 애플리케이션 간에 데이터를 주고받기 위해 설계된 고전적인 통신 프로토콜입니다. 1987년 Windows 2.0에서 처음 도입되었으며, 공유 메모리를 사용하여 애플리케이션 간에 메시지를 전달하는 클라이언트-서버 모델을 따릅니다.

### 1.1 DDE의 목적
- 한 애플리케이션의 데이터가 변경될 때 다른 애플리케이션에 자동으로 반영 (예: Excel 시트의 값이 Word 문서의 표에 자동 업데이트).
- 사용자 개입 없이 실시간 데이터 스트리밍 처리.
- 레거시 시스템과의 하위 호환성 유지.

## 2. DDE 통신 아키텍처
DDE 통신은 세 가지 계층으로 정의됩니다.

| 계층 (Layer) | 설명 | 예시 |
| :--- | :--- | :--- |
| **Application** | 데이터를 제공하거나 요청하는 프로그램의 이름 | `Excel`, `Winword`, `CMD` |
| **Topic** | 데이터 교환이 이루어지는 데이터의 논리적 그룹 | 파일 경로, `System` |
| **Item** | 실제 전송되는 세부 데이터 단위 | 셀 주소 (R1C1), 특정 명령 |

## 3. Microsoft Office에서의 DDE 작동 방식
Office 문서(Word, Excel) 내에서 DDE는 **필드 코드(Field Code)** 형식을 통해 구현됩니다. 
- **필드 중괄호:** `{ }` 내에 명령어를 삽입합니다.
- **주요 명령어:**
    - `DDE`: 요청 시에만 데이터를 업데이트합니다.
    - `DDEAUTO`: 문서가 열릴 때 자동으로 데이터를 업데이트합니다.

## 4. OLE(Object Linking and Embedding)와의 관계
DDE는 OLE 기술의 조상격으로, OLE 1.0은 내부적으로 DDE를 기반으로 구축되었습니다. 최신 Office 버전은 더 안전한 OLE 2.0(COM 기반)을 주로 사용하지만, 하위 호환성을 위해 DDE 엔진은 여전히 운영체제와 Office 내에 잔존하고 있습니다.

## 5. 보안적 관점에서의 DDE
DDE는 매크로(VBA)를 사용하지 않고도 외부 프로세스를 호출할 수 있다는 점에서 '파일리스(Fileless)' 공격의 핵심 기법으로 활용됩니다. 
- **공격자의 이점:** 매크로 실행 차단 정책을 우회할 수 있으며, 사용자에게는 단순한 "데이터 업데이트" 경고창만 노출됩니다.
