# 70730.2 DDE 기반 공격 시나리오 분석

## 1. DDE 공격의 특징
DDE 공격은 "Macro-less" 공격으로 분류됩니다. 피해자가 문서를 열 때 매크로 활성화 버튼(콘텐츠 사용) 대신, 외부 데이터 업데이트 여부를 묻는 시스템 메시지를 이용합니다.

## 2. 일반적인 공격 흐름 (Attack Flow)

### 2.1 초기 침투 (Delivery)
- **피싱 메일:** 송장, 급여 명세서, 보안 업데이트 안내 등으로 위장된 Word/Excel 첨부 파일 전송.
- **Social Engineering:** 사용자가 문서를 열도록 유도.

### 2.2 실행 단계 (Execution)
1. **문서 오픈:** 사용자가 첨부 파일을 실행.
2. **첫 번째 경고창:** "이 문서에는 다른 파일을 참조하는 링크가 포함되어 있습니다. 링크의 데이터로 이 문서를 업데이트하시겠습니까?" (사용자가 '예' 클릭 유도)
3. **두 번째 경고창:** 특정 애플리케이션(예: cmd.exe)을 시작하려 한다는 경고. (사용자가 '예' 클릭 시 악성 코드 실행)

### 2.3 페이로드 로딩 (Payload Loading)
- DDE 필드 내의 명령어가 실행되어 `powershell.exe` 또는 `bitsadmin.exe` 등을 호출.
- 원격 서버(C2)에서 최종 악성 페이로드(RAT, Ransomware 등)를 다운로드 및 메모리 상에서 실행.

## 3. 공격 시나리오 예시: APT 캠페인
- **금융권 타겟:** Excel 파일 내에 DDEAUTO를 삽입하여 주식 정보 업데이트로 위장.
- **정부 기관 타겟:** Word 문서의 컨트롤러(Control) 개체에 DDE를 숨겨두어 분석가의 눈을 피함.

## 4. DDE 공격의 위험성
- **파일리스 특성:** 디스크에 악성 파일이 직접 생성되지 않고 시스템 도구(LOLBins)를 이용하므로 전통적인 백신(AV) 탐지가 어렵습니다.
- **신뢰성 오용:** Microsoft Office라는 신뢰할 수 있는 프로세스 내부에서 발생하므로 행위 기반 탐지 로직을 우회할 가능성이 높습니다.
