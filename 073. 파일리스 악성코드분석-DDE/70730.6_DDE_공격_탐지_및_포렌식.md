# 70730.6 DDE 공격 탐지 및 포렌식

## 1. 로그 기반 탐지 (SIEM/EDR)

### 1.1 Sysmon 이벤트 ID 1 (Process Creation)
- **탐지 규칙:** `ParentImage`가 `winword.exe`, `excel.exe`, `powerpnt.exe` 중 하나이면서 `Image`가 `cmd.exe`, `powershell.exe`인 경우.
- **상관관계 분석:** 단시간 내에 발생하는 다수의 하위 프로세스 생성 패턴 확인.

### 1.2 Windows 이벤트 로그
- **Event ID 4688:** 프로세스 생성 로그 (명령줄 포함 옵션 활성화 필요).

## 2. 파일 아티팩트 포렌식

### 2.1 Office 필드 코드 추출 (OLEVBA / OLEDUMP)
`Oletools`를 사용하여 문서 내의 DDE 문자열을 정적으로 추출합니다.
```bash
oledump.py sample.docx -s 4 -d  # 특정 스트림에서 데이터 추출
olevba sample.docx              # DDE 및 매크로 정보 분석
```

### 2.2 XML 구조 수동 분석
`.docx` 파일의 확장자를 `.zip`으로 변경 후 압축 해제.
- `word/document.xml` 파일 내에서 `<w:instrText>` 태그 확인.
- `DDE`, `DDEAUTO` 키워드 검색.

## 3. 레지스트리 흔적
Office 프로그램이 DDE를 통해 외부 애플리케이션을 호출할 때 남을 수 있는 레지스트리 흔적을 조사합니다.
- `HKEY_CURRENT_USER\Software\Microsoft\Office\<version>\<app>\Security\Trusted Documents\TrustRecords`

## 4. YARA 룰 예시
```yara
rule Detect_DDE_Malicious_Doc {
    strings:
        $s1 = "dde" nocase
        $s2 = "ddeauto" nocase
        $s3 = "cmd.exe" nocase
        $s4 = "powershell" nocase
    condition:
        (uint32(0) == 0x04034b50 or uint32(0) == 0xd0cf11e0) and
        (($s1 or $s2) and ($s3 or $s4))
}
```
