# 70730.5 동적 분석을 통한 페이로드 추적

## 1. 분석 환경 구성
- **Isolation:** 외부 인터넷이 차단된 폐쇄망 VM(Windows 10/11).
- **Monitoring Tools:** 
    - Process Hacker / Process Explorer
    - Sysmon (System Monitor)
    - Procmon (Process Monitor)
    - Wireshark / Fiddler

## 2. 단계별 분석 절차

### 2.1 프로세스 트리 모니터링
문서를 실행한 후 발생하는 하위 프로세스(Child Process)를 추적합니다.
- **정상:** `winword.exe` (단독 실행)
- **이상:** `winword.exe` -> `cmd.exe` -> `powershell.exe` -> ...
- **주안점:** 상위 프로세스가 Office 프로그램인데 갑자기 시스템 도구가 실행되는지 확인.

### 2.2 명령줄 인자(Command Line) 분석
`Procmon` 또는 `Process Hacker`를 통해 실행된 자식 프로세스의 명령줄 인자를 추출합니다.
- 예: `powershell -enc JABz...` (Base64 인코딩된 스크립트 발견 시 디코딩 수행)

### 2.3 네트워크 트래픽 분석
- `Wireshark`를 통해 특정 도메인으로의 HTTP GET/POST 요청 확인.
- 다운로드되는 파일의 시그니처(MZ, PK 등)를 확인하여 페이로드 유형 파악.

## 3. 메모리 포렌식 (Advanced)
- 프로세스가 종료되기 전 `Volatility`나 `Process Dump`를 사용하여 메모리 덤프 수행.
- 메모리 상에 존재하는 쉘코드(Shellcode)나 압축 해제된 스크립트 추출.

## 4. API 후킹 및 로깅
- `Any.Run`과 같은 온라인 샌드박스를 활용하여 DDE 호출 시 발생하는 API 이벤트를 시각적으로 분석.
