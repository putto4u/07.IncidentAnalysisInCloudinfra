# 7. 결론 및 베스트 프랙티스

## 7.1 효과적인 이벤트 탐지 전략 (Best Practices)
1. **최소 권한의 원칙 (Least Privilege)**: 이벤트 발생 자체를 줄이기 위해 IAM 권한을 최소화합니다.
2. **구조화된 로깅**: 모든 로그를 JSON 등 기계가 읽기 쉬운 포맷으로 통일합니다.
3. **거버넌스 준수**: 전 계정, 전 리전에 대해 로깅 서비스(CloudTrail 등)가 항상 켜져 있도록 강제(Service Control Policy)합니다.
4. **정기적인 'Red Teaming'**: 침해 사고 시나리오를 바탕으로 탐지 시스템이 제대로 동작하는지 모의 훈련을 실시합니다.

## 7.2 최신 기술 트렌드
- **CNAPP (Cloud Native Application Protection Platform)**: CSPM, CWPP, CIEM을 하나로 통합하여 개발부터 운영까지 전체 라이프사이클을 보호하는 개념.
- **DSPM (Data Security Posture Management)**: 데이터의 위치, 중요도, 접근 권한을 중심으로 보안 위협을 탐지.
- **Generative AI in Security**: LLM을 활용하여 복잡한 보안 로그를 자연어로 요약하거나, 탐지 규칙을 자동으로 생성.

## 7.3 마무리
클라우드 보안은 '완벽한 방어'가 아닌 '지속적인 가시성 확보와 신속한 탐지/대응'의 영역입니다. 인프라가 변하는 속도에 맞춰 보안 탐지 전략도 지속적으로 최적화(Tuning)되어야 합니다.
