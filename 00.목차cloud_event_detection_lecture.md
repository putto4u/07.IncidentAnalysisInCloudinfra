# 강의명: 클라우드 환경에서 발생하는 이벤트 탐지 이해

## 1. 클라우드 이벤트 탐지 개요

- 클라우드 컴퓨팅과 보안 책임 공유 모델(Shared Responsibility Model)
- 이벤트 탐지의 정의 및 필요성
- 전통적인 온프레미스 환경과의 차이점

## 2. 클라우드 인프라 내 주요 이벤트 유형

- **제어 평면(Control Plane) 이벤트**: API 호출, 리소스 생성/삭제, 설정 변경 (예: AWS CloudTrail)
- **데이터 평면(Data Plane) 이벤트**: S3 객체 접근, 데이터베이스 쿼리 등
- **네트워크 이벤트**: VPC Flow Logs, DNS 쿼리 로그, 로드 밸런서 로그
- **워크로드 및 애플리케이션 이벤트**: 시스템 로그, 애플리케이션 에러 로그, 컨테이너 런타임 이벤트

## 3. 이벤트 수집 및 모니터링 메커니즘

- CSP별 기본 모니터링 도구 이해 (AWS CloudWatch, Azure Monitor, GCP Cloud Logging)
- 로그 통합 및 중앙 집중화 전략
- 실시간 스트리밍 처리와 배치 처리의 차이

## 4. 탐지 기술 및 분석 방법론

- **규칙 기반 탐지 (Rule-based Detection)**: 특정 패턴 및 임계치 설정
- **이상 징후 탐지 (Anomaly Detection)**: 머신러닝을 활용한 평소와 다른 행위 식별
- **위협 인텔리전스 활용**: 알려진 악성 IP 및 도메인 대조
- **런타임 보안 (Runtime Security)**: 컨테이너 및 서버리스 환경 내 실행 중인 프로세스 감시

## 5. 주요 위협 시나리오별 탐지 사례

- 자격 증명 탈취 및 비정상적인 로그인 시도
- 리소스 오남용 (예: 크립토재킹을 위한 인스턴스 생성)
- 데이터 유출 (Data Exfiltration) 시도 탐지
- 권한 상승(Privilege Escalation) 및 백도어 생성

## 6. 대응 및 자동화 (Incident Response)

- 알림 및 가시성 확보 (Dashboard, Slack/Email 알림)
- 자동화된 조치 (Automated Remediation): Lambda 등을 활용한 격리 및 차단
- 보안 오케스트레이션 및 자동 대응 (SOAR) 개념 소개

## 7. 결론 및 베스트 프랙티스

- '최소 권한의 원칙'과 로깅 전략
- 지속적인 모니터링 및 탐지 규칙 최적화
- 향후 클라우드 보안 트렌드 (CNAPP, DSPM 등)
