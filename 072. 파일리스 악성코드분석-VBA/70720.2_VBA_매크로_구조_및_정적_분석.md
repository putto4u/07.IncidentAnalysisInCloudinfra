# VBA 매크로 구조 및 정적 분석

## 1. 개요
VBA 매크로를 분석하기 위해서는 오피스 문서의 내부 구조를 이해해야 합니다. 특히 구형 바이너리 형식(OLE)과 신규 XML 기반 형식(OOXML) 내에서 매크로가 저장되는 위치와 실행 방식을 파악하는 것이 중요합니다.

## 2. 상세 분석 내용
### OLE 구조와 vbaProject.bin 분석 방법
- **OLE (Object Linking and Embedding)**: .doc, .xls와 같은 구형 오피스 문서 파일 형식입니다. `vbaProject.bin` 파일 내에 매크로 코드가 스트림 형태로 저장됩니다.
- **분석 도구**: `olebrowse` 등을 통해 내부 스트림 구조를 확인하고, `vbaProject.bin`에서 실제 소스 코드가 포함된 모듈 스트림을 식별합니다.

### AutoOpen, Document_Open 등 자동 실행 트리거 분석
- **이벤트 핸들러**: 사용자가 문서를 열 때 자동으로 코드를 실행하기 위해 공격자는 특정 함수명을 사용합니다.
- **주요 트리거**:
    - `AutoOpen` (Word) / `Auto_Open` (Excel): 문서가 열릴 때 즉시 실행됩니다.
    - `Document_Open` (Word) / `Workbook_Open` (Excel): 개체 기반의 자동 실행 이벤트입니다.
    - `AutoClose`, `Document_Close`: 문서가 닫힐 때 실행되어 흔적을 삭제하거나 추가 작업을 수행합니다.

## 3. 대응 및 방어 전략
- **정적 시그니처 검사**: 안티바이러스 및 EDR 솔루션에서 알려진 자동 실행 트리거 함수명을 탐지 규칙에 활용합니다.
- **파일 형식 제한**: 매크로가 포함될 수 없는 형식(.docx, .xlsx)을 권장하고, 매크로 활성 형식(.docm, .xlsm)에 대한 집중 모니터링을 실시합니다.

## 4. 요약 및 결론
VBA 매크로는 특정 자동 실행 트리거를 통해 사용자의 개입 없이 코드를 실행할 수 있습니다. `vbaProject.bin`과 같은 내부 바이너리 구조를 분석하여 매크로의 존재 여부와 초기 진입점을 찾는 것이 정적 분석의 핵심입니다.
