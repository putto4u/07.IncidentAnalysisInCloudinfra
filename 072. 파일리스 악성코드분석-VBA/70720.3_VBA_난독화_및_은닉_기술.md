# VBA 난독화 및 은닉 기술

## 1. 개요
공격자는 보안 솔루션의 정적 탐지를 우회하기 위해 다양한 난독화 및 은닉 기법을 사용합니다. 코드의 가독성을 떨어뜨리고 악성 행위를 숨기는 것이 주된 목적입니다.

## 2. 상세 분석 내용
### 문자열 결합, ASCII 변환, 가짜 함수 활용 분석
- **문자열 조작**: `powershell.exe`와 같은 민감한 키워드를 직접 사용하지 않고 `"p" & "owershell" & ".exe"`와 같이 분할 결합하거나, `Chr()` 함수를 이용해 ASCII 값으로 변환하여 표현합니다.
- **가짜 함수(Junk Code)**: 실행에 아무런 영향을 주지 않는 수많은 무의미한 함수와 변수를 삽입하여 분석가의 분석을 방해하고 탐지 엔진의 연산량을 늘립니다.

### 문서 내 숨겨진 컨트롤 및 텍스트 박스 활용 기법
- **UserForms 활용**: VBA 폼(UserForm) 내의 캡션, 태그, 혹은 텍스트 박스 속성에 악성 페이로드나 인코딩된 데이터를 숨겨둡니다.
- **문서 본문 은닉**: 흰색 글씨체, 아주 작은 폰트 크기 등으로 문서 본문에 데이터를 숨긴 뒤, 매크로 실행 시 해당 데이터를 읽어들여 실행합니다.

## 3. 대응 및 방어 전략
- **동적 분석 연계**: 난독화된 코드는 결국 실행 시점에 원래의 모습을 드러냅니다. 샌드박스 환경에서 매크로가 실행될 때 호출되는 API나 프로세스 생성 행위를 모니터링해야 합니다.
- **매크로 코드 검사**: 비정상적으로 많은 문자열 결합 연산자(&)나 `Chr()` 함수의 과도한 사용을 탐지 지표로 활용합니다.

## 4. 요약 및 결론
난독화는 공격자가 시간을 벌기 위한 수단입니다. 문자열 변조 기법과 문서 내 개체 속성을 활용한 데이터 은닉 방식을 이해하면, 복잡해 보이는 코드 속에서도 핵심적인 악성 로직을 찾아낼 수 있습니다.
