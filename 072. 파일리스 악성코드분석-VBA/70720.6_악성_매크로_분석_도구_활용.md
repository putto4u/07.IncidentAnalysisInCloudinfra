# 악성 매크로 분석 도구 활용

## 1. 개요
신속하고 정확한 악성 매크로 분석을 위해서는 특화된 도구의 활용이 필수적입니다. 정적 분석 도구는 수동 분석의 수고를 덜어주고, 숨겨진 위험 요소를 자동으로 식별해 줍니다.

## 2. 상세 분석 내용
### Oletools(olevba, oledump)를 이용한 신속한 분석
- **olevba**: 매크로 소스 코드를 추출하고, 위험한 API 호출, 문자열 난독화, 자동 실행 트리거 등을 요약해서 보여줍니다.
- **oledump**: OLE 파일 내의 개별 스트림을 확인하고, 매크로가 포함된 스트림(보통 'M'으로 표시됨)을 선별적으로 추출하여 분석할 수 있게 합니다.
- **분석 프로세스**: `oledump`로 구조 파악 -> `olevba`로 코드 분석 및 위험 지표 확인 순으로 진행합니다.

### 정적 스캔 우회를 위한 P-code 공격 분석
- **VBA 저장 방식**: VBA는 소스 코드와 함께 컴파일된 바이너리 형태인 'P-code'를 함께 저장합니다. 오피스는 실행 시 P-code를 우선적으로 사용합니다.
- **VBA Stomping**: 공격자가 소스 코드 영역을 무의미한 코드로 덮어쓰고 P-code 영역에만 악성 코드를 남겨두는 기법입니다. 소스 코드만 분석하는 도구는 이를 놓칠 수 있습니다.

## 3. 대응 및 방어 전략
- **교차 분석**: 소스 코드 영역과 P-code 영역을 모두 검사할 수 있는 분석 도구(pcodedmp 등)를 사용합니다.
- **도구 업데이트**: 최신 매크로 기법(VBA Stomping 등)을 지원하는 최신 버전의 분석 도구(Oletools 0.60 이상)를 활용합니다.

## 4. 요약 및 결론
도구 활용은 분석의 효율성을 극대화합니다. 특히 P-code 공격과 같은 정적 우회 기법을 이해하고, 상황에 맞는 도구를 선택하여 분석 범위를 넓히는 것이 중요합니다.
