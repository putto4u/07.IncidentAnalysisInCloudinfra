# VBA 및 Windows API 활용 분석

## 1. 개요
고급 악성 매크로는 단순히 쉘 명령을 내리는 수준을 넘어, 직접 Windows API를 호출하여 정교한 침투 및 은닉 행위를 수행합니다. 이를 통해 메모리 직접 조작이나 프로세스 인젝션이 가능해집니다.

## 2. 상세 분석 내용
### 매크로 내에서 직접 호출되는 Win32 API 추적
- **API 선언**: `Declare Sub` 또는 `Declare Function` 구문을 사용하여 `kernel32.dll`, `user32.dll` 등 외부 DLL의 함수를 불러옵니다.
- **주요 악성 API**: 
    - `URLDownloadToFile`: 파일을 다운로드합니다.
    - `VirtualAlloc`, `WriteProcessMemory`, `CreateRemoteThread`: 쉘코드 실행 및 프로세스 인젝션에 사용됩니다.
    - `GetTickCount`, `Sleep`: 샌드박스의 가속 실행을 감지하거나 분석 시간을 지연시킵니다.

### 프로세스 인젝션 및 가상환경 탐지 루틴 분석
- **인젝션 기법**: 매크로 실행 프로세스(예: winword.exe) 내부에 쉘코드를 삽입하거나, 다른 정상 프로세스를 생성한 뒤 코드를 주입하여 실행합니다.
- **안티-VM/안티-샌드박스**: 특정 파일 경로의 존재 여부, 레지스트리 키, 디스크 용량, 코어 개수 등을 확인하여 분석 환경일 경우 동작을 멈춥니다.

## 3. 대응 및 방어 전략
- **AMSI (Antimalware Scan Interface) 연동**: 최신 오피스 버전은 VBA 실행 시 AMSI를 통해 실시간으로 코드를 검사하므로, 이를 우회하는 코드를 중점적으로 모니터링합니다.
- **API 호출 가시성 확보**: EDR 솔루션을 통해 비정상적인 API 호출 패턴(메모리 할당 후 스레드 생성 등)을 실시간 탐지합니다.

## 4. 요약 및 결론
VBA에서의 Windows API 활용은 공격의 정교함을 보여줍니다. API 선언부와 인자 값을 분석하면 매크로가 수행하려는 최종적인 악성 행위(쉘코드 실행, 인젝션 등)를 명확히 파악할 수 있습니다.
