# 103.1 네트워크 포렌식 아키텍처 및 데이터 소스

## 1. 네트워크 포렌식 개요
네트워크 포렌식은 네트워크 트래픽을 수집, 기록 및 분석하여 정보 보안 사고의 원인을 규명하고 공격자의 행위를 추적하는 과정입니다. 시스템 포렌식과 달리 '데이터의 전송' 과정에 집중합니다.

## 2. 주요 데이터 소스 및 수집 방식
### 2.1 Full Packet Capture (FPC)
- **정의:** 네트워크를 통과하는 모든 패킷을 페이로드까지 포함하여 저장하는 방식입니다.
- **장점:** 완벽한 가시성을 제공하며, 공격자가 주고받은 파일이나 실행한 명령어를 그대로 복구할 수 있습니다.
- **단점:** 막대한 저장 공간과 처리 성능이 필요합니다.

### 2.2 Flow Data (NetFlow, IPFIX)
- **정의:** 통신의 메타데이터(출발지/목적지 IP, 포트, 프로토콜, 패킷 수, 바이트 수 등)만을 기록합니다.
- **장점:** 저장 공간 효율성이 높고 대규모 네트워크의 트래픽 흐름을 한눈에 파악하기 용이합니다.
- **용도:** 대량의 데이터 유출 탐지, 비정상적인 통신 패턴(DDoS, 스캔) 분석에 적합합니다.

## 3. 네트워크 장비 로그의 가치
- **Firewall Log:** 차단된 세션과 허용된 세션의 기록을 통해 공격자의 초기 시도 확인.
- **IDS/IPS Log:** 이미 알려진 공격 패턴에 대한 탐지 결과 제공.
- **Web Proxy Log:** 내부 사용자의 외부 웹사이트 접속 기록, 특히 C2 서버와의 통신 및 악성 코드 다운로드 경로 추적에 필수적입니다.

## 4. 수집 아키텍처 구성
- **TAP (Test Access Point):** 물리적으로 회선을 분기하여 트래픽 손실 없이 수집.
- **SPAN (Switched Port Analyzer):** 스위치의 미러링 기능을 이용. 설정이 간편하지만 스위치 부하 시 패킷 유실 가능성이 있습니다.
