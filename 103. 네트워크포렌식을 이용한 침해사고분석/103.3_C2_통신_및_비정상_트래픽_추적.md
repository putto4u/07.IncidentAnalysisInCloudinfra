# 103.3 C2 통신 및 비정상 트래픽 추적

## 1. C2 (Command and Control) 통신의 특징
공격자는 감염된 호스트를 제어하기 위해 외부의 C2 서버와 통신합니다. 이를 탐지하는 것은 침해 사고 분석의 핵심입니다.

## 2. 비컨(Beacon) 패턴 분석
- **정의:** 악성 코드가 주기적으로 C2 서버에 접속하여 명령이 있는지 확인하는 행위입니다.
- **분석 방법:** 통신 간격(Interval)과 지터(Jitter) 분석.
- **식별:** 네트워크 로그에서 특정 목적지 IP로 일정 시간마다 발생하는 소량의 패킷 전송 기록을 추적합니다.

## 3. 통신 은닉 기법 및 탐지
### 3.1 비표준 포트 활용
- 웹 서비스 포트(80, 443)가 아닌 포트로 HTTP 통신을 하거나, 반대로 웹 포트를 통해 SSH/RDP 통신을 시도하는 경우를 탐지합니다.

### 3.2 터널링 (Tunneling)
- **DNS Tunneling:** DNS 쿼리 페이로드에 데이터를 숨겨 방화벽을 우회.
- **ICMP Tunneling:** Ping 패킷의 Data 영역에 명령어나 데이터를 숨겨 통신.
- **분석:** 패킷의 크기가 비정상적으로 크거나 프로토콜 규약을 벗어난 데이터가 포함되어 있는지 확인합니다.

## 4. 좀비 PC 및 봇넷 식별
- 다수의 내부 호스트가 동일한 외부 IP로 동시 통신하거나 유사한 패턴을 보일 때 봇넷 감염을 의심합니다.
- 위협 인텔리전스(CTI) 피드를 활용하여 알려진 악성 IP와의 연결을 대조합니다.
