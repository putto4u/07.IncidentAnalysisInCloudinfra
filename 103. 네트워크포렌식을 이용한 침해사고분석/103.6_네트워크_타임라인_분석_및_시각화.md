# 103.6 네트워크 타임라인 분석 및 시각화

## 1. 네트워크 이벤트 재구성
침해 사고의 전체 그림을 이해하기 위해서는 네트워크 상에서 발생한 일들을 시간 순서대로 나열해야 합니다.

## 2. 타임라인 생성 프로세스
1. **데이터 통합:** 방화벽 로그, 웹 로그, PCAP 분석 결과를 하나의 시간축으로 결합.
2. **정규화:** 서로 다른 로그 형식을 표준 포맷으로 변환.
3. **핵심 이벤트 추출:** 단순 접속이 아닌, 파일 다운로드, C2 연결 성공, 대량 유출 시작 등 결정적 순간을 마킹.

## 3. 시각화 기법
### 3.1 Flow Graph
- 시스템 간의 패킷 교환 순서(Sequence Diagram)를 시각화하여 특정 공격 기법(예: 취약점 익스플로잇) 시점 확인.

### 3.2 Graph Database 활용 (예: Neo4j)
- '호스트'를 노드로, '통신'을 링크로 표현.
- 내부망 이동(Lateral Movement) 경로를 직관적으로 파악할 수 있습니다.

### 3.3 히트맵 (Heatmap)
- 시간대별 트래픽 양을 시각화하여 공격자가 활동한 주요 시간대를 파악합니다.

## 4. 맥락 분석 (Contextual Analysis)
- 특정 네트워크 이벤트가 발생하기 전후에 어떤 시스템 로그가 남았는지 연계하여 공격자의 의도를 파악합니다.
