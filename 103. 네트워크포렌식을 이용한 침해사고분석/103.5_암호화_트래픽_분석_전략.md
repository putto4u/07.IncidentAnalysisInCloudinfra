# 103.5 암호화 트래픽 분석 전략

## 1. 암호화 트래픽의 도전 과제
최근 트래픽의 90% 이상이 HTTPS(TLS)로 암호화되어 있어, 단순한 패킷 캡처만으로는 내부 페이로드를 확인할 수 없습니다.

## 2. SSL/TLS 핸드쉐이크 분석 (비복호화 분석)
복호화 없이도 메타데이터를 통해 위협을 탐지할 수 있습니다.
- **SNI (Server Name Indication):** 접속하려는 도메인 확인.
- **Certificate Analysis:** 유효하지 않은 인증서, 자기 서명(Self-signed) 인증서, 또는 블랙리스트에 등록된 인증서 확인.
- **JA3 Fingerprint:** 클라이언트의 TLS 핸드쉐이크 특성을 추출하여 특정 악성 코드나 라이브러리를 식별.

## 3. 가시성 확보 방안 (복호화 분석)
### 3.1 SSL/TLS Inspection (MitM)
- 보안 장비(Proxy, Firewall)에서 인증서를 대행 발급하여 중간에서 복호화한 후 분석.
- 기업 내부 망에서 가장 흔히 사용되는 방식입니다.

### 3.2 Key Log File 활용
- 분석 대상 PC에서 `SSLKEYLOGFILE` 환경 변수를 설정하여 브라우저의 세션 키를 기록.
- Wireshark에 이 키 파일을 연동하여 암호화된 트래픽을 실시간으로 복호화하여 확인.

## 4. 암호화된 C2 통신 탐지
- 패킷의 크기, 도착 간격 등 통계적 특성(Traffic Analysis)을 머신러닝 모델에 학습시켜 암호화된 상태에서도 악성 행위를 추정합니다.
