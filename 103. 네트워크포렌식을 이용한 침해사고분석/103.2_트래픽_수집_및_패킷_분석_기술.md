# 103.2 트래픽 수집 및 패킷 분석 기술

## 1. 분석 도구 활용 기법
### 1.1 Wireshark
- **Display Filter:** 분석 효율을 높이는 핵심 기능 (예: `http.request` 또는 `tcp.flags.syn == 1 && tcp.flags.ack == 0`).
- **Follow Stream:** TCP/UDP/HTTP 스트림을 재구성하여 공격자와 대상 시스템 간의 대화 내용을 텍스트 형태로 확인.
- **Expert Information:** 체크섬 오류, 재전송 등 네트워크 이상 징후 자동 식별.

### 1.2 Tshark 및 CLI 도구
- 대용량 PCAP 파일 분석 시 GUI의 한계를 극복하기 위해 사용합니다.
- `tshark -r file.pcap -Y "http.request" -T fields -e http.host`와 같은 명령으로 특정 필드만 빠르게 추출 가능합니다.

## 2. 계층별 프로토콜 이상 행위 식별
### 2.1 DNS (Domain Name System)
- **정상:** 짧은 쿼리와 단순 응답.
- **이상:** 비정상적으로 긴 서브도메인(Data Exfiltration 의심), 대량의 NXDOMAIN 응답(DGA 알고리즘 사용 의심).

### 2.2 HTTP/HTTPS
- **User-Agent 분석:** 표준 브라우저가 아닌 파이썬 스크립트나 커스텀 도구의 흔적 탐색.
- **Method 분석:** GET/POST 외에 PUT, OPTIONS 등 비정상적인 메소드 사용 확인.

### 2.3 SMB (Server Message Block)
- 내부망 이동(Lateral Movement) 시 가장 많이 악용되는 프로토콜.
- 비정상적인 파일 공유 접근 및 실행 파일(.exe, .ps1) 전송 이력 감시.
