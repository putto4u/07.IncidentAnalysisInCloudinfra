# 70640.1 DDoS 공격의 진화와 봇넷(Botnet)의 구조

## 1. DDoS 공격의 정의와 진화
DDoS(Distributed Denial of Service) 공격은 여러 대의 분산된 시스템(좀비 PC/봇)을 동원하여 타깃 시스템의 가용 리소스를 고갈시킴으로써 정상적인 서비스를 방해하는 공격입니다.

### 공격의 진화 단계
1.  **과거 (Flooding 위주):** 단순히 많은 양의 패킷을 보내 대역폭을 점유하는 방식 (예: Trinoo, TFN).
2.  **현재 (복합 및 지능형):** L7 공격, 증폭 공격, 보안 장비 무력화 공격 등이 결합된 다중 경로(Multi-vector) 공격.
3.  **미래 (IoT 및 AI 기반):** 보안이 취약한 IoT 기기를 대량 활용한 미라이(Mirai) 봇넷 및 AI를 이용한 탐지 우회.

## 2. 봇넷(Botnet)의 아키텍처
봇넷은 봇마스터가 원격에서 명령을 내리는 계층적 구조를 가집니다.

### 2.1 C2(Command and Control) 아키텍처 유형
*   **중앙 집중형 (IRC, HTTP):**
    *   **IRC:** 고전적인 방식으로 채팅 채널을 통해 명령 전달.
    *   **HTTP:** 웹 서버를 활용하여 명령을 수신. 탐지가 어렵고 일반 트래픽으로 위장이 용이함.
*   **분산형 (P2P):**
    *   중앙 C2 서버 없이 봇끼리 명령을 전파. 특정 서버 차단으로 봇넷을 와해시키기 어려움.
*   **Fast-Flux:**
    *   하나의 도메인에 수많은 IP를 짧은 주기로 변경하여 C2 서버의 실제 위치를 은닉.

### 2.2 핵심 구성 요소
1.  **봇마스터 (Botmaster):** 공격을 기획하고 명령을 내리는 주체.
2.  **C2 서버:** 명령을 전달하고 봇 상태를 관리하는 서버.
3.  **좀비 PC (Bot):** 악성코드에 감염되어 실제 공격을 수행하는 단말.
4.  **피해 대상 (Victim):** 공격의 타깃이 되는 서버나 서비스.
