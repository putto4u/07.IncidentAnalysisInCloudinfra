# 70640.6 탐지 우회 및 안티-분석 기법

## 1. 네트워크 트래픽 난독화
IDS/IPS 등의 보안 장비를 우회하기 위한 기법입니다.
*   **커스텀 프로토콜:** 표준 포트(80, 443)를 사용하면서 내부 데이터는 독자적인 암호화/난독화 알고리즘 사용.
*   **TLS/SSL 터널링:** 공격 명령을 HTTPS로 암호화하여 패킷 분석 차단.
*   **Padding 및 Jitter:** 패킷 크기를 무작위로 키우거나 전송 간격을 불규칙하게 조절하여 패턴 탐지 우회.

## 2. 안티-리버싱 (Anti-Reversing)
분석가가 코드를 분석하는 것을 방해하는 기법입니다.
*   **패킹(Packing) 및 난독화:** UPX 등 알려진 패커 또는 커스텀 패커를 사용하여 바이너리 가독성 저하.
*   **가상화/샌드박스 탐지:** 
    *   VMware, VirtualBox 특유의 파일, 레지스트리, MAC 주소 확인.
    *   `CPUID`, `IN` 명령 등을 통한 하드웨어 특성 체크.
*   **안티-디버깅 (Anti-Debugging):**
    *   `IsDebuggerPresent()` API 호출.
    *   `CheckRemoteDebuggerPresent()`, `NtQueryInformationProcess()`를 이용한 디버깅 상태 확인.
    *   `Exception handling`을 이용한 실행 흐름 왜곡.

## 3. 코드 동적 보호
*   **자기 암호화 (Self-Decryption):** 실행 중에만 코드 섹션을 복호화하여 정적 분석 무력화.
*   **API 후킹 탐지:** 분석 툴이 사용하는 시스템 API 후킹 여부를 체크하여 실행 중단.

## 4. 분석가 대응 전략
*   **Unpacking:** `OEP(Original Entry Point)`를 찾아 덤프 파일 생성.
*   **Patching:** 안티-디버깅 루틴을 `NOP` 처리하거나 조건문을 반전시켜 우회.
*   **ScyllaHide:** 디버거 탐지를 방지하는 플러그인 활용.
