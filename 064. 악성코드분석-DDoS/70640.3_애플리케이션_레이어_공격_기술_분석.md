# 70640.3 애플리케이션 레이어(L7) 공격 기술 분석

## 1. HTTP Flooding 공격
웹 서버의 동적 컨텐츠 처리 리소스를 고갈시키는 공격입니다.

### 1.1 HTTP GET/POST Flood
*   **GET Flood:** 인덱스 페이지나 검색 결과 페이지와 같이 서버 리소스를 많이 사용하는 URL을 무한 반복 호출.
*   **POST Flood:** 게시판 글 쓰기나 로그인 요청과 같이 DB 연동이 발생하는 요청을 대량으로 수행하여 DB 커넥션 풀을 고갈시킴.

## 2. Low & Slow 공격 기법
적은 양의 트래픽으로 연결 세션을 길게 유지하여 서버의 동시 접속 한계(Concurrent Connections)를 초과시키는 방식입니다.

### 2.1 Slowloris (Slow HTTP Header DoS)
*   **원리:** HTTP 헤더의 끝을 알리는 빈 라인(`\r\n\r\n`)을 보내지 않고, 비정상적인 헤더를 조금씩 지속적으로 보내 서버가 헤더 수신을 완료하기 위해 세션을 계속 유지하게 만듦.

### 2.2 R.U.D.Y (R-U-Dead-Yet? / Slow HTTP POST)
*   **원리:** POST 요청 시 `Content-Length`를 크게 설정한 후, 실제 데이터를 아주 느린 속도(예: 수 초당 1바이트)로 보내 서버의 소켓을 장시간 점유함.

### 2.3 Slow Read DoS
*   **원리:** 서버가 보내주는 응답 데이터를 아주 느리게 읽음으로써 서버의 송신 버퍼와 세션 리소스를 고갈시킴.

## 3. 정상 트래픽과의 변별점 식별
L7 공격은 패킷 하나하나가 정상적이므로 통계적 분석이 중요합니다.

*   **User-Agent 일관성:** 봇넷에 설정된 특정 User-Agent 문자열이 반복적으로 관찰됨.
*   **요청 간격:** 정상 사용자와 달리 기계적으로 일정한 주기로 요청이 들어옴.
*   **Referer 부재 또는 고정:** 정상적인 페이지 이동 경로(Referer)를 거치지 않고 직접 특정 URL만 반복 호출.
*   **비정상적인 페이지 요청 순서:** 로그인 페이지 접근 없이 바로 로그인 API를 호출하는 등의 행위.
