# 70640.2 네트워크 레이어 공격 기술 분석

## 1. 전송 계층(L4) 기반 Flooding 공격
네트워크 대역폭을 소모하거나 서버의 세션 연결 리소스를 고갈시키는 방식입니다.

### 1.1 SYN Flood
*   **원리:** TCP 3-Way Handshake 과정에서 클라이언트가 SYN 패킷을 보낸 후 서버의 SYN+ACK 응답에 대해 ACK를 보내지 않아 서버의 Half-Open 연결 큐를 가득 채우는 공격.
*   **패킷 특징:** 다수의 Spoofed IP(위조된 IP)를 사용하여 비정상적인 세션 요청 생성.

### 1.2 UDP Flood
*   **원리:** 타깃의 임의의 포트로 대량의 UDP 패킷을 전송하여 타깃이 해당 포트가 열려있는지 확인하고 ICMP Destination Unreachable 응답을 생성하게 하여 시스템 리소스를 소모시킴.
*   **특징:** 세션 연결이 필요 없으므로 매우 빠른 속도로 패킷 생성이 가능.

## 2. 증폭 공격 (Amplification Attack)
적은 양의 요청으로 거대한 응답을 유도하여 타깃을 공격하는 방식입니다.

### 2.1 주요 증폭 공격 유형
*   **DNS Amplification:** 소스 IP를 타깃 IP로 변조하여 DNS 서버에 대량의 레코드 조회를 요청. 타깃은 거대한 DNS 응답 패킷을 받게 됨.
*   **NTP Amplification:** `monlist` 명령을 사용하여 서버의 최근 접속 목록을 요청. 요청 대비 응답 크기가 수십 배 이상 큼.
*   **SNMP/Memcached Amplification:** UDP 기반 서비스들의 응답 특성을 이용한 대규모 트래픽 생성.

## 3. 패킷 분석 및 식별
*   **동일 패턴 반복:** 페이로드에 특정 문자열이나 0x00, 0xFF 등의 데이터가 반복됨.
*   **비정상적인 트래픽 급증:** PPS(Packet Per Second)와 대역폭(BPS)이 임계치를 초과.
*   **Fragmentation 공격:** IP 단편화(Fragment)를 이용해 방화벽의 검사를 우회하거나 서버의 재조립 리소스를 소모시킴.
