# 70640.4 DDoS 악성코드 동적 분석

## 1. 분석 환경 구성
DDoS 악성코드는 실제 공격 트래픽을 대량으로 발생시키므로 격리된 샌드박스 환경이 필수적입니다.
*   **Virtual Machine:** 호스트와 네트워크가 격리된 상태.
*   **INetSim:** 가상 네트워크 서비스를 제공하여 악성코드가 외부 C2와 통신하는 것처럼 속임.
*   **Wireshark/Fiddler:** 발생하는 패킷을 캡처하고 분석.

## 2. 네트워크 소켓 및 스레드 분석
### 2.1 소켓 생성 로직
악성코드는 대량의 패킷을 보내기 위해 여러 개의 소켓을 동시에 생성합니다.
*   `socket()`, `connect()`, `sendto()` 함수 호출 추적.
*   Raw Socket 사용 여부 확인 (IP 헤더 위조를 위해 사용됨).

### 2.2 멀티스레딩 및 멀티프로세싱
공격 성능을 극대화하기 위해 다중 스레드를 활용합니다.
*   `CreateThread()` (Windows) 또는 `pthread_create()` (Linux) 호출 확인.
*   스레드 내에서 무한 루프(`while(1)`)를 돌며 패킷 전송을 수행하는지 분석.

## 3. 명령 수신 및 파싱 루틴
C2로부터 어떤 명령을 받고 어떻게 해석하는지 분석하는 것이 핵심입니다.
*   **C2 주소 식별:** 바이너리 내에 하드코딩된 IP/Domain 또는 DGA(Domain Generation Algorithm) 분석.
*   **커맨드 구조:**
    *   `ATTACK_TYPE`: SYN, UDP, HTTP 등 공격 유형 코드.
    *   `TARGET_IP`: 공격 대상 주소.
    *   `DURATION`: 공격 지속 시간.
    *   `THREADS`: 사용할 스레드 수.
*   **파싱 로직 역공학:** `switch-case` 문이나 `if-else` 문을 통해 수신된 바이트 데이터를 공격 명령으로 변환하는 코드 영역 식별.

## 4. 모니터링 포인트
*   **Process Hacker:** CPU 사용량 급증 및 다수 소켓 연결 상태 모니터링.
*   **API Monitor:** `ws2_32.dll` (Windows)의 네트워크 API 호출 파라미터(Target IP, Port) 확인.
