# 70640.5 자동 실행 및 자가 복제 기능 분석

## 1. 초기 감염 및 전파 (Propagation)
DDoS 봇은 세력을 확장하기 위해 새로운 취약점을 스캐닝하고 전파되는 특성을 가집니다.

### 1.1 취약점 스캐닝 (Scanning)
*   **임의 IP 생성:** 무작위로 IP 대역을 생성하여 특정 포트(예: 22, 23, 80, 445)가 열려있는지 확인.
*   **Brute-Force 공격:** SSH/Telnet 등에 흔히 쓰이는 계정 정보를 대입하여 로그인 시도 (예: Mirai 봇넷의 기본 계정 리스트).
*   **RCE 취약점 이용:** 보안 패치가 되지 않은 웹 서비스나 IoT 기기의 원격 코드 실행(RCE) 취약점을 공격.

### 1.2 자가 복제 (Self-Replication)
*   침투에 성공하면 자신의 복사본을 다운로드(`wget`, `curl`)하거나 쉘 코드를 통해 직접 주입.

## 2. 지속성 확보 (Persistence)
시스템이 재부팅되어도 다시 실행되도록 하는 기법입니다.

### 2.1 등록 방식
*   **Registry (Windows):** `Run`, `RunOnce` 키에 자신을 등록.
*   **Service (Windows/Linux):** 시스템 서비스로 등록하여 배경에서 자동 실행.
*   **Crontab (Linux):** 일정 주기마다 실행되도록 스케줄링.
*   **Init Scripts:** `/etc/init.d/` 또는 `systemd` 유닛 파일 생성.

## 3. 실행 파일 은닉 및 보호
*   **Process Masquerading:** 정상 프로세스명(예: `svchost.exe`, `httpd`)으로 위장하여 실행.
*   **File Hiding:** 속성을 숨김(`+h`)으로 설정하거나, 루트킷 기능을 사용하여 파일 목록에서 제거.
*   **Dropper/Loader 사용:** 본체 파일은 암호화된 상태로 존재하다가 메모리 상에서만 실행(Fileless 기법 연계).

## 4. 분석 방법
*   **Autoruns (Sysinternals):** 자동 실행 설정의 변경 사항 확인.
*   **Procmon:** 파일 시스템 및 레지스트리 쓰기 행위 실시간 캡처.
